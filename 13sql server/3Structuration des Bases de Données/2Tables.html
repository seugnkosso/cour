1. Création de tables :

Pour créer une table dans SQL Server Management Studio (SSMS), vous pouvez utiliser la syntaxe SQL suivante :

sql
CREATE TABLE nom_de_la_table (
    nom_colonne1 type_de_donnée_contraintes,
    nom_colonne2 type_de_donnée_contraintes,
    ...
    CONSTRAINT nom_contrainte PRIMARY KEY (nom_colonne)
);

Par exemple :

sql
CREATE TABLE Employes (
    ID INT PRIMARY KEY,
    Nom VARCHAR(50),
    Age INT,
    Salaire DECIMAL(10, 2)
);

Cela crée une table nommée "Employes" avec quatre colonnes : ID, Nom, Age et Salaire.

2. Types de données :

SQL Server propose divers types de données pour répondre à différents besoins. Voici quelques types de données courants :

  • INT : Entier (de -2^31 (-2,147,483,648) à 2^31-1 (2,147,483,647))
  • VARCHAR(n) : Chaîne de caractères de longueur variable jusqu'à n caractères
  • DECIMAL(p, s) : Nombre décimal avec p chiffres au total et s chiffres après la virgule
  • DATE : Date (AAAA-MM-JJ)
  • TIME : Heure (HH:MM:SS)
  • DATETIME : Date et heure (AAAA-MM-JJ HH:MM:SS)
  • BIT : Valeur booléenne (0 ou 1)

3. Clés primaires :

Une clé primaire est une colonne ou un ensemble de colonnes qui identifient de manière unique chaque enregistrement dans une table. Pour définir une clé primaire lors de la création de table, utilisez la clause PRIMARY KEY.

Exemple :

sql
CREATE TABLE Employes (
    ID INT PRIMARY KEY,
    Nom VARCHAR(50),
    Age INT,
    Salaire DECIMAL(10, 2)
);

Dans cet exemple, la colonne ID est définie comme clé primaire. Cela garantit que chaque valeur dans la colonne ID est unique pour chaque enregistrement dans la table.

Vous pouvez également ajouter une contrainte de clé primaire à une table existante à l'aide de la syntaxe ALTER TABLE :

sql
ALTER TABLE nom_de_la_table
ADD CONSTRAINT nom_contrainte PRIMARY KEY (nom_colonne);

Assurez-vous que les valeurs de la colonne clé primaire sont uniques pour chaque ligne. Sinon, l'insertion de données échouera avec une violation de contrainte de clé primaire.

Ce tutoriel devrait vous aider à démarrer avec la création de tables, les types de données et les clés primaires dans SQL Server Management Studio (SSMS). N'hésitez pas à explorer davantage pour en apprendre plus sur d'autres fonctionnalités et concepts SQL Server.

4. Clés étrangères :

Les clés étrangères sont utilisées pour établir une relation entre deux tables. Elles garantissent l'intégrité référentielle des données en veillant à ce qu'une valeur dans une colonne (clé étrangère) corresponde à une valeur existante dans une autre colonne (clé primaire) dans une table différente.

Voici un exemple de création de table avec une clé étrangère :

sql
CREATE TABLE Commandes (
    ID INT PRIMARY KEY,
    ID_Client INT,
    DateCommande DATE,
    FOREIGN KEY (ID_Client) REFERENCES Clients(ID)
);

Dans cet exemple, la colonne ID_Client dans la table Commandes est une clé étrangère qui fait référence à la colonne ID dans la table Clients. Cela garantit que chaque commande est associée à un client existant dans la table des clients.

5. Contraintes de vérification :

Les contraintes de vérification sont utilisées pour garantir que les valeurs dans une colonne respectent certaines conditions spécifiées. Par exemple, vous pouvez vous assurer qu'une colonne ne contient que des valeurs comprises dans une plage spécifique ou respectant un format particulier.

Exemple :

sql
CREATE TABLE Employes (
    ID INT PRIMARY KEY,
    Nom VARCHAR(50),
    Age INT CHECK (Age >= 18),
    Salaire DECIMAL(10, 2) CHECK (Salaire >= 0)
);

Dans cet exemple, les contraintes CHECK sont utilisées pour s'assurer que l'âge des employés est supérieur ou égal à 18 et que le salaire est positif.

6. Contraintes par défaut :

Les contraintes par défaut spécifient les valeurs par défaut pour les nouvelles lignes d'une table si aucune valeur n'est spécifiée explicitement lors de l'insertion de données.

Exemple :

sql
CREATE TABLE Employes (
    ID INT PRIMARY KEY,
    Nom VARCHAR(50),
    Age INT,
    Salaire DECIMAL(10, 2),
    DateEmbauche DATE DEFAULT GETDATE()
);

Dans cet exemple, la colonne DateEmbauche aura la date actuelle par défaut si aucune date n'est spécifiée lors de l'insertion d'un nouvel employé.

En utilisant les clés étrangères, les contraintes de vérification et les contraintes par défaut, vous pouvez garantir l'intégrité et la cohérence des données dans votre base de données SQL Server.
