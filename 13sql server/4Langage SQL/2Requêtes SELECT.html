1. Installation et Configuration de Microsoft SQL Server et SSMS :

Avant de commencer à écrire des requêtes SQL, assurez-vous d'avoir installé Microsoft SQL Server et SSMS. Vous pouvez les télécharger depuis le site Web de Microsoft et les installer sur votre machine.

2. Connexion à la Base de Données :

Après avoir installé SSMS, lancez-le et connectez-vous à votre serveur SQL Server en utilisant vos identifiants. Une fois connecté, vous pouvez commencer à exécuter des requêtes SQL.

3. Sélection de Données Simples :

Pour sélectionner des données simples à partir d'une table, vous pouvez utiliser la syntaxe suivante :

sql
SELECT colonne1, colonne2
FROM nom_table;

Par exemple, pour sélectionner tous les enregistrements de la table "Employés" avec leurs noms et prénoms :

sql
SELECT Nom, Prénom
FROM Employés;

4. Utilisation de l'opérateur WHERE :

L'opérateur WHERE est utilisé pour filtrer les enregistrements en fonction d'une condition spécifique. Voici comment l'utiliser :

sql
SELECT colonne1, colonne2
FROM nom_table
WHERE condition;

Par exemple, pour sélectionner les employés dont le salaire est supérieur à 50000 :

sql
SELECT Nom, Prénom
FROM Employés
WHERE Salaire > 50000;

5. Utilisation de l'opérateur WHERE avec des Opérateurs Logiques :

Vous pouvez utiliser des opérateurs logiques tels que AND, OR et NOT avec l'opérateur WHERE pour créer des conditions complexes. Par exemple :

sql
SELECT colonne1, colonne2
FROM nom_table
WHERE condition1 AND/OR condition2;

Par exemple, pour sélectionner les employés dont le salaire est supérieur à 50000 et qui travaillent dans le département "Ventes" :

sql
SELECT Nom, Prénom
FROM Employés
WHERE Salaire > 50000 AND Département = 'Ventes';

Ceci constitue une introduction de base à l'utilisation de SELECT et WHERE dans Microsoft SQL Server avec SSMS. Vous pouvez explorer davantage les fonctionnalités avancées telles que les fonctions intégrées, les jointures, les sous-requêtes, etc., pour approfondir vos compétences en SQL.

C. Utilisation des Opérateurs Logiques (AND, OR, NOT) :

Les opérateurs logiques sont utilisés pour combiner des conditions dans les clauses WHERE. Voici comment ils fonctionnent :

  • AND : Retourne des résultats si toutes les conditions sont vraies.
  • OR : Retourne des résultats si au moins l'une des conditions est vraie.
  • NOT : Retourne des résultats si la condition est fausse.

Exemple d'utilisation :

sql
SELECT colonne1, colonne2
FROM nom_table
WHERE condition1 AND/OR/NOT condition2;

Par exemple, pour sélectionner les employés dont le salaire est supérieur à 50000 et qui travaillent dans le département "Ventes", ou ceux dont le salaire est inférieur à 30000 :

sql
SELECT Nom, Prénom
FROM Employés
WHERE (Salaire > 50000 AND Département = 'Ventes') OR Salaire < 30000;

D. Utilisation de l'Opérateur LIKE pour la Recherche de Motifs :

L'opérateur LIKE est utilisé pour effectuer des recherches de motifs dans les données textuelles. Il est souvent utilisé avec les caractères génériques '%' (pour représenter zéro, un ou plusieurs caractères) et '_' (pour représenter un seul caractère).

Exemple d'utilisation :

sql
SELECT colonne1, colonne2
FROM nom_table
WHERE colonne LIKE 'motif';

Par exemple, pour sélectionner les employés dont le nom commence par "Smi" :

sql
SELECT Nom, Prénom
FROM Employés
WHERE Nom LIKE 'Smi%';

Cela sélectionnera tous les employés dont le nom commence par "Smi".

E. Utilisation des Fonctions Agrégées (SUM, AVG, COUNT, MIN, MAX) :

Les fonctions agrégées sont utilisées pour effectuer des calculs sur un ensemble de valeurs et retourner un résultat unique. Voici quelques-unes des fonctions agrégées les plus couramment utilisées :

  • SUM() : Calcule la somme des valeurs.
  • AVG() : Calcule la moyenne des valeurs.
  • COUNT() : Compte le nombre d'enregistrements.
  • MIN() : Retourne la valeur minimale.
  • MAX() : Retourne la valeur maximale.

Exemple d'utilisation :

sql
SELECT SUM(colonne) AS somme, AVG(colonne) AS moyenne, COUNT(colonne) AS total, MIN(colonne) AS minimum, MAX(colonne) AS maximum
FROM nom_table;

Par exemple, pour calculer la somme, la moyenne, le nombre total, la valeur minimale et la valeur maximale du salaire des employés :

sql
SELECT SUM(Salaire) AS Somme_Salaire, AVG(Salaire) AS Moyenne_Salaire, COUNT(*) AS Nombre_Employés, MIN(Salaire) AS Salaire_Minimum, MAX(Salaire) AS Salaire_Maximum
FROM Employés;

F. Groupement de Données avec GROUP BY :

La clause GROUP BY est utilisée pour regrouper les enregistrements qui ont des valeurs communes dans une ou plusieurs colonnes. Elle est souvent utilisée avec les fonctions agrégées pour effectuer des calculs sur chaque groupe de résultats.

Exemple d'utilisation :

sql
SELECT colonne1, fonction_agrégée(colonne2)
FROM nom_table
GROUP BY colonne1;

Par exemple, pour calculer la somme du salaire de chaque département :

sql
SELECT Département, SUM(Salaire) AS Somme_Salaire
FROM Employés
GROUP BY Département;

G. Filtres de Groupes avec HAVING :

La clause HAVING est utilisée pour filtrer les résultats d'une requête GROUP BY basée sur une condition spécifiée. Elle agit comme une clause WHERE pour les groupes.

Exemple d'utilisation :

sql
SELECT colonne1, fonction_agrégée(colonne2)
FROM nom_table
GROUP BY colonne1
HAVING condition;

Par exemple, pour sélectionner les départements dont le salaire moyen est supérieur à 50000 :

sql
SELECT Département, AVG(Salaire) AS Moyenne_Salaire
FROM Employés
GROUP BY Département
HAVING AVG(Salaire) > 50000;

Cela sélectionnera uniquement les départements dont la moyenne des salaires est supérieure à 50000.
