A. Concepts de base des audits de sécurité

1. Qu'est-ce qu'un audit de sécurité ?

Un audit de sécurité dans Microsoft SQL Server est un mécanisme permettant de surveiller et d'enregistrer les actions effectuées sur une base de données ou sur le serveur lui-même. Cela inclut les tentatives de connexion, les modifications de données, les créations d'objets, etc.

2. Les éléments clés d'un audit de sécurité :

a. Spécifications d'audit :

  • Définit les actions à auditer (par exemple, SELECT, INSERT, UPDATE, DELETE).
  • Peut être configuré au niveau de la base de données ou du serveur.

b. Audit :

  • Contient des informations sur les événements capturés.
  • Peut stocker les données d'audit dans des fichiers ou des tables.

3. Les types d'audits de sécurité :

a. Audits de base de données :

  • Capturer les actions au niveau de la base de données, comme les modifications de données.

b. Audits du serveur :

  • Surveiller les actions au niveau du serveur, telles que les tentatives de connexion.

B. Configuration des audits dans SQL Server

1. Utilisation de SQL Server Management Studio (SSMS) :

a. Connexion à SSMS :

  • Lancez SQL Server Management Studio.
  • Connectez-vous à votre serveur SQL Server en utilisant vos informations d'identification.

b. Création d'une spécification d'audit :

  1. Dans l'Explorateur d'objets, cliquez avec le bouton droit sur "Audit" sous le serveur ou la base de données où vous souhaitez créer l'audit.
  2. Sélectionnez "Nouvelle spécification d'audit".
  3. Définissez les actions à auditer et spécifiez où stocker les données d'audit.

c. Activation de l'audit :

  1. Cliquez avec le bouton droit sur la spécification d'audit que vous venez de créer.
  2. Choisissez "Activer".

d. Consultation des données d'audit :

  1. Dans l'Explorateur d'objets, développez "Audit".
  2. Cliquez avec le bouton droit sur l'audit que vous souhaitez consulter.
  3. Sélectionnez "Afficher les fichiers journaux d'audit" pour afficher les données d'audit stockées dans les fichiers.

2. Utilisation des commandes T-SQL :

a. Création d'une spécification d'audit :

sql
USE [master];
CREATE SERVER AUDIT [AuditName]
TO FILE (FILEPATH = 'C:\SQLAudits\')
WITH (ON_FAILURE = CONTINUE);

b. Activation de l'audit :

sql
ALTER SERVER AUDIT [AuditName] WITH (STATE = ON);

c. Consultation des données d'audit :

sql
SELECT * FROM fn_get_audit_file('C:\SQLAudits\*.sqlaudit', NULL, NULL);

Cela devrait vous donner une bonne base pour commencer à utiliser les audits de sécurité dans Microsoft SQL Server avec SSMS. N'hésitez pas à explorer davantage les fonctionnalités pour répondre à vos besoins spécifiques.

C. Types d'événements auditables

1. Connexions :

  • Tentatives de connexion réussies ou échouées à la base de données ou au serveur.

2. Modifications de données :

  • INSERT, UPDATE, DELETE : Modifications apportées aux données dans les tables.

3. Modifications de schéma :

  • CREATE, ALTER, DROP : Modifications de la structure des objets de base de données.

4. Autorisations :

  • GRANT, REVOKE : Octroi ou révocation des autorisations sur des objets de base de données.

5. Événements du serveur :

  • Arrêt/Redémarrage du serveur, changements de configuration, etc.

D. Consultation et analyse des résultats d'audit

1. Utilisation de SSMS :

a. Affichage des résultats d'audit :

  • Dans l'Explorateur d'objets, développez l'audit que vous souhaitez consulter.
  • Cliquez sur "Afficher les fichiers journaux d'audit".
  • Double-cliquez sur un fichier pour l'ouvrir et afficher son contenu.

b. Analyse des résultats :

  • Utilisez les filtres et les options de tri pour rechercher des événements spécifiques.
  • Identifiez les tendances, les schémas ou les activités suspectes.

2. Utilisation de commandes T-SQL :

a. Consultation des données d'audit :

  • Utilisez la fonction fn_get_audit_file pour lire les fichiers journaux d'audit.
  • Utilisez des prédicats et des clauses WHERE pour filtrer les résultats selon vos besoins.

b. Exemple de requête T-SQL pour consulter les résultats :

sql
SELECT event_time, succeeded, server_principal_name, database_principal_name, statement
FROM fn_get_audit_file('C:\SQLAudits\*.sqlaudit', DEFAULT, DEFAULT)
WHERE action_id = 'LO';

3. Utilisation de rapports SSMS :

a. Rapports prédéfinis :

  • SSMS propose des rapports prédéfinis pour l'analyse des données d'audit.
  • Accédez à "Rapports" dans SSMS, puis à "Gestion du serveur", et choisissez parmi les rapports disponibles.

b. Personnalisation des rapports :

  • Vous pouvez également créer des rapports personnalisés en utilisant les données d'audit comme source.

En utilisant ces méthodes, vous pouvez consulter et analyser les résultats des audits de sécurité dans Microsoft SQL Server. Assurez-vous de comprendre les événements auditables pertinents pour votre environnement et d'utiliser les outils disponibles pour une analyse approfondie.
