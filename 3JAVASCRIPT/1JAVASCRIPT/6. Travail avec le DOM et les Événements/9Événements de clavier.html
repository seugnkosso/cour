1. Introduction

Le Document Object Model (DOM) est une représentation hiérarchique des éléments HTML/XML d'une page web, facilitant la manipulation de ces éléments par JavaScript. Les événements de clavier dans le DOM permettent de détecter et de réagir aux actions de l'utilisateur sur le clavier.

2. Structure du DOM liée aux événements de clavier

Le DOM organise les éléments de la page sous forme d'une structure arborescente, où chaque élément est représenté par un nœud. Les événements de clavier sont déclenchés sur des éléments spécifiques du DOM. Voici une structure simple :

3. Types d'événements de clavier

3.1 keydown

L'événement keydown est déclenché lorsque la touche d'un clavier est enfoncée.

javascript
document.getElementById('myInput').addEventListener('keydown', function(event) {
  console.log('keydown event:', event.key);
});

3.2 keyup

L'événement keyup est déclenché lorsque la touche d'un clavier est relâchée.

javascript
document.getElementById('myInput').addEventListener('keyup', function(event) {
  console.log('keyup event:', event.key);
});

3.3 keypress

L'événement keypress était utilisé pour détecter la pression d'une touche, mais il est maintenant obsolète dans de nombreux navigateurs. Il est recommandé d'utiliser keydown ou keyup à la place.

4. Différences entre keydown, keyup et keypress

Les principaux points de distinction entre keydown, keyup et l'ancien keypress sont :

  • keydown : Déclenché lorsque la touche est enfoncée.
  • keyup : Déclenché lorsque la touche est relâchée.
  • keypress : Obsolète dans de nombreux navigateurs, remplacé par keydown et keyup. Historiquement utilisé pour détecter la pression d'une touche après que la touche a été enfoncée.

5. Conclusion

Les événements de clavier dans le DOM sont essentiels pour créer des interactions utilisateur dynamiques. En comprenant la structure du DOM et en utilisant les événements keydown et keyup, vous pouvez créer des fonctionnalités réactives et conviviales sur vos pages web.

1. Introduction

Les propriétés d'événements de clavier fournissent des informations détaillées sur l'événement clavier en cours. En utilisant ces propriétés, vous pouvez personnaliser votre logique de gestion d'événements.

2. Propriétés communes à tous les événements clavier

2.1. event.key

La propriété key contient la valeur de la touche pressée ou relâchée sous forme de chaîne de caractères.

javascript
document.addEventListener('keydown', function(event) {
  console.log('Touche enfoncée :', event.key);
});

2.2. event.code

La propriété code contient la valeur de la touche physique du clavier, indépendamment de la disposition du clavier.

javascript
document.addEventListener('keydown', function(event) {
  console.log('Code de la touche :', event.code);
});

2.3. event.ctrlKey, event.altKey, event.shiftKey, event.metaKey

Ces propriétés booléennes indiquent si les touches Ctrl, Alt, Shift ou Meta (Commande sur Mac) sont enfoncées pendant l'événement.

javascript
document.addEventListener('keydown', function(event) {
  if (event.ctrlKey) {
    console.log('La touche Ctrl est enfoncée.');
  }
});

3. Utilisation des propriétés spécifiques à keydown, keyup et keypress

3.1. Propriétés spécifiques à keydown et keyup

3.1.1. event.keyCode

La propriété keyCode contient le code numérique de la touche pressée ou relâchée. Cependant, elle est obsolète et est remplacée par key dans les nouvelles implémentations.

3.1.2. event.repeat

La propriété repeat indique si l'événement est généré à la suite d'une répétition automatique due au maintien enfoncé d'une touche.

3.2. Propriétés spécifiques à keypress (obsolète)

3.2.1. event.charCode

La propriété charCode contenait le code Unicode du caractère associé à la touche pressée lors d'un événement keypress. Cependant, elle est obsolète et remplacée par key ou code.

  console.log('Shift :', event.shiftKey);
  console.log('Meta :', event.metaKey);

  // Utilisation spécifique à keydown/keyup
  console.log('Repeat :', event.repeat);

  // Utilisation spécifique à keypress (obsolète)
  console.log('CharCode :', event.charCode);
});

5. Conclusion

En comprenant les propriétés communes à tous les événements clavier et les propriétés spécifiques à keydown, keyup et keypress (obsolète), vous pouvez créer des fonctionnalités plus avancées et réactives en fonction des touches spécifiques enfoncées par l'utilisateur. Cependant, il est recommandé d'utiliser key à la place de keyCode et charCode pour garantir une compatibilité avec les dernières implémentations.
