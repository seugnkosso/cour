1. Définition des Fonctions

En JavaScript, une fonction est un bloc de code réutilisable qui peut être appelé à partir d'autres parties du programme. Une fonction peut prendre des paramètres en entrée, effectuer des opérations, et renvoyer un résultat.

Exemple de Déclaration de Fonction :
javascript
// Déclaration de fonction
function additionner(a, b) {
    return a + b;
}

// Appel de fonction
let resultat = additionner(5, 3);
console.log(resultat); // Affiche 8

2. Avantages de l'Utilisation de Fonctions

Les fonctions offrent plusieurs avantages dans le développement logiciel :

  • Réutilisabilité : Une fonction peut être appelée plusieurs fois à partir de différentes parties du programme, évitant ainsi la redondance du code.

  • Modularité : Les fonctions permettent de diviser un programme en modules logiques, facilitant la maintenance et la compréhension du code.

  • Abstraction : En utilisant des fonctions, on peut encapsuler la logique et les détails d'implémentation, fournissant une abstraction plus élevée.

Exemple d'Avantages :
javascript
// Sans fonction
let total = 0;
for (let i = 1; i <= 10; i++) {
    total += i;
}
console.log(total); // Affiche 55

// Avec fonction
function calculerSomme(n) {
    let somme = 0;
    for (let i = 1; i <= n; i++) {
        somme += i;
    }
    return somme;
}

let resultat = calculerSomme(10);
console.log(resultat); // Affiche 55

3. Déclaration vs Expression de Fonctions

En JavaScript, il existe deux façons de créer des fonctions : la déclaration de fonction et l'expression de fonction.

3.1 Déclaration de Fonction :
javascript
// Déclaration de fonction
function multiplier(x, y) {
    return x * y;
}

Dans ce cas, la fonction est déclarée avant toute exécution de code. Elle peut être appelée n'importe où dans le script.

3.2 Expression de Fonction :
javascript
// Expression de fonction
let diviser = function(a, b) {
    return a / b;
};

Dans ce cas, la fonction est assignée à une variable. L'expression de fonction doit être définie avant son appel.

Exemple d'Utilisation :
javascript
console.log(multiplier(2, 3)); // Appel de la fonction déclarée
console.log(diviser(10, 2));    // Appel de la fonction exprimée

Il est important de noter que les déclarations de fonction sont « hissées » (hoisting), ce qui signifie qu'elles sont déplacées au début de leur contexte d'exécution.

5.1 Portée Globale vs Locale

La portée des variables fait référence à la zone du code où une variable peut être utilisée. En JavaScript, les variables peuvent être définies avec une portée globale ou locale.

  • Portée Globale : Une variable globale est déclarée en dehors de toutes les fonctions et peut être accédée de n'importe où dans le code.

    javascript
    let variableGlobale = "Je suis une variable globale";

function afficherGlobale() {
    console.log(variableGlobale);
}

afficherGlobale(); // Affiche "Je suis une variable globale"

  • Portée Locale : Une variable locale est déclarée à l'intérieur d'une fonction et n'est accessible que dans cette fonction.

    javascript
    function afficherLocale() {
    let variableLocale = "Je suis une variable locale";
    console.log(variableLocale);
}

afficherLocale(); // Affiche "Je suis une variable locale"
// console.log(variableLocale); // Cela générera une erreur car variableLocale n'est pas définie ici

5.2 Closures

Les closures se produisent lorsque des fonctions sont définies à l'intérieur d'une autre fonction et qu'elles ont accès aux variables de la fonction parente, même après que la fonction parente ait terminé son exécution.

  • Exemple de closure :

    javascript
    function creerIncrementer(pas) {
    let compteur = 0;

    function incrementer() {
        compteur += pas;
        console.log(compteur);
    }

    return incrementer;
}

let incrementerParDeux = creerIncrementer(2);
incrementerParDeux(); // Affiche 2
incrementerParDeux(); // Affiche 4

5.3 Lexical Scoping

Le Lexical Scoping se réfère à la façon dont la portée des variables est déterminée en fonction de leur emplacement physique dans le code source. Les fonctions en JavaScript suivent le scoping lexical, ce qui signifie qu'elles ont accès aux variables déclarées dans leur portée parente.

  • Exemple de Lexical Scoping :

    javascript
    function externe() {
    let variableExterne = "Je suis une variable externe";

    function interne() {
        console.log(variableExterne);
    }

    interne(); // Affiche "Je suis une variable externe"
}

6.1 Définition des Fonctions Récursives

Une fonction récursive est une fonction qui s'appelle elle-même, soit directement, soit indirectement à travers une série de rappels. La récursivité est souvent utilisée pour résoudre des problèmes qui peuvent être décomposés en des sous-problèmes de taille identique ou similaire.

6.2 Exemples Pratiques de Fonctions Récursives

  • Exemple de Calcul de Factorielle :

    La factorielle d'un nombre n (notée n!) est le produit de tous les entiers de 1 à n.

    javascript
    function factorielle(n) {
    if (n === 0 || n === 1) {
        return 1;
    } else {
        return n * factorielle(n - 1);
    }
}

let resultat = factorielle(5);
console.log(resultat); // Affiche 120

  • Exemple de Calcul de Fibonacci :

    La suite de Fibonacci est une séquence où chaque nombre est la somme des deux précédents (0, 1, 1, 2, 3, 5, 8, ...).

    javascript
    function fibonacci(n) {
    if (n <= 1) {
        return n;
    } else {
        return fibonacci(n - 1) + fibonacci(n - 2);
    }
}

let resultatFibonacci = fibonacci(6);
console.log(resultatFibonacci); // Affiche 8

6.3 Optimisation de la Récursivité

Les fonctions récursives peuvent être gourmandes en termes de ressources, en particulier pour des entrées importantes. L'optimisation de la récursivité peut être réalisée en utilisant la "mémoïsation" ou en convertissant la récursivité en itérativité.

  • Mémoïsation pour Optimiser la Récursivité :

    La mémoïsation implique le stockage en mémoire des résultats de fonctions récursives déjà calculés pour éviter de recalculer les mêmes résultats.

    javascript
    function fibonacciMemo(n, memo = {}) {
    if (n <= 1) {
        return n;
    }

    if (!memo[n]) {
        memo[n] = fibonacciMemo(n - 1, memo) + fibonacciMemo(n - 2, memo);
    }

    return memo[n];
}

let resultatFibMemo = fibonacciMemo(6);
console.log(resultatFibMemo); // Affiche 8

  • Conversion de Récursivité en Itérativité :

    Parfois, une approche itérative peut être plus efficace que la récursivité. Dans l'exemple de Fibonacci, cela peut être fait à l'aide d'une boucle.

    javascript
    function fibonacciIteratif(n) {
    let a = 0, b = 1, temp;

    for (let i = 2; i <= n; i++) {
        temp = a + b;
        a = b;
        b = temp;
    }

    return b;
}

let resultatFibIteratif = fibonacciIteratif(6);
console.log(resultatFibIteratif); // Affiche 8

8.1 Introduction aux Callbacks

Les callbacks sont des fonctions qui sont passées comme arguments à d'autres fonctions. Ils sont souvent utilisés dans des opérations asynchrones pour gérer le résultat d'une tâche une fois qu'elle est terminée.

    console.log("Callback exécuté !");
}

effectuerTacheAsync(callbackApresTache);

8.2 Utilisation des Promesses

Les promesses sont un moyen de gérer les opérations asynchrones de manière plus lisible et structurée. Elles représentent une valeur qui peut être disponible maintenant, dans le futur ou jamais.

  • Exemple de Promesse :

    javascript
    function effectuerTacheAsync() {
    return new Promise((resolve, reject) => {
        setTimeout(() => {
            console.log("Tâche effectuée !");
            resolve();
        }, 1000);
    });
}

effectuerTacheAsync()
    .then(() => {
        console.log("Promesse résolue !");
    })
    .catch((erreur) => {
        console.error("Erreur :", erreur);
    });

Les fonctions asynchrones, déclarées avec async, simplifient l'écriture du code asynchrone en utilisant l'opérateur await pour attendre la résolution d'une promesse.

        console.error("Erreur :", erreur);

9.1 Introduction aux Générateurs

Les fonctions génératrices sont une fonction spéciale qui permet de générer une séquence de valeurs itérables. Elles sont définies en utilisant la syntaxe function* et contiennent des instructions yield pour produire des valeurs au fur et à mesure.

9.2 Utilisation de l'instruction yield

L'instruction yield est utilisée à l'intérieur d'une fonction génératrice pour produire une valeur et mettre en pause l'exécution de la fonction. À chaque appel suivant de la méthode next(), la fonction reprend son exécution à partir du dernier yield.

  • Exemple de Fonction Génératrice :

    javascript
    function* genererSequence() {
    yield 1;
    yield 2;
    yield 3;
}

let sequence = genererSequence();

console.log(sequence.next().value); // Affiche 1
console.log(sequence.next().value); // Affiche 2
console.log(sequence.next().value); // Affiche 3

9.3 Exemples Pratiques

Les fonctions génératrices sont souvent utilisées pour générer des séquences infinies, des itérations paresseuses, ou pour simplifier des algorithmes complexes.

  • Exemple de Séquence Fibonacci :

    javascript
    function* fibonacci() {
    let [a, b] = [0, 1];
    while (true) {
        yield a;
        [a, b] = [b, a + b];
    }
}

let generateurFib = fibonacci();
console.log(generateurFib.next().value); // Affiche 0
console.log(generateurFib.next().value); // Affiche 1
console.log(generateurFib.next().value); // Affiche 1
console.log(generateurFib.next().value); // Affiche 2

Les fonctions génératrices offrent une manière élégante de gérer les séquences de valeurs dans JavaScript. 11.1 Utilisation de try, catch, finally

La gestion des erreurs en JavaScript se fait à l'aide des blocs try, catch, et éventuellement finally.

  • Exemple avec try et catch :

    javascript
    try {
    // Code susceptible de générer une erreur
    let resultat = 10 / 0; // Tentative de division par zéro
    console.log(resultat);
} catch (erreur) {
    // Bloc exécuté en cas d'erreur
    console.error("Erreur :", erreur.message);
} finally {
    // Bloc exécuté que l'erreur se produise ou non
    console.log("Bloc finally exécuté.");
}

12.1 Nommage des Fonctions

Un bon nommage des fonctions est essentiel pour rendre votre code lisible et compréhensible. Suivez les conventions de nommage pour améliorer la lisibilité.

  • Exemple de Nommage :

    javascript
    // Mauvais nommage
function x() {
    // Corps de la fonction
}

// Bon nommage
function calculerSomme(a, b) {
    // Corps de la fonction
    return a + b;
}

12.2 Éviter les Effets de Bord

Les effets de bord sont des modifications de l'état global du programme qui peuvent rendre le code difficile à comprendre et à maintenir. Privilégiez les fonctions pures qui n'ont pas d'effets de bord.

  • Exemple d'Effet de Bord :

    javascript
    let total = 0;

function ajouterAuTotal(nombre) {
    total += nombre;
}

ajouterAuTotal(5);
console.log(total); // Effet de bord : Affiche 5

  • Exemple de Fonction Pure :

    javascript
    function calculerSomme(a, b) {
    return a + b;
}

let resultat = calculerSomme(3, 4);
console.log(resultat); // Pas d'effet de bord : Affiche 7

12.3 DRY (Don't Repeat Yourself) avec les Fonctions

Évitez la duplication de code en regroupant les fonctionnalités répétées dans des fonctions. Cela favorise la maintenance et la réutilisation du code.

  • Exemple de DRY :

    javascript
    // Duplication de code
function calculerSurfaceRectangle(longueur, largeur) {
    return longueur * largeur;
}

function calculerPerimetreRectangle(longueur, largeur) {
    return 2 * (longueur + largeur);
}

// Version DRY
function calculerRectangle(longueur, largeur, type) {
    switch (type) {
        case 'surface':
            return longueur * largeur;
        case 'perimetre':
            return 2 * (longueur + largeur);
        default:
            return null;
    }
}
