1. Insertion de données dans la base de données

Avant de commencer, vous aurez besoin d'une base de données. Pour cet exemple, nous allons utiliser un exemple simple avec une base de données SQLite et un serveur Node.js avec Express.

1.1 Configuration de la base de données SQLite

javascript
const sqlite3 = require('sqlite3').verbose();
const db = new sqlite3.Database('ma_base_de_donnees.db');

// Création d'une table pour stocker les données
db.run('CREATE TABLE IF NOT EXISTS utilisateurs (id INTEGER PRIMARY KEY AUTOINCREMENT, nom TEXT, email TEXT)');

1.2 Insertion de données

javascript
// Route pour l'insertion de données
app.post('/utilisateurs', (req, res) => {
  const { nom, email } = req.body;

  // Insertion des données dans la base de données
  db.run('INSERT INTO utilisateurs (nom, email) VALUES (?, ?)', [nom, email], function(err) {
    if (err) {
      return res.status(500).json({ message: 'Erreur lors de l\'insertion des données', error: err.message });
    }

    // Renvoi d'informations après l'insertion réussie
    res.json({
      message: 'Données insérées avec succès',
      utilisateur: { id: this.lastID, nom, email }
    });
  });
});

const { body, validationResult } = require('express-validator');

1. Récupération de données depuis la base de données

Pour l'opération Read (lecture), nous allons créer une API pour récupérer les données depuis la base de données. Nous allons utiliser le même exemple avec SQLite et Express.

1.1 Récupération de tous les utilisateurs

javascript
// Route pour récupérer tous les utilisateurs
app.get('/utilisateurs', (req, res) => {
  db.all('SELECT * FROM utilisateurs', (err, rows) => {
    if (err) {
      return res.status(500).json({ message: 'Erreur lors de la récupération des utilisateurs', error: err.message });
    }

    res.json({ utilisateurs: rows });
  });
});

2. Utilisation de requêtes pour filtrer les résultats

2.1 Récupération d'un utilisateur spécifique par son ID

javascript
app.get('/utilisateurs/:id', (req, res) => {
  const userId = req.params.id;

  db.get('SELECT * FROM utilisateurs WHERE id = ?', [userId], (err, row) => {
    if (err) {
      return res.status(500).json({ message: 'Erreur lors de la récupération de l\'utilisateur', error: err.message });
    }

    if (!row) {
      return res.status(404).json({ message: 'Utilisateur non trouvé' });
    }

    res.json({ utilisateur: row });
  });
});

1. Modification de données dans la base de données

Pour l'opération Update (mise à jour), nous allons créer une API qui permet de modifier les données existantes dans la base de données. Nous allons continuer à utiliser SQLite et Express pour illustrer les exemples.

1.1 Modification d'un utilisateur par son ID

javascript
// Route pour la mise à jour d'un utilisateur par son ID
app.put('/utilisateurs/:id', (req, res) => {
  const userId = req.params.id;
  const { nom, email } = req.body;

  db.run('UPDATE utilisateurs SET nom = ?, email = ? WHERE id = ?', [nom, email, userId], function(err) {
    if (err) {
      return res.status(500).json({ message: 'Erreur lors de la mise à jour de l\'utilisateur', error: err.message });
    }

    if (this.changes === 0) {
      return res.status(404).json({ message: 'Utilisateur non trouvé' });
    }

    // Retour d'informations après la mise à jour réussie
    res.json({
      message: 'Utilisateur mis à jour avec succès',
      utilisateur: { id: userId, nom, email }
    });
  });
});

1. Suppression de données de la base de données

L'opération Delete (suppression) permet de retirer des données de la base de données. Nous allons continuer à utiliser SQLite et Express pour illustrer les exemples.

1.1 Suppression d'un utilisateur par son ID

javascript
// Route pour la suppression d'un utilisateur par son ID
app.delete('/utilisateurs/:id', (req, res) => {
  const userId = req.params.id;

  db.run('DELETE FROM utilisateurs WHERE id = ?', [userId], function(err) {
    if (err) {
      return res.status(500).json({ message: 'Erreur lors de la suppression de l\'utilisateur', error: err.message });
    }

    if (this.changes === 0) {
      return res.status(404).json({ message: 'Utilisateur non trouvé' });
    }

    // Retour d'informations après la suppression réussie
    res.json({ message: 'Utilisateur supprimé avec succès' });
  });
});

2. Confirmation de suppression

La confirmation de suppression est déjà incluse dans l'exemple de la section 1. Si la suppression réussit, un message confirmant la suppression est renvoyé dans la réponse.

3. Effets de cascade lors de la suppression

L'effet de cascade fait référence à la possibilité de supprimer automatiquement les enregistrements associés dans d'autres tables lors de la suppression d'un enregistrement parent. Cela dépend du modèle de données et des contraintes de clé étrangère dans votre base de données.

3.1 Exemple d'effet de cascade

javascript
// Création d'une table avec clé étrangère (exemple)
db.run('CREATE TABLE IF NOT EXISTS commandes (id INTEGER PRIMARY KEY AUTOINCREMENT, utilisateur_id INTEGER, FOREIGN KEY(utilisateur_id) REFERENCES utilisateurs(id) ON DELETE CASCADE)');

Dans cet exemple, la suppression d'un utilisateur entraînerait la suppression automatique de toutes les commandes associées grâce à l'effet de cascade.

4. Gestion des erreurs de suppression

Les erreurs de suppression sont traitées dans l'exemple de la section 1. Le statut de réponse est défini à 500 en cas d'erreur, et un message d'erreur est renvoyé dans le corps de la réponse. De plus, si aucune ligne n'a été supprimée (this.changes === 0), cela signifie que l'utilisateur n'a pas été trouvé, et une réponse 404 est renvoyée.
