1. Concept de course de promesses

La méthode Promise.race() est une fonction native JavaScript qui prend un tableau de promesses en argument et renvoie une nouvelle promesse qui est résolue ou rejetée en fonction de la première promesse du tableau à être résolue ou rejetée.

Le concept de "course de promesses" peut être compris comme une compétition entre plusieurs promesses pour voir laquelle sera la première à se résoudre ou à être rejetée. La première promesse à être résolue ou rejetée détermine le résultat de la promesse retournée par Promise.race().

2. Utilisation de Promise.race()

La syntaxe de base de Promise.race() est la suivante :

javascript
Promise.race(iterable);
  • iterable: Un tableau ou tout autre objet itérable contenant des promesses.

La promesse retournée par Promise.race() sera résolue avec la valeur ou rejetée avec la raison de la première promesse du tableau à être résolue ou rejetée.

3. Exemples pratiques

3.1. Utilisation de Promise.race() avec des délais

javascript
const promise1 = new Promise((resolve) => setTimeout(resolve, 1000, 'Première promesse'));
const promise2 = new Promise((resolve) => setTimeout(resolve, 500, 'Deuxième promesse'));

Promise.race([promise1, promise2])
  .then((result) => console.log(result))
  .catch((error) => console.error(error));

Dans cet exemple, la promesse retournée par Promise.race() sera résolue avec la valeur de la promesse2, car elle se résout plus rapidement que la promesse1.

3.2. Utilisation de Promise.race() avec des appels réseau

javascript
const fetchResource1 = fetch('https://api.example.com/resource1');
const fetchResource2 = fetch('https://api.example.com/resource2');

Promise.race([fetchResource1, fetchResource2])
  .then((response) => response.json())
  .then((data) => console.log(data))
  .catch((error) => console.error(error));

Dans cet exemple, la promesse retournée par Promise.race() sera résolue avec la première réponse réseau à revenir, que ce soit celle de fetchResource1 ou fetchResource2.

4. Gestion des promesses concurrentes

La gestion des promesses concurrentes avec Promise.race() peut être utile lorsque vous avez plusieurs tâches à effectuer et que vous voulez savoir laquelle se termine en premier. Cela peut être particulièrement utile dans les situations où la rapidité de réponse est cruciale.

javascript
const promise1 = performTask1();
const promise2 = performTask2();
const promise3 = performTask3();

Promise.race([promise1, promise2, promise3])
  .then((result) => console.log('La première tâche terminée :', result))
  .catch((error) => console.error('Une des tâches a échoué :', error));

Dans cet exemple, Promise.race() sera résolu avec le résultat de la première tâche à être complétée, ce qui peut être utile pour optimiser les performances dans certaines situations.

En résumé, Promise.race() est une fonction puissante pour gérer des promesses concurrentes et peut être utilisée dans divers scénarios pour améliorer l'efficacité et la réactivité de votre code.

1. Introduction à Promise.allSettled()

La méthode Promise.allSettled() est une fonction native JavaScript introduite dans ECMAScript 2020 (ES11). Cette méthode prend un tableau de promesses en argument et renvoie une nouvelle promesse qui est résolue avec un tableau de résultats une fois que toutes les promesses du tableau initial ont été résolues ou rejetées.

Contrairement à Promise.all(), qui est rejetée dès qu'une seule des promesses est rejetée, Promise.allSettled() attend que toutes les promesses soient résolues ou rejetées avant de renvoyer le résultat. Cela permet de traiter toutes les promesses, indépendamment de leur succès ou de leur échec.

2. Différence entre Promise.all() et Promise.allSettled()

La principale différence entre Promise.all() et Promise.allSettled() réside dans la façon dont ils traitent les rejets.

  • Promise.all(): Rejette immédiatement si l'une des promesses du tableau est rejetée.
  • Promise.allSettled(): Attend que toutes les promesses soient résolues ou rejetées et renvoie un tableau contenant le résultat de chaque promesse, qu'elle ait été résolue ou rejetée.

3. Traitement des résultats même en cas d'échec

Avec Promise.allSettled(), vous pouvez accéder aux résultats de toutes les promesses, qu'elles aient réussi ou échoué. Voici un exemple :

javascript
const promises = [Promise.resolve(42), Promise.reject("Échec"), Promise.resolve("Succès")];

Promise.allSettled(promises)
  .then((results) => {
    results.forEach((result) => {
      if (result.status === 'fulfilled') {
        console.log('Résultat réussi :', result.value);
      } else if (result.status === 'rejected') {
        console.log('Résultat en échec :', result.reason);
      }
    });
  });

Dans cet exemple, la promesse retournée par Promise.allSettled() sera résolue avec un tableau d'objets décrivant le statut de chaque promesse du tableau initial.

4. Utilisation de Promise.allSettled() avec des promesses mixtes

Promise.allSettled() est particulièrement utile lorsque vous avez un tableau de promesses mixtes, c'est-à-dire des promesses qui peuvent réussir ou échouer. Voici un exemple :

javascript
const promises = [Promise.resolve("Succès"), asyncFunction(), Promise.resolve(42)];

Promise.allSettled(promises)
  .then((results) => {
    results.forEach((result) => {
      if (result.status === 'fulfilled') {
        console.log('Résultat réussi :', result.value);
      } else if (result.status === 'rejected') {
        console.log('Résultat en échec :', result.reason);
      }
    });
  });

Dans cet exemple, même si asyncFunction() échoue, Promise.allSettled() attendra que toutes les promesses soient résolues ou rejetées avant de renvoyer le résultat.

5. Exemples illustratifs

5.1. Utilisation de Promise.allSettled() avec des appels réseau

javascript
const fetchResource1 = fetch('https://api.example.com/resource1');
const fetchResource2 = fetch('https://api.example.com/resource2');

Promise.allSettled([fetchResource1, fetchResource2])
  .then((results) => {
    results.forEach((result, index) => {
      if (result.status === 'fulfilled') {
        console.log(`Ressource ${index + 1} réussie :`, result.value);
      } else if (result.status === 'rejected') {
        console.log(`Ressource ${index + 1} en échec :`, result.reason);
      }
    });
  });

Dans cet exemple, Promise.allSettled() sera résolu avec les résultats de tous les appels réseau, qu'ils aient réussi ou échoué.

Promise.allSettled() est un outil puissant pour gérer des situations où vous souhaitez traiter toutes les promesses, même en cas d'échec. Il offre une approche plus robuste pour la gestion des promesses dans des scénarios divers.

1. Cas d'utilisation pratique

La combinaison de Promise.race() et Promise.allSettled() peut être puissante dans des scénarios où vous souhaitez obtenir le premier résultat (qu'il soit réussi ou en échec) parmi plusieurs tâches concurrentes, tout en collectant tous les résultats, qu'ils réussissent ou échouent.

1.1. Exemple de gestion de plusieurs requêtes réseau avec Promise.race() et Promise.allSettled()

javascript
const urls = ['https://api.example.com/resource1', 'https://api.example.com/resource2', 'https://api.example.com/resource3'];

// Création d'un tableau de promesses pour chaque requête réseau
const fetchPromises = urls.map(url => fetch(url));

// Utilisation de Promise.race() pour obtenir le premier résultat
const firstResultPromise = Promise.race(fetchPromises);

// Utilisation de Promise.allSettled() pour obtenir tous les résultats
const allResultsPromise = Promise.allSettled(fetchPromises);

// Utilisation d'une fonction pour traiter les résultats
function handleResults(firstResult, allResults) {
  console.log('Premier résultat (qu\'il réussisse ou échoue) :', firstResult);

  console.log('Tous les résultats (qu\'ils réussissent ou échouent) :');
  allResults.forEach((result, index) => {
    if (result.status === 'fulfilled') {
      console.log(`Ressource ${index + 1} réussie :`, result.value);
    } else if (result.status === 'rejected') {
      console.log(`Ressource ${index + 1} en échec :`, result.reason);
    }
  });
}

// Appel de la fonction lorsque les deux promesses sont résolues
Promise.all([firstResultPromise, allResultsPromise])
  .then(([firstResult, allResults]) => handleResults(firstResult, allResults))
  .catch(error => console.error('Une erreur s\'est produite :', error));

Dans cet exemple, Promise.race() est utilisé pour obtenir le premier résultat (qu'il réussisse ou échoue) parmi les requêtes réseau. Ensuite, Promise.allSettled() est utilisé pour obtenir tous les résultats (qu'ils réussissent ou échouent). La fonction handleResults est appelée pour traiter les résultats une fois que les deux promesses sont résolues.
