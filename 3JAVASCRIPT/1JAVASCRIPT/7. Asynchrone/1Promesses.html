A. Définition des Promesses

Les Promesses sont un concept clé dans le développement JavaScript moderne, offrant une approche asynchrone plus propre et plus lisible que les callbacks traditionnels. Une Promesse représente la réussite ou l'échec d'une opération asynchrone, et elle peut être dans l'un des trois états : pending (en attente), fulfilled (réalisée avec succès), ou rejected (rejetée).

  1. Création d'une Promesse : Pour créer une Promesse, utilisez le constructeur Promise. Il prend une fonction appelée exécuteur avec deux paramètres : resolve et reject.

    javascript
    const maPromesse = new Promise((resolve, reject) => {
    // Logique asynchrone ici
    if (condition) {
        resolve("Opération réussie !");
    } else {
        reject("Erreur : Opération échouée !");
    }
});

  2. Utilisation de .then() et .catch() : Vous pouvez attacher des gestionnaires de succès (.then()) et des gestionnaires d'échec (.catch()) à une Promesse.

    javascript
    maPromesse
    .then((resultat) => {
        console.log(resultat);
    })
    .catch((erreur) => {
        console.error(erreur);
    });

B. Importance des Promesses dans le JavaScript moderne

Les Promesses jouent un rôle crucial dans le développement moderne, en particulier dans le contexte du DOM JavaScript. Voici quelques raisons pour lesquelles elles sont importantes :

  1. Asynchronisme amélioré : Les Promesses simplifient la gestion des opérations asynchrones en évitant le piège des callbacks enchevêtrés, rendant le code plus lisible.

  2. Traitement des erreurs amélioré : La séparation claire entre le succès et l'échec facilite la gestion des erreurs, ce qui n'est pas toujours aussi évident avec les callbacks.

  3. Composition facilitée : Les Promesses peuvent être chaînées, ce qui facilite la composition d'opérations asynchrones séquentielles.

C. Avantages par rapport aux callbacks traditionnels

Les Promesses offrent plusieurs avantages par rapport aux callbacks traditionnels :

  1. Lisibilité améliorée : Le code utilisant des Promesses est souvent plus lisible et plus facile à comprendre en raison de la séparation claire entre les différents états de l'opération.

  2. Gestion des erreurs simplifiée : Avec les Promesses, les erreurs peuvent être gérées de manière centralisée avec .catch(), simplifiant ainsi la gestion des erreurs dans les opérations asynchrones.

  3. Évite le "Callback Hell" : Les Promesses permettent d'éviter le "Callback Hell", un phénomène où les callbacks imbriqués rendent le code difficile à lire et à maintenir.

En conclusion, l'utilisation des Promesses dans le JavaScript moderne, en particulier dans le contexte du DOM, améliore la lisibilité du code, facilite la gestion des opérations asynchrones et offre une manière plus propre de traiter les erreurs.

A. État des Promesses

Les Promesses peuvent se trouver dans l'un des trois états suivants :

  1. Pending (En attente) : L'état initial d'une Promesse. Cela signifie que l'opération asynchrone associée à la Promesse n'est pas encore terminée.

  2. Fulfilled (Réalisée avec succès) : L'état atteint lorsque l'opération asynchrone se termine avec succès. La Promesse renvoie une valeur qui peut être capturée par le gestionnaire de succès (then()).

  3. Rejected (Rejetée) : L'état atteint lorsque l'opération asynchrone échoue. La Promesse renvoie une raison de rejet qui peut être capturée par le gestionnaire d'échec (catch()).

B. Cycle de vie d'une Promesse

Le cycle de vie d'une Promesse comprend les étapes suivantes :

  1. Création : La Promesse est créée à l'aide du constructeur Promise, et son exécuteur est immédiatement appelé.

    javascript
    const maPromesse = new Promise((resolve, reject) => {
    // Logique asynchrone ici
});

  2. Pending (En attente) : La Promesse est initialement en état d'attente tant que l'opération asynchrone n'est pas terminée.

  3. Fulfillment (Réussite) : Si l'opération asynchrone réussit, la Promesse passe à l'état fulfilled avec la valeur spécifiée dans resolve().

    javascript
    resolve("Opération réussie !");

  4. Rejection (Rejet) : Si l'opération asynchrone échoue, la Promesse passe à l'état rejected avec la raison spécifiée dans reject().

    javascript
    reject("Erreur : Opération échouée !");

C. Chaînage de Promesses

Le chaînage de Promesses permet de séquencer des opérations asynchrones de manière lisible et ordonnée. Utilisez .then() pour gérer le succès et .catch() pour gérer les erreurs.

javascript
maPromesse
    .then((resultat) => {
        console.log(resultat);
        // Retourne une nouvelle Promesse ou une valeur
        return autreOperationAsynchrone();
    })
    .then((autreResultat) => {
        console.log(autreResultat);
    })
    .catch((erreur) => {
        console.error(erreur);
    });

D. Composition asynchrone avec les Promesses

La composition asynchrone avec les Promesses permet d'exécuter des opérations asynchrones en séquence. Utilisez Promise.all() pour exécuter plusieurs Promesses en parallèle et attendre qu'elles soient toutes terminées.

javascript
const promesse1 = operationAsynchrone1();
const promesse2 = operationAsynchrone2();

Promise.all([promesse1, promesse2])
    .then((resultats) => {
        // Les deux Promesses ont réussi
        console.log(resultats[0], resultats[1]);
    })
    .catch((erreur) => {
        // Au moins une des Promesses a échoué
        console.error(erreur);
    });

En comprenant les états des Promesses, leur cycle de vie, le chaînage de Promesses et la composition asynchrone, vous serez en mesure d'écrire un code JavaScript plus lisible et efficace lors de la manipulation du DOM.

A. Syntaxe de base

1. Utilisation du constructeur Promise

La création d'une Promesse commence par le constructeur Promise, qui prend une fonction appelée exécuteur en paramètre. Cette fonction a deux arguments, resolve et reject.

javascript
const maPromesse = new Promise((resolve, reject) => {
    // Logique asynchrone ici
});

2. La fonction resolve()

La fonction resolve est utilisée pour indiquer que l'opération asynchrone associée à la Promesse a réussi. Vous pouvez également passer une valeur qui sera renvoyée dans le gestionnaire de succès (then()).

javascript
resolve("Opération réussie !");

3. La fonction reject()

La fonction reject est utilisée pour indiquer que l'opération asynchrone associée à la Promesse a échoué. Vous pouvez également passer une raison de rejet qui sera renvoyée dans le gestionnaire d'échec (catch()).

javascript
reject("Erreur : Opération échouée !");

B. Exemples pratiques de création de Promesses

1. Promesse résolue

Créons une Promesse simple qui se résout après un délai de 1 seconde.

javascript
const promesseResolue = new Promise((resolve) => {
    setTimeout(() => {
        resolve("Promesse résolue après 1 seconde");
    }, 1000);
});

promesseResolue
    .then((resultat) => {
        console.log(resultat);
    })
    .catch((erreur) => {
        console.error(erreur);
    });

2. Promesse rejetée

Créons une Promesse qui est rejetée immédiatement.

javascript
const promesseRejetee = new Promise((resolve, reject) => {
    reject("Promesse rejetée immédiatement");
});

promesseRejetee
    .then((resultat) => {
        console.log(resultat);
    })
    .catch((erreur) => {
        console.error(erreur);
    });

3. Création de Promesses asynchrones

Créons une Promesse asynchrone qui effectue une requête AJAX (simulée ici avec setTimeout).

javascript
const promesseAsynchrone = new Promise((resolve, reject) => {
    setTimeout(() => {
        const succes = true; // Simulons le succès de la requête
        if (succes) {
            resolve("Données récupérées avec succès !");
        } else {
            reject("Erreur : Impossible de récupérer les données.");
        }
    }, 2000);
});

promesseAsynchrone
    .then((resultat) => {
        console.log(resultat);
    })
    .catch((erreur) => {
        console.error(erreur);
    });

En comprenant la syntaxe de base pour créer une Promesse, ainsi que les exemples pratiques de Promesses résolues, rejetées et asynchrones, vous serez prêt à utiliser ces concepts lors de la manipulation du DOM en JavaScript.

A. Utilisation de .then() pour traiter le succès

1. Chaînage de .then()

La méthode .then() est utilisée pour attacher des gestionnaires de succès à une Promesse. Elle prend une fonction de rappel qui sera exécutée lorsque la Promesse est résolue avec succès.

javascript
maPromesse
    .then((resultat) => {
        console.log(resultat);
    });

2. Gestion de la valeur résolue

Vous pouvez également récupérer la valeur résolue par la Promesse dans le gestionnaire .then().

javascript
maPromesse

B. Utilisation de .catch() pour gérer les erreurs

1. Gestion des erreurs avec .catch()

La méthode .catch() est utilisée pour attacher des gestionnaires d'erreur à une Promesse. Elle prend une fonction de rappel qui sera exécutée lorsque la Promesse est rejetée.

javascript
maPromesse
    .catch((erreur) => {
        console.error("Erreur :", erreur);
    });

2. Chaînage de .catch() après .then()

Vous pouvez chaîner la méthode .catch() après la méthode .then() pour gérer à la fois le succès et les erreurs.

javascript
maPromesse
    .then((resultat) => {

    })

javascript
maPromesse

A. Utilisation de .finally()

1. Exécution de code après la résolution ou le rejet

La méthode .finally() est utilisée pour attacher des gestionnaires qui seront exécutés après que la Promesse a été résolue ou rejetée, quel que soit le résultat.

javascript
maPromesse
    .then((resultat) => {
        console.log("Opération réussie avec la valeur :", resultat);
    })
    .catch((erreur) => {
        console.error("Erreur :", erreur);
    })
    .finally(() => {
        console.log("Code exécuté après résolution ou rejet de la Promesse");
    });

B. Utilisation de Promise.all()

1. Exécution simultanée de plusieurs Promesses

La méthode Promise.all() est utilisée pour exécuter simultanément plusieurs Promesses. Elle prend un tableau de Promesses en paramètre et renvoie une nouvelle Promesse qui est résolue lorsque toutes les Promesses du tableau sont résolues, ou rejetée si l'une d'entre elles est rejetée.

javascript
const promesse1 = operationAsynchrone1();
const promesse2 = operationAsynchrone2();

Promise.all([promesse1, promesse2])
    .then((resultats) => {
        console.log("Toutes les Promesses ont été résolues :", resultats);
    })
    .catch((erreur) => {
        console.error("Au moins une des Promesses a échoué :", erreur);
    });

2. Gestion des résultats de toutes les Promesses

Si vous avez besoin de traiter individuellement les résultats de chaque Promesse dans Promise.all(), utilisez la déstructuration.

javascript
const promesse1 = operationAsynchrone1();
const promesse2 = operationAsynchrone2();

Promise.all([promesse1, promesse2])
    .then(([resultat1, resultat2]) => {
        console.log("Résultat de la Promesse 1 :", resultat1);
        console.log("Résultat de la Promesse 2 :", resultat2);
    })
    .catch((erreur) => {
        console.error("Au moins une des Promesses a échoué :", erreur);
    });

En comprenant comment utiliser .finally() pour exécuter du code après la résolution ou le rejet d'une Promesse, ainsi que Promise.all() pour exécuter simultanément plusieurs Promesses et gérer leurs résultats, vous serez en mesure de gérer les erreurs de manière avancée lors de la manipulation du DOM en JavaScript.

A. Éviter les anti-patterns avec les Promesses

1. Éviter les Promesses non traitées

Assurez-vous de toujours attacher des gestionnaires .then() ou .catch() à vos Promesses pour éviter les Promesses non traitées, ce qui peut conduire à des erreurs difficiles à diagnostiquer.

javascript
maPromesse
    .then((resultat) => {
        console.log(resultat);
    })
    .catch((erreur) => {
        console.error(erreur);
    });

2. Éviter le piège du "Pyramid of Doom"

Utilisez le chaînage de Promesses plutôt que les callbacks enchevêtrés pour éviter le "Pyramid of Doom" (callbacks imbriqués excessifs), ce qui rend le code plus lisible.

javascript
promesse1
    .then((resultat1) => {
        return promesse2(resultat1);
    })
    .then((resultat2) => {
        return promesse3(resultat2);
    })
    .then((resultatFinal) => {
        console.log(resultatFinal);
    })
    .catch((erreur) => {
        console.error(erreur);
    });

B. Utilisation de Promises avec d'autres fonctionnalités JavaScript

1. Utilisation de Promises avec async/await

L'utilisation d'async/await simplifie la gestion des Promesses en rendant le code plus synchrone et lisible.

javascript
async function operationAsynchrone() {
    try {
        const resultat1 = await promesse1();
        const resultat2 = await promesse2(resultat1);
        const resultatFinal = await promesse3(resultat2);
        console.log(resultatFinal);
    } catch (erreur) {
        console.error(erreur);
    }
}

2. Utilisation de Promises avec Promise.race()

Promise.race() renvoie une nouvelle Promesse qui est résolue ou rejetée dès que l'une des Promesses dans un tableau est résolue ou rejetée.

javascript
const promesse1 = operationAsynchrone1();
const promesse2 = operationAsynchrone2();

Promise.race([promesse1, promesse2])
    .then((premiereResolue) => {
        console.log("La première Promesse résolue :", premiereResolue);
    })
    .catch((premiereRejetee) => {
        console.error("La première Promesse rejetée :", premiereRejetee);
    });

3. Utilisation de Promises avec Promise.allSettled()

Promise.allSettled() renvoie une nouvelle Promesse qui est résolue avec un tableau de résultats après que toutes les Promesses dans le tableau d'entrée ont été résolues ou rejetées.

javascript
const promesse1 = operationAsynchrone1();
const promesse2 = operationAsynchrone2();

Promise.allSettled([promesse1, promesse2])
    .then((resultats) => {
        resultats.forEach((resultat) => {
            if (resultat.status === "fulfilled") {
                console.log("Promesse réussie avec la valeur :", resultat.value);
            } else if (resultat.status === "rejected") {
                console.error("Promesse rejetée avec la raison :", resultat.reason);
            }
        });
    });

En suivant ces bonnes pratiques et astuces, vous pourrez écrire un code plus propre, plus lisible et plus robuste lors de l'utilisation des Promesses dans la manipulation du DOM en JavaScript.
