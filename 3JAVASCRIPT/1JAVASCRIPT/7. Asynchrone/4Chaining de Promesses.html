1. Définition des Promesses

1.1 Qu'est-ce qu'une Promesse?

Une promesse en JavaScript est un objet représentant l'achèvement ou l'échec d'une opération asynchrone. Elle permet de traiter des opérations asynchrones de manière plus propre et plus lisible.

1.2 Structure d'une Promesse

Une promesse a généralement trois états:

  • Pendante (pending): L'état initial, la promesse est en attente d'être résolue ou rejetée.
  • Résolue (fulfilled): L'opération asynchrone est réussie, et la promesse renvoie une valeur.
  • Rejetée (rejected): L'opération asynchrone a échoué, et la promesse renvoie une raison du rejet.

1.3 Création d'une Promesse

javascript
const maPromesse = new Promise((resolve, reject) => {
  // Code asynchrone ici

  // Succès
  resolve('Résultat réussi');

  // Erreur
  reject('Erreur rencontrée');
});

1.4 Utilisation de then et catch

javascript
maPromesse
  .then((resultat) => {
    console.log('Succès:', resultat);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

2. Importance du Chaining de Promesses

2.1 Qu'est-ce que le Chaining de Promesses?

Le Chaining de Promesses permet d'enchaîner plusieurs opérations asynchrones de manière séquentielle et lisible. Cela évite les problèmes liés aux callbacks en cascade (callback hell).

2.2 Exemple de Chaining de Promesses

javascript
const premierePromesse = () => {
  return new Promise((resolve, reject) => {
    resolve('Première étape réussie');
  });
};

const deuxiemePromesse = (message) => {
  return new Promise((resolve, reject) => {
    resolve(`${message}, Deuxième étape réussie`);
  });
};

premierePromesse()
  .then((resultatPremierePromesse) => {
    return deuxiemePromesse(resultatPremierePromesse);
  })
  .then((resultatFinal) => {
    console.log(resultatFinal);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

En utilisant le Chaining de Promesses, chaque .then renvoie une nouvelle promesse, permettant de chaîner plusieurs opérations asynchrones de manière claire et ordonnée.

C'est une introduction aux Promesses et au Chaining de Promesses en JavaScript. Dans les sections suivantes, nous explorerons davantage les concepts avancés et les bonnes pratiques.

2. Basics du Chaining de Promesses

2.1 Création d'une Promesse

La création d'une promesse en JavaScript se fait à l'aide du constructeur Promise. Ce constructeur prend une fonction callback avec deux paramètres, resolve et reject. resolve est appelé lorsque l'opération asynchrone réussit, et reject est appelé en cas d'échec.

javascript
const maPromesse = new Promise((resolve, reject) => {
  // Code asynchrone ici

  // Succès
  resolve('Résultat réussi');

  // Erreur
  reject('Erreur rencontrée');
});

2.2 Résolution et Rejet de Promesses

Une promesse peut être résolue avec succès en appelant la fonction resolve et en passant une valeur en argument. Elle peut également être rejetée en appelant la fonction reject avec une raison du rejet.

javascript
const promesseReussie = new Promise((resolve, reject) => {
  resolve('Opération réussie');
});

const promesseEchouee = new Promise((resolve, reject) => {
  reject('Opération échouée');
});

2.3 Syntaxe du Chaining de Promesses

Le Chaining de Promesses permet d'enchaîner plusieurs opérations asynchrones de manière séquentielle et lisible. La méthode then est utilisée pour attacher une fonction de rappel qui sera exécutée lorsque la promesse est résolue avec succès.

javascript
maPromesse
  .then((resultat) => {
    console.log('Succès:', resultat);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

L'utilisation de then crée une nouvelle promesse, ce qui permet de chaîner plusieurs opérations asynchrones. Voici un exemple de Chaining de Promesses:

javascript
const premierePromesse = () => {
  return new Promise((resolve, reject) => {
    resolve('Première étape réussie');
  });
};

const deuxiemePromesse = (message) => {
  return new Promise((resolve, reject) => {
    resolve(`${message}, Deuxième étape réussie`);
  });
};

premierePromesse()
  .then((resultatPremierePromesse) => {
    return deuxiemePromesse(resultatPremierePromesse);
  })
  .then((resultatFinal) => {
    console.log(resultatFinal);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

Chaque .then renvoie une nouvelle promesse, permettant de créer une séquence ordonnée d'opérations asynchrones. Le catch est utilisé pour gérer les erreurs dans n'importe quelle partie de la chaîne.

3. Retourner une Promesse dans une Fonction

3.1 Utilisation de return

Souvent, dans les fonctions asynchrones, vous voudrez retourner une promesse pour permettre au code appelant d'effectuer des opérations supplémentaires une fois que l'opération asynchrone est terminée. Voici comment vous pouvez utiliser return pour retourner une promesse dans une fonction :

javascript
function fonctionAvecPromesse() {
  return new Promise((resolve, reject) => {
    // Opération asynchrone ici

    if (/* opération réussie */) {
      resolve('Succès');
    } else {
      reject('Échec');
    }
  });
}

// Utilisation de la fonction avec le Chaining de Promesses
fonctionAvecPromesse()
  .then((resultat) => {
    console.log('Résultat:', resultat);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

3.2 Manipulation de la Promesse Résultante

Vous pouvez également manipuler la promesse résultante à l'intérieur de la fonction en utilisant d'autres méthodes de résolution, telles que Promise.resolve ou Promise.reject. Cela peut être utile dans divers scénarios.

javascript
function fonctionAvecPromesseManipulee() {
  // Opération asynchrone ici

  if (/* opération réussie */) {
    return Promise.resolve('Succès');
  } else {
    return Promise.reject('Échec');
  }
}

// Utilisation de la fonction avec le Chaining de Promesses
fonctionAvecPromesseManipulee()
  .then((resultat) => {
    console.log('Résultat:', resultat);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

En utilisant Promise.resolve, vous pouvez retourner une promesse résolue avec une valeur, et avec Promise.reject, vous pouvez retourner une promesse rejetée avec une raison.

Cela vous permet de contrôler finement le flux de votre application, en fournissant des résultats ou des erreurs à chaque étape du processus asynchrone. Vous pouvez ensuite utiliser le Chaining de Promesses pour créer des séquences plus complexes et lisibles.

5. Chaining Avancé

5.1 Utilisation de .then() Multiple

Dans le Chaining de Promesses, il est possible d'utiliser plusieurs méthodes .then() pour gérer différents résultats successifs. Chaque .then() peut effectuer une opération spécifique en fonction du résultat de la promesse précédente.

javascript
function premierePromesse() {
  return new Promise((resolve, reject) => {
    resolve('Premier résultat');
  });
}

premierePromesse()
  .then((resultat) => {
    console.log(resultat); // Premier résultat
    return 'Deuxième résultat';
  })
  .then((resultat) => {
    console.log(resultat); // Deuxième résultat
    return 'Troisième résultat';
  })
  .then((resultat) => {
    console.log(resultat); // Troisième résultat
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

Chaque .then() reçoit le résultat de la promesse précédente et retourne une nouvelle valeur, qui peut être utilisée dans la prochaine étape du Chaining.

5.2 Enchaînement de Promesses en Série

En utilisant des fonctions qui retournent des promesses, vous pouvez créer une séquence d'opérations asynchrones en série. Chaque étape dépend du résultat de la précédente.

javascript
function premiereEtape() {
  return new Promise((resolve) => {
    resolve('Première étape réussie');
  });
}

function deuxiemeEtape(message) {
  return new Promise((resolve) => {
    resolve(`${message}, Deuxième étape réussie`);
  });
}

function troisiemeEtape(message) {
  return new Promise((resolve) => {
    resolve(`${message}, Troisième étape réussie`);
  });
}

premiereEtape()
  .then((resultat) => deuxiemeEtape(resultat))
  .then((resultat) => troisiemeEtape(resultat))
  .then((resultatFinal) => {
    console.log(resultatFinal);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

Chaque étape renvoie une nouvelle promesse, et le Chaining assure l'exécution en séquence. Cela rend le code plus lisible et plus facile à maintenir.

5.3 Utilisation de .finally()

La méthode .finally() est utilisée pour spécifier un bloc de code à exécuter, que la promesse soit résolue ou rejetée. Cela peut être utile pour effectuer des opérations de nettoyage ou des actions communes, quel que soit le résultat.

javascript
function promesseAvecFinally() {
  return new Promise((resolve, reject) => {
    // Opération asynchrone ici
    resolve('Succès');
    // Ou reject('Échec');
  });
}

promesseAvecFinally()
  .then((resultat) => {
    console.log('Résultat:', resultat);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  })
  .finally(() => {
    console.log('Code exécuté peu importe le résultat de la promesse');
  });

Le bloc .finally() sera toujours exécuté, que la promesse soit résolue avec succès, rejetée ou si une exception se produit pendant l'exécution du .then() ou du .catch().

6. Utilisation de Promise.all()

6.1 Introduction à Promise.all()

Promise.all() est une fonction native de JavaScript qui prend un tableau de promesses en entrée et retourne une nouvelle promesse qui est résolue avec un tableau de résultats une fois que toutes les promesses du tableau d'entrée ont été résolues. Si l'une des promesses est rejetée, la promesse résultante est rejetée avec la raison du premier rejet.

6.2 Syntaxe de Promise.all()

La syntaxe de base de Promise.all() est la suivante :

javascript
Promise.all([promesse1, promesse2, ...])
  .then((resultats) => {
    // Toutes les promesses sont résolues avec succès
    console.log('Résultats:', resultats);
  })
  .catch((erreur) => {
    // Une des promesses a été rejetée
    console.error('Erreur:', erreur);
  });

6.3 Exécution Simultanée de Promesses

Promise.all() est particulièrement utile lorsque vous avez plusieurs opérations asynchrones indépendantes que vous souhaitez exécuter simultanément. Cela peut améliorer les performances en comparaison avec l'exécution séquentielle des promesses.

javascript
const promesse1 = () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve('Résultat de la première promesse');
    }, 1000);
  });
};

const promesse2 = () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve('Résultat de la deuxième promesse');
    }, 500);
  });
};

Promise.all([promesse1(), promesse2()])
  .then((resultats) => {
    console.log('Résultats:', resultats);
    // Output après environ 1000ms (temps de la promesse la plus lente)
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

Les deux promesses (promesse1 et promesse2) sont exécutées simultanément, et le .then() ne sera appelé qu'une fois que toutes les promesses seront résolues.

6.4 Gestion des Résultats de Promesses Multiples

Les résultats des promesses multiples sont renvoyés sous forme de tableau dans l'ordre d'origine du tableau d'entrée. Vous pouvez ensuite utiliser ces résultats comme bon vous semble.

javascript
const promesse1 = () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve('Résultat de la première promesse');
    }, 1000);
  });
};

const promesse2 = () => {
  return new Promise((resolve) => {
    setTimeout(() => {
      resolve('Résultat de la deuxième promesse');
    }, 500);
  });
};

Promise.all([promesse1(), promesse2()])
  .then((resultats) => {
    const [resultatPromesse1, resultatPromesse2] = resultats;
    console.log('Résultat de la première promesse:', resultatPromesse1);
    console.log('Résultat de la deuxième promesse:', resultatPromesse2);
  })
  .catch((erreur) => {
    console.error('Erreur:', erreur);
  });

Utiliser la déstructuration pour extraire les résultats individuels rend le code plus lisible et facilite le traitement ultérieur des données.
