A. Introduction aux fonctions asynchrones

1. Promesses (Promises)

Les promesses sont un moyen d'effectuer des opérations asynchrones en JavaScript. Elles peuvent être dans l'état "pending", "resolved" (accomplie) ou "rejected" (rejetée).

javascript
const myPromise = new Promise((resolve, reject) => {
  // Code asynchrone
  if (/* opération réussie */) {
    resolve("La promesse a été résolue avec succès !");
  } else {
    reject("La promesse a échoué !");
  }
});

myPromise
  .then((successMessage) => {
    console.log(successMessage);
  })
  .catch((errorMessage) => {
    console.error(errorMessage);
  });

2. Async/Await

L'opérateur async est utilisé pour déclarer une fonction asynchrone. L'opérateur await est utilisé à l'intérieur d'une fonction asynchrone pour attendre qu'une promesse soit résolue ou rejetée.

javascript
async function myAsyncFunction() {
  try {
    const result = await myPromise;
    console.log(result);
  } catch (error) {
    console.error(error);
  }
}

myAsyncFunction();

B. Utilisation de try...catch avec les fonctions asynchrones

L'utilisation de try...catch avec les fonctions asynchrones permet de gérer les erreurs de manière similaire à la gestion des erreurs dans les fonctions synchrones.

javascript
async function fetchData() {
  try {
    const response = await fetch('https://example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Une erreur s\'est produite :', error);
  }
}

fetchData();

C. Gestion des erreurs dans les chaînes de promesses (Promise chaining)

Il est important de gérer les erreurs lors de la création de chaînes de promesses pour éviter de manquer des erreurs dans le flux asynchrone.

javascript
function fetchUserData(userId) {
  return fetch(`https://api.example.com/users/${userId}`)
    .then((response) => {
      if (!response.ok) {
        throw new Error('Échec de la requête');
      }
      return response.json();
    })
    .then((userData) => {
      console.log(userData);
    })
    .catch((error) => {
      console.error('Une erreur s\'est produite :', error);
    });
}

fetchUserData(123);

D. Utilisation de Promise.reject() pour générer des erreurs dans les fonctions asynchrones

Vous pouvez utiliser Promise.reject() pour générer délibérément une promesse rejetée à l'intérieur d'une fonction asynchrone.

javascript
async function simulateError() {
  return new Promise((resolve, reject) => {
    reject(new Error('Erreur générée délibérément.'));
  });
}

async function myAsyncFunction() {
  try {
    await simulateError();
  } catch (error) {
    console.error('Une erreur s\'est produite :', error);
  }
}

myAsyncFunction();

A. Introduction à Promise.all()

La méthode Promise.all() est une fonction pratique en JavaScript qui prend un tableau de promesses et retourne une nouvelle promesse qui est résolue avec un tableau de résultats une fois que toutes les promesses du tableau d'entrée sont résolues, ou rejetée dès qu'une des promesses du tableau d'entrée est rejetée.

B. Utilisation basique de Promise.all()

Voici comment utiliser Promise.all() de manière basique :

javascript
const promise1 = fetch('https://api.example.com/data1');
const promise2 = fetch('https://api.example.com/data2');

Promise.all([promise1, promise2])
  .then((responses) => {
    // Traiter les réponses ici
    console.log(responses[0].json());
    console.log(responses[1].json());
  })
  .catch((error) => {
    console.error('Au moins une promesse a été rejetée :', error);
  });

C. Gestion des erreurs dans Promise.all()

1. Le rejet immédiat si une promesse est rejetée

Promise.all() est conçu pour rejeter dès qu'une des promesses du tableau est rejetée. Cela peut être utile si vous souhaitez que toutes les promesses réussissent pour considérer l'opération comme réussie.

javascript
const promise1 = fetch('https://api.example.com/data1');
const promise2 = Promise.reject(new Error('Une erreur s\'est produite'));

Promise.all([promise1, promise2])
  .then((responses) => {
    // Cette partie ne sera pas atteinte car une promesse a été rejetée
  })
  .catch((error) => {
    console.error('Au moins une promesse a été rejetée :', error);
  });

2. Collecte des résultats réussis malgré les erreurs

Si vous souhaitez collecter les résultats réussis malgré les erreurs, vous pouvez combiner Promise.all() avec Promise.allSettled().

javascript
const promise1 = fetch('https://api.example.com/data1');
const promise2 = Promise.reject(new Error('Une erreur s\'est produite'));

Promise.allSettled([promise1, promise2])
  .then((results) => {
    const successfulResults = results
      .filter((result) => result.status === 'fulfilled')
      .map((result) => result.value);

    console.log('Résultats réussis :', successfulResults);
  })
  .catch((error) => {
    console.error('Une erreur s\'est produite :', error);
  });

D. Exemples pratiques de Promise.all() avec gestion des erreurs

const loadResources = async () => {

    console.error('Une erreur s\'est produite lors du chargement des ressources :', error);

loadResources();

  } catch (error) {
    console.error('Au moins un formulaire est invalide :', error);
  }

validateForms();

A. Utilisation de Promise.race() pour gérer la première promesse résolue ou rejetée

La méthode Promise.race() prend un tableau de promesses et retourne une nouvelle promesse qui est résolue ou rejetée en fonction de la première promesse du tableau à être résolue ou rejetée.

javascript
const promise1 = new Promise((resolve) => setTimeout(() => resolve('Résolu après 1000ms'), 1000));
const promise2 = new Promise((_, reject) => setTimeout(() => reject(new Error('Rejeté après 500ms')), 500));

Promise.race([promise1, promise2])
  .then((result) => {
    console.log('La première promesse résolue est :', result);
  })
  .catch((error) => {
    console.error('La première promesse rejetée est :', error);
  });

B. Utilisation de try...catch avec async/await et Promise.all()

Combinez try...catch avec async/await pour gérer les erreurs de manière concise et lisible, même lors de l'utilisation de Promise.all().

javascript
async function fetchData() {
  const promises = [fetch('https://api.example.com/data1'), fetch('https://api.example.com/data2')];

  try {
    const responses = await Promise.all(promises);
    const data = await Promise.all(responses.map(response => response.json()));
    console.log('Données récupérées avec succès :', data);
  } catch (error) {
    console.error('Une erreur s\'est produite lors du chargement des données :', error);
  }
}

fetchData();

A. Consignation des erreurs

La consignation des erreurs est essentielle pour le débogage et l'amélioration continue de votre application. Utilisez les fonctions de consignation appropriées, telles que console.error() ou des bibliothèques de consignation tierces.

javascript
try {
  // Code susceptible de générer une erreur
} catch (error) {
  console.error('Une erreur s\'est produite :', error);
}

Si vous utilisez un environnement Node.js, envisagez d'utiliser des bibliothèques de consignation plus avancées telles que Winston ou Bunyan.

B. Gestion des erreurs côté client et côté serveur

Côté client (Navigateur)

  1. Utilisation de try...catch dans le code asynchrone : Encadrez le code asynchrone avec try...catch pour capturer les erreurs asynchrones.
javascript
async function fetchData() {
  try {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error('Une erreur s\'est produite lors du chargement des données :', error);
  }
}
  1. Gestion des erreurs globales : Utilisez l'événement window.onerror pour capturer les erreurs globales et les consigner.
javascript
window.onerror = function(message, source, lineno, colno, error) {
  console.error('Erreur globale détectée :', message, source, lineno, colno, error);
};

Côté serveur (Node.js)

  1. Middleware pour la gestion des erreurs : Utilisez des middlewares tels que Express.js error handling middleware pour gérer les erreurs côté serveur de manière centralisée.
javascript
app.use((err, req, res, next) => {
  console.error('Erreur détectée côté serveur :', err);
  res.status(500).send('Erreur interne du serveur.');
});
  1. Utilisation de process.on('uncaughtException') : Capturez les exceptions non gérées au niveau du processus.
javascript
process.on('uncaughtException', (err) => {
  console.error('Exception non gérée détectée :', err);
  process.exit(1);
});
