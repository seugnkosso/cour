2.1. Syntaxe de base

2.1.1. Constructeur de Promesse

Une promesse est créée à l'aide du constructeur Promise. Le constructeur prend une fonction exécutoire avec deux paramètres : resolve et reject.

javascript
const maPromesse = new Promise((resolve, reject) => {
  // Code asynchrone à l'intérieur de la promesse
});

2.1.2. Paramètre de la fonction exécutoire

La fonction exécutoire est le cœur de la promesse. Elle contient le code asynchrone que la promesse encapsule. Cette fonction prend deux paramètres : resolve et reject.

  • resolve: Utilisé pour indiquer que la promesse a réussi et renvoyer une valeur.
  • reject: Utilisé pour indiquer que la promesse a échoué et renvoyer un motif d'échec.

2.2. Résolution et Rejet de Promesses

2.2.1. Résoudre une Promesse avec resolve()

Pour marquer la promesse comme résolue, utilisez la fonction resolve() en passant une valeur en paramètre.

javascript
const maPromesse = new Promise((resolve, reject) => {
  // Code asynchrone réussi
  resolve("Promesse résolue avec succès !");
});

2.2.2. Rejeter une Promesse avec reject()

Pour marquer la promesse comme rejetée, utilisez la fonction reject() en passant un motif d'échec en paramètre.

javascript
const maPromesse = new Promise((resolve, reject) => {
  // Code asynchrone échoué
  reject("La promesse a échoué avec une raison spécifique.");
});

Exemple complet

javascript
const maPromesse = new Promise((resolve, reject) => {
  const reussite = true;

  if (reussite) {
    resolve("Promesse résolue avec succès !");
  } else {
    reject("La promesse a échoué avec une raison spécifique.");
  }
});

// Utilisation de la promesse
maPromesse
  .then((resultat) => {
    console.log("Succès :", resultat);
  })
  .catch((raison) => {
    console.error("Échec :", raison);
  });

Dans cet exemple, la promesse est créée avec un code asynchrone simulé, et ensuite, la méthode .then() est utilisée pour gérer le succès, et la méthode .catch() est utilisée pour gérer l'échec.

C'est ainsi que vous créez, résolvez et rejetez des promesses en JavaScript !

3.1. Création de Fonctions Asynchrones

3.1.1. Mots-clés async et await

JavaScript propose les mots-clés async et await pour faciliter la gestion des fonctions asynchrones.

  • async: Déclare qu'une fonction est asynchrone et retournera une Promesse.
  • await: Attend la résolution d'une Promesse à l'intérieur d'une fonction asynchrone.
javascript
async function fonctionAsynchrone() {
  // Code asynchrone ici
  const resultat = await promesseAsynchrone();
  // Suite du code après la résolution de la promesse
  return resultat;
}

3.1.2. Fonctions Retournant des Promesses

Les fonctions asynchrones retournent généralement des Promesses. Vous pouvez utiliser le mot-clé return avec une valeur ou utiliser return Promise.resolve(valeur) pour retourner une Promesse résolue.

javascript
async function fonctionAvecPromesse() {
  // Code asynchrone ici
  return "Résultat de la promesse";
}

3.2. Exemples Pratiques

3.2.1. Fonction de Téléchargement Asynchrone

Imaginons une fonction qui télécharge une ressource depuis un serveur de manière asynchrone.

javascript
async function telechargerRessource(url) {
  return new Promise((resolve, reject) => {
    // Simulation d'une requête HTTP
    setTimeout(() => {
      const reussite = true;
      if (reussite) {
        resolve(`Ressource téléchargée depuis ${url}`);
      } else {
        reject("Échec du téléchargement de la ressource");
      }
    }, 2000);
  });
}

// Utilisation de la fonction
async function utiliserTelechargement() {
  try {
    const resultat = await telechargerRessource("https://example.com/ressource");
    console.log(resultat);
  } catch (erreur) {
    console.error(erreur);
  }
}

utiliserTelechargement();

3.2.2. Fonction de Lecture de Fichier Asynchrone

Créons une fonction qui lit un fichier de manière asynchrone.

javascript
const fs = require('fs').promises; // Module pour la gestion de fichiers

async function lireFichier(chemin) {
  try {
    const contenu = await fs.readFile(chemin, 'utf-8');
    return contenu;
  } catch (erreur) {
    throw new Error(`Erreur de lecture du fichier : ${erreur.message}`);
  }
}

// Utilisation de la fonction
async function utiliserLectureFichier() {
  try {
    const contenu = await lireFichier('chemin/vers/mon-fichier.txt');
    console.log("Contenu du fichier :", contenu);
  } catch (erreur) {
    console.error(erreur.message);
  }
}

utiliserLectureFichier();

5.1. then(), catch(), et finally()

Les méthodes then(), catch(), et finally() sont utilisées pour le chaînage des Promesses.

5.1.1. Exécution de Code après la Résolution

La méthode then() est utilisée pour exécuter du code après la résolution d'une Promesse.

javascript
const promesse = new Promise((resolve, reject) => {
  // Code asynchrone réussi
  resolve("Résolu avec succès !");
});

promesse.then((resultat) => {
  console.log("Résolu avec :", resultat);
});

5.1.2. Gestion des Erreurs avec catch()

La méthode catch() est utilisée pour gérer les erreurs dans une Promesse.

javascript
const promesse = new Promise((resolve, reject) => {
  // Code asynchrone échoué
  reject("Échec avec une raison spécifique.");
});

promesse.catch((raison) => {
  console.error("Échec avec raison :", raison);
});

5.1.3. Exécution de Code après la Résolution ou le Rejet

La méthode finally() est utilisée pour exécuter du code après la résolution ou le rejet, quel que soit le résultat.

javascript
const promesse = new Promise((resolve, reject) => {
  // Code asynchrone réussi ou échoué
  resolve("Résolu avec succès !");
});

promesse
  .then((resultat) => {
    console.log("Résolu avec :", resultat);
  })
  .catch((raison) => {
    console.error("Échec avec raison :", raison);
  })
  .finally(() => {
    console.log("Code exécuté après la résolution ou le rejet.");
  });

Exemple Complet

javascript
function effectuerTache() {
  return new Promise((resolve, reject) => {
    const reussite = true;

    if (reussite) {
      resolve("Tâche effectuée avec succès !");
    } else {
      reject("Échec dans l'accomplissement de la tâche.");
    }
  });
}

effectuerTache()
  .then((resultat) => {
    console.log(resultat);
    return "Résultat pour la prochaine tâche.";
  })
  .then((resultatSuivant) => {
    console.log(resultatSuivant);
    throw new Error("Une erreur s'est produite.");
  })
  .catch((erreur) => {
    console.error(erreur.message);
  })
  .finally(() => {
    console.log("Code exécuté après la résolution ou le rejet.");
  });

Dans cet exemple, la fonction effectuerTache() retourne une Promesse. En utilisant les méthodes then(), catch(), et finally(), vous pouvez chaîner des actions pour gérer le succès, les erreurs, et exécuter du code après la résolution ou le rejet. Ce mécanisme de chaînage est essentiel pour créer des séquences de tâches asynchrones bien organisées dans JavaScript.

6.1. Promise.all()

6.1.1. Utilisation avec un Tableau de Promesses

La méthode Promise.all() est utilisée pour attendre la résolution de toutes les Promesses d'un tableau.

javascript
const promesse1 = Promise.resolve("Résultat 1");
const promesse2 = Promise.resolve("Résultat 2");
const promesse3 = Promise.resolve("Résultat 3");

const tableauDePromesses = [promesse1, promesse2, promesse3];

Promise.all(tableauDePromesses)
  .then((resultats) => {
    console.log("Résultats :", resultats);
  })
  .catch((erreur) => {
    console.error("Erreur :", erreur);
  });

6.1.2. Attente de la Résolution de Toutes les Promesses

Promise.all() attend la résolution de toutes les Promesses du tableau. Si l'une d'entre elles est rejetée, l'ensemble de la Promesse sera rejeté.

6.2. Promise.race()

6.2.1. Résolution dès qu'une Promesse est Résolue

La méthode Promise.race() est utilisée pour attendre la résolution de la première Promesse du tableau.

javascript
const promesse1 = new Promise((resolve) => setTimeout(() => resolve("Résultat 1"), 2000));
const promesse2 = new Promise((resolve) => setTimeout(() => resolve("Résultat 2"), 1000));
const promesse3 = new Promise((resolve) => setTimeout(() => resolve("Résultat 3"), 3000));

const tableauDePromesses = [promesse1, promesse2, promesse3];

Promise.race(tableauDePromesses)
  .then((resultat) => {
    console.log("Premier résultat résolu :", resultat);
  })
  .catch((erreur) => {
    console.error("Erreur :", erreur);
  });

6.2.2. Utilisation avec un Tableau de Promesses

Promise.race() attend la résolution de la première Promesse du tableau. Si la première Promesse est rejetée, l'ensemble de la Promesse sera rejeté.

Exemple Complet
