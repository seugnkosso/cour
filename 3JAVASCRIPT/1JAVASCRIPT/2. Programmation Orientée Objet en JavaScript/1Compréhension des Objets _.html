1.1 Définition des objets

En JavaScript, les objets sont des entités qui regroupent des propriétés et des méthodes. Les propriétés sont des variables associées à l'objet, tandis que les méthodes sont des fonctions associées à l'objet. Les objets peuvent être utilisés pour représenter des entités du monde réel ou des structures de données complexes.

Exemple de définition d'un objet :

javascript
// Définition d'un objet représentant une personne
let personne = {
    nom: "John",
    age: 30,
    adresse: {
        rue: "123 Rue Principale",
        ville: "Villeville",
        codePostal: "12345"
    },
    parler: function() {
        console.log("Bonjour, je m'appelle " + this.nom + " et j'ai " + this.age + " ans.");
    }
};

1.2 Syntaxe de création d'objets

Il existe plusieurs façons de créer des objets en JavaScript. Une des méthodes les plus courantes est l'utilisation de la notation littérale d'objet.

Notation littérale d'objet :

javascript
let monObjet = {}; // Crée un objet vide

// Ajout de propriétés à l'objet
monObjet.nom = "Objet1";
monObjet.valeur = 42;

// Ou création d'un objet avec des propriétés initiales
let autreObjet = { propriete1: "valeur1", propriete2: "valeur2" };

1.3 Accéder aux propriétés et méthodes

On peut accéder aux propriétés et aux méthodes d'un objet en utilisant la notation point (.) ou la notation crochet ([]).

Accéder aux propriétés :

javascript
console.log(personne.nom); // Affiche "John"
console.log(personne.adresse.ville); // Affiche "Villeville"

Accéder aux méthodes :

javascript
personne.parler(); // Affiche "Bonjour, je m'appelle John et j'ai 30 ans."

Dans l'exemple ci-dessus, this fait référence à l'objet lui-même, permettant d'accéder à ses propriétés à l'intérieur de ses méthodes.

Ce tutoriel constitue une introduction de base aux objets en JavaScript. Il existe de nombreux concepts avancés liés aux objets, tels que l'héritage, la programmation orientée objet, etc., qui peuvent être explorés à mesure que vous approfondissez vos connaissances en JavaScript.

2.1 Objets primitifs vs Objets complexes

En JavaScript, il existe deux types d'objets : les objets primitifs et les objets complexes.

Objets primitifs :

Les types primitifs (number, string, boolean, null, undefined) sont des valeurs simples stockées directement dans la variable. Ils sont immuables, ce qui signifie que toute modification crée une nouvelle valeur.

Exemple :

javascript
let x = 5; // Number (primitif)
let texte = "Hello"; // String (primitif)
let estVrai = true; // Boolean (primitif)

Objets complexes :

Les objets complexes (arrays, objects, functions) sont des références stockées dans la variable, pointant vers l'emplacement en mémoire où l'objet est stocké. La modification de l'objet modifie l'objet d'origine.

Exemple :

javascript
let tableau = [1, 2, 3]; // Array (objet complexe)
let personne = { nom: "John", age: 30 }; // Object (objet complexe)

2.2 Objets natifs (String, Number, Array, etc.)

JavaScript fournit des objets natifs pour traiter différentes valeurs et effectuer diverses opérations.

String :

Les objets String permettent de manipuler des chaînes de caractères.

Exemple :

javascript
let message = "Hello";
console.log(message.length); // Affiche la longueur de la chaîne
console.log(message.toUpperCase()); // Affiche la chaîne en majuscules

Number :

Les objets Number facilitent la manipulation de nombres.

Exemple :

javascript
let nombre = 42.5;
console.log(nombre.toFixed(1)); // Affiche le nombre avec une décimale

Array :

Les objets Array facilitent le stockage et la manipulation de collections d'éléments.

Exemple :

javascript
let fruits = ["pomme", "orange", "banane"];
console.log(fruits.length); // Affiche la longueur du tableau
fruits.push("fraise"); // Ajoute un élément à la fin du tableau

2.3 Objets personnalisés

Vous pouvez créer vos propres objets personnalisés pour représenter des entités spécifiques à votre application.

Exemple :

javascript
// Définition d'un objet Personne
function Personne(nom, age) {
    this.nom = nom;
    this.age = age;
    this.parler = function() {
        console.log("Bonjour, je m'appelle " + this.nom + " et j'ai " + this.age + " ans.");
    };
}

// Création d'une instance de Personne
let john = new Personne("John", 30);
john.parler(); // Affiche "Bonjour, je m'appelle John et j'ai 30 ans."

En résumé, les objets en JavaScript peuvent être de types primitifs ou complexes. Les objets natifs fournissent des fonctionnalités prédéfinies pour manipuler des données, tandis que les objets personnalisés permettent de créer des structures de données adaptées à vos besoins spécifiques. Exploitez ces concepts pour écrire un code JavaScript plus efficace et modulaire.

4.2 Prototypes et héritage

Les prototypes permettent de partager des propriétés et des méthodes entre plusieurs objets. Tous les objets en JavaScript ont un prototype.

Ajouter une méthode au prototype :

javascript
Personne.prototype.presenter = function() {
    console.log("Je m'appelle " + this.nom + " et j'ai " + this.age + " ans.");
};

// Utilisation de la méthode ajoutée au prototype
john.presenter(); // Affiche "Je m'appelle John et j'ai 30 ans."

Héritage avec les prototypes :

javascript
// Définition d'une sous-classe
function Employe(nom, age, poste) {
    Personne.call(this, nom, age); // Appel du constructeur de la classe parente
    this.poste = poste;
}

// Héritage du prototype de la classe parente
Employe.prototype = Object.create(Personne.prototype);
Employe.prototype.constructor = Employe;

// Utilisation de la sous-classe
let employe1 = new Employe("Alice", 28, "Développeur");
employe1.presenter(); // Utilise la méthode du prototype de la classe parente
console.log(employe1.poste); // Affiche "Développeur"

5.1 Déclarer des classes

En ECMAScript 2015 (ES6) et les versions ultérieures, les classes ont été introduites pour simplifier la syntaxe de la programmation orientée objet en JavaScript.

Déclaration d'une classe :

javascript
class Personne {
    constructor(nom, age) {
        this.nom = nom;
        this.age = age;
    }

    presenter() {
        console.log(`Je m'appelle ${this.nom} et j'ai ${this.age} ans.`);
    }
}

// Utilisation de la classe
let john = new Personne("John", 30);
john.presenter(); // Affiche "Je m'appelle John et j'ai 30 ans."

5.2 Héritage de classe

Les classes permettent l'héritage, ce qui signifie qu'une classe peut hériter des propriétés et des méthodes d'une autre classe.

Exemple d'héritage de classe :

javascript
class Employe extends Personne {
    constructor(nom, age, poste) {
        super(nom, age); // Appel du constructeur de la classe parente
        this.poste = poste;
    }

    // Redéfinition de la méthode presenter
    presenter() {
        console.log(`Je m'appelle ${this.nom}, j'ai ${this.age} ans et je suis ${this.poste}.`);
    }
}

// Utilisation de la classe dérivée
let employe1 = new Employe("Alice", 28, "Développeur");
employe1.presenter(); // Affiche "Je m'appelle Alice, j'ai 28 ans et je suis Développeur."

5.3 Méthodes statiques

Les méthodes statiques sont des méthodes associées à la classe plutôt qu'à une instance spécifique. Elles sont définies à l'aide du mot-clé static.

Exemple de méthode statique :

javascript
class Utilitaire {
    static addition(a, b) {
        return a + b;
    }
}

// Utilisation de la méthode statique
let resultat = Utilitaire.addition(5, 7);
console.log(resultat); // Affiche 12

Les classes en JavaScript simplifient la syntaxe de la programmation orientée objet et offrent une manière plus moderne et concise de créer des objets et d'implémenter l'héritage. En comprenant ces concepts, vous serez en mesure de structurer votre code de manière plus claire et modulaire.

6.1 Extraction de propriétés

La destructuration d'objet est une fonctionnalité JavaScript qui permet d'extraire des propriétés d'un objet et de les stocker dans des variables distinctes. Cela facilite la manipulation des données contenues dans un objet.

Exemple d'extraction de propriétés :

javascript
let personne = { nom: "John", age: 30, ville: "Villeville" };

// Extraction de propriétés dans des variables
let { nom, age } = personne;

console.log(nom); // Affiche "John"
console.log(age); // Affiche 30

6.2 Assignation de nouvelles variables

La destructuration d'objet peut également être utilisée pour assigner de nouvelles variables avec des noms différents que ceux des propriétés d'origine.

Exemple d'assignation de nouvelles variables :

javascript
let personne = { nom: "John", age: 30, ville: "Villeville" };

// Assignation de nouvelles variables avec des noms différents
let { nom: prenom, age: annee } = personne;

console.log(prenom); // Affiche "John"
console.log(annee); // Affiche 30

La destructuration d'objet est utile pour simplifier l'accès aux propriétés d'un objet et pour rendre le code plus lisible. Elle est largement utilisée dans les situations où vous devez extraire des données spécifiques d'un objet.

En comprenant comment utiliser la destructuration d'objet, vous pouvez rendre votre code JavaScript plus concis et élégant, tout en améliorant la lisibilité et la maintenabilité de votre code.

7.1 Opérateur 'in'

L'opérateur in est utilisé pour vérifier si une propriété spécifique existe dans un objet. Il renvoie true si la propriété est trouvée, sinon false.

Exemple d'utilisation de l'opérateur 'in' :

javascript
let personne = { nom: "John", age: 30, ville: "Villeville" };

// Vérification de l'existence d'une propriété
console.log("nom" in personne); // Affiche true
console.log("profession" in personne); // Affiche false

L'opérateur in est utile pour éviter les erreurs lors de l'accès à des propriétés qui pourraient ne pas exister dans un objet.

7.2 Opérateur 'instanceof'

L'opérateur instanceof est utilisé pour vérifier si un objet est une instance d'une classe particulière. Il renvoie true si l'objet est une instance de la classe, sinon false.

Exemple d'utilisation de l'opérateur 'instanceof' :

javascript
class Personne {
    constructor(nom, age) {
        this.nom = nom;
        this.age = age;
    }
}

let john = new Personne("John", 30);

// Vérification de l'instance
console.log(john instanceof Personne); // Affiche true
console.log(john instanceof Object); // Affiche true (car toutes les instances sont des objets)
console.log(john instanceof Array); // Affiche false (car john n'est pas une instance d'Array)

L'opérateur instanceof est couramment utilisé pour vérifier le type d'objet dans le cadre de la programmation orientée objet.

En comprenant l'utilisation de l'opérateur in pour vérifier l'existence de propriétés et de l'opérateur instanceof pour vérifier le type d'objet, vous pouvez renforcer la robustesse de votre code en évitant les erreurs potentielles et en effectuant des vérifications de type appropriées.

8.1 Object.keys(), Object.values(), Object.entries()

En JavaScript, les méthodes Object.keys(), Object.values(), et Object.entries() sont utiles pour travailler avec les propriétés des objets.

Object.keys() :

Renvoie un tableau contenant les noms des propriétés d'un objet.

javascript
let personne = { nom: "John", age: 30, ville: "Villeville" };
let proprietes = Object.keys(personne);

console.log(proprietes); // Affiche ["nom", "age", "ville"]

Object.values() :

Renvoie un tableau contenant les valeurs des propriétés d'un objet.

javascript
let personne = { nom: "John", age: 30, ville: "Villeville" };
let valeurs = Object.values(personne);

console.log(valeurs); // Affiche ["John", 30, "Villeville"]

Object.entries() :

Renvoie un tableau de tableaux, chaque tableau interne contenant une paire clé-valeur d'un objet.

javascript
let personne = { nom: "John", age: 30, ville: "Villeville" };
let entrees = Object.entries(personne);

console.log(entrees); // Affiche [["nom", "John"], ["age", 30], ["ville", "Villeville"]]

8.2 Object.assign() et la copie d'objets

La méthode Object.assign() est utilisée pour copier les valeurs de toutes les propriétés énumérables propres de un ou plusieurs objets source vers un objet cible.

Exemple :

javascript
let objetCible = { a: 1, b: 2 };
let objetSource = { b: 3, c: 4 };

let nouvelObjet = Object.assign({}, objetCible, objetSource);

console.log(nouvelObjet); // Affiche { a: 1, b: 3, c: 4 }

Dans cet exemple, Object.assign() copie les propriétés de objetCible et objetSource dans un nouvel objet. La syntaxe {} crée un nouvel objet vide qui est la cible de la copie.

8.3 Object.freeze(), Object.seal(), Object.preventExtensions()

Ces méthodes sont utilisées pour contrôler la modification des objets.

Object.freeze() :

Empêche toute modification des propriétés d'un objet (ajout, suppression, modification de valeur).

Object.seal() :

Empêche l'ajout ou la suppression de propriétés d'un objet, mais permet la modification des valeurs existantes.

Object.preventExtensions() :

Empêche l'ajout de nouvelles propriétés à un objet, mais permet la modification des valeurs existantes et la suppression de propriétés existantes.

Exemple :

javascript
let personne = { nom: "John", age: 30 };

Object.freeze(personne);
// Ou Object.seal(personne);
// Ou Object.preventExtensions(personne);

personne.age = 31; // Pas d'erreur, mais la modification est ignorée
personne.profession = "Développeur"; // Erreur si Object.freeze(), mais pas avec Object.seal() ou Object.preventExtensions()

console.log(personne); // Affiche { nom: "John", age: 30 }

En comprenant et en utilisant ces méthodes, vous pouvez mieux contrôler la manipulation des objets dans votre code JavaScript, améliorant ainsi la stabilité et la sécurité de votre application.

9.1 Syntaxe JSON

JSON (JavaScript Object Notation) est un format de données léger qui est facile à lire et à écrire. Il est également facile à analyser et à générer pour les machines. La syntaxe JSON est dérivée de la notation des objets JavaScript, mais elle est indépendante du langage.

Voici un exemple de syntaxe JSON :

json
{
  "nom": "John",
  "age": 30,
  "ville": "Villeville",
  "estEtudiant": false,
  "competences": ["JavaScript", "HTML", "CSS"]
}

9.2 Convertir entre JSON et objets JavaScript

En JavaScript, vous pouvez convertir entre des objets JavaScript et du JSON à l'aide de méthodes spécifiques.

Convertir un objet JavaScript en JSON :

javascript
let personne = {
  nom: "John",
  age: 30,
  ville: "Villeville",
  estEtudiant: false,
  competences: ["JavaScript", "HTML", "CSS"]
};

let jsonPersonne = JSON.stringify(personne);

console.log(jsonPersonne);

Convertir du JSON en objet JavaScript :

javascript
let jsonPersonne = '{"nom":"John","age":30,"ville":"Villeville","estEtudiant":false,"competences":["JavaScript","HTML","CSS"]}';

let personne = JSON.parse(jsonPersonne);

console.log(personne);

Convertit un objet JavaScript en une chaîne JSON.

javascript

La sérialisation JSON (JSON.stringify()) et la désérialisation JSON (JSON.parse()) sont couramment utilisées pour échanger des données entre un serveur et un client, stocker des données dans des fichiers, ou transférer des données entre différentes parties d'une application. En comprenant ces méthodes, vous pouvez travailler efficacement avec des données au format JSON en JavaScript.

10.1 Références d'objets

En JavaScript, les objets sont souvent manipulés par référence. Cela signifie que lorsqu'un objet est assigné à une autre variable ou passé comme argument à une fonction, c'est la référence à l'objet qui est manipulée, pas une copie de l'objet.

Exemple de références d'objets :

javascript
let objet1 = { propriete: "valeur" };
let objet2 = objet1; // objet2 fait référence au même objet que objet1

objet2.propriete = "nouvelleValeur";
console.log(objet1.propriete); // Affiche "nouvelleValeur"

Dans cet exemple, objet1 et objet2 font référence au même objet en mémoire. Toute modification de l'objet via objet2 se reflète également dans objet1.

javascript

    console.log(propriete + ": " + personne[propriete]);
}

Object.keys() avec forEach :

javascript
let personne = { nom: "John", age: 30, ville: "Villeville" };

Object.keys(personne).forEach(function(propriete) {
    console.log(propriete + ": " + personne[propriete]);

Object.entries() avec forEach :

13.1 Nommage des propriétés

Le nommage des propriétés en JavaScript est une pratique importante pour rendre le code lisible et compréhensible. Adopter des conventions de nommage cohérentes améliore la maintenabilité du code.

Conventions de nommage :

  • Utilisez une notation camelCase pour les propriétés (par exemple, nomUtilisateur, dateNaissance).
  • Choisissez des noms significatifs qui décrivent clairement le rôle de la propriété.
  • Évitez les noms trop courts ou trop longs.
javascript
let personne = {
    nom: "John",
    age: 30,
    dateNaissance: "1992-01-15"
};

13.2 Éviter la mutation des objets

La mutation des objets (modification de leur état après leur création) peut rendre le code complexe à comprendre et à maintenir. Favorisez l'immutabilité lorsque cela est possible.

Exemple d'immutabilité :

javascript
let personne = {
    nom: "John",
    age: 30
};

// Créer une nouvelle personne avec une modification
let nouvellePersonne = { ...personne, age: 31 };

L'utilisation de la syntaxe { ...objet } crée une copie de l'objet initial, et la modification est effectuée sur cette copie plutôt que sur l'objet d'origine.

13.3 Encapsulation et abstraction

Encapsulation et abstraction sont des concepts clés en programmation orientée objet. Ils impliquent de masquer les détails internes de l'implémentation et de fournir une interface externe cohérente.

Encapsulation :

  • Encapsulez les détails internes d'un objet en les rendant privés (par exemple, avec des propriétés privées ou des méthodes privées).
  • Exposez une interface publique pour interagir avec l'objet.
javascript
function CompteBancaire(soldeInitial) {
    let solde = soldeInitial;

    this.getSolde = function() {
        return solde;
    };

    this.deposer = function(montant) {
        solde += montant;
    };

    this.retirer = function(montant) {
        if (montant <= solde) {
            solde -= montant;
        } else {
            console.log("Solde insuffisant.");
        }
    };
}

Abstraction :

  • Fournissez une interface simplifiée pour interagir avec un objet, masquant les détails complexes.
  • Permettez aux utilisateurs de l'objet de se concentrer sur l'utilisation plutôt que sur la mise en œuvre.
javascript
function Voiture(modele) {
    this.modele = modele;

    this.demarrer = function() {
        console.log("La voiture démarre.");
    };

    this.conduire = function() {
        console.log("La voiture roule.");
    };
}

let maVoiture = new Voiture("Sedan");
maVoiture.demarrer();
maVoiture.conduire();

En suivant ces bonnes pratiques, vous pouvez écrire un code JavaScript plus lisible, modulaire et maintenable. Adopter des conventions de nommage, favoriser l'immutabilité et utiliser l'encapsulation et l'abstraction amélioreront la qualité de votre code.
