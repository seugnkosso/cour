1.1 Définition des classes :

Les classes sont un concept fondamental en programmation orientée objet (POO) qui permet de créer des objets réutilisables en regroupant des données et des fonctionnalités liées. En JavaScript, les classes ont été introduites dans ECMAScript 6 (ES6), offrant une syntaxe plus claire et simplifiée pour la création d'objets.

Voici comment vous pouvez définir une classe en JavaScript :

javascript
class Personne {
  constructor(nom, age) {
    this.nom = nom;
    this.age = age;
  }

  direBonjour() {
    console.log(`Bonjour, je m'appelle ${this.nom} et j'ai ${this.age} ans.`);
  }
}

// Utilisation de la classe
const personne1 = new Personne('Alice', 30);
personne1.direBonjour();

Dans cet exemple, Personne est une classe avec un constructeur pour initialiser les propriétés nom et age, ainsi qu'une méthode direBonjour pour afficher un message de salutation.

1.2 Avantages des classes :

  • Encapsulation : Les classes encapsulent les données et le comportement associé, ce qui permet de structurer le code de manière plus organisée.

  • Réutilisabilité : Une fois qu'une classe est définie, vous pouvez créer autant d'objets de cette classe que nécessaire, favorisant la réutilisabilité du code.

  • Héritage : Les classes peuvent être héritées, permettant la création de sous-classes qui héritent des propriétés et méthodes de la classe parent.

  • Polymorphisme : Les objets créés à partir de différentes classes peuvent être traités de manière uniforme, simplifiant ainsi la gestion des types d'objets.

1.3 Relation avec la programmation orientée objet :

Les classes sont un élément essentiel de la programmation orientée objet (POO). La POO repose sur plusieurs concepts clés, dont l'encapsulation, l'héritage, le polymorphisme, et l'abstraction. Les classes en JavaScript fournissent une implémentation de ces concepts, permettant aux développeurs de structurer leur code de manière plus orientée objet.

  • Encapsulation : Les classes encapsulent les données (propriétés) et le comportement (méthodes) dans une unité, facilitant la gestion et la maintenance du code.

  • Héritage : Les classes permettent de créer une hiérarchie d'objets, où une classe peut hériter des propriétés et des méthodes d'une autre. Cela favorise la réutilisabilité du code.

  • Polymorphisme : Les classes permettent de traiter des objets de différentes classes de manière uniforme, simplifiant ainsi les opérations sur des types d'objets variés.

En résumé, les classes en JavaScript facilitent la mise en œuvre de principes de la POO, offrant une manière plus structurée et réutilisable de concevoir des programmes.

2.1 Déclaration de classe :

En ES6, la déclaration d'une classe se fait à l'aide du mot-clé class. Voici un exemple de déclaration de classe simple :

javascript
class Animal {
  constructor(nom) {
    this.nom = nom;
  }

  direBonjour() {
    console.log(`Bonjour, je suis ${this.nom}.`);
  }
}

// Utilisation de la classe
const monAnimal = new Animal('Fido');
monAnimal.direBonjour();

Dans cet exemple, une classe Animal est déclarée avec un constructeur qui prend un paramètre nom. La classe a également une méthode direBonjour qui affiche un message.

2.2 Méthodes dans une classe :

Les méthodes d'une classe peuvent être définies à l'intérieur de la déclaration de classe. Voici un exemple avec plusieurs méthodes :

javascript
class Calculatrice {

    return a + b;
  }

  soustraction(a, b) {
    return a - b;

    return a * b;

// Utilisation de la classe

Dans cet exemple, la classe Calculatrice a trois méthodes : addition, soustraction, et multiplication.

2.3 Constructeurs :

Le constructeur est une méthode spéciale appelée lors de la création d'une nouvelle instance de la classe. Il est utilisé pour initialiser les propriétés de l'objet. Exemple :

javascript
class Personne {
  constructor(nom, age) {
    this.nom = nom;
    this.age = age;

  }

// Utilisation de la classe avec le constructeur

personne1.direBonjour();

Ici, le constructeur de la classe Personne prend deux paramètres (nom et age) et initialise les propriétés correspondantes de l'instance.

3.1 Utilisation du mot-clé extends :

L'héritage en JavaScript permet à une classe d'utiliser les propriétés et les méthodes d'une autre classe. Le mot-clé extends est utilisé pour établir une relation d'héritage entre deux classes. Voici un exemple :

javascript
class Animal {
  constructor(nom) {
    this.nom = nom;
  }

  parler() {
    console.log(`${this.nom} fait du bruit.`);
  }
}

class Chien extends Animal {
  aboyer() {
    console.log(`${this.nom} aboie : Woof ! Woof !`);
  }
}

// Utilisation de l'héritage
const monChien = new Chien('Fido');
monChien.parler(); // Hérité de la classe Animal
monChien.aboyer(); // Propre à la classe Chien

Dans cet exemple, la classe Chien hérite de la classe Animal à l'aide du mot-clé extends.

3.2 Surcharge de méthodes :

La surcharge de méthodes permet à une classe enfant de fournir une implémentation différente pour une méthode héritée. Voici un exemple :

javascript
class Animal {
  parler() {
    console.log("Les animaux font du bruit.");
  }
}

class Chien extends Animal {
  parler() {
    console.log("Le chien aboie.");
  }
}

// Utilisation de la surcharge de méthode
const monChien = new Chien();
monChien.parler(); // Affiche "Le chien aboie."

La méthode parler dans la classe Chien remplace la méthode parler de la classe Animal.

3.3 Appel de la méthode parente avec super :

Lorsque vous surchargez une méthode, vous pouvez appeler la version de la méthode de la classe parente à l'aide du mot-clé super. Cela permet d'étendre le comportement existant sans le remplacer complètement. Exemple :

javascript
class Animal {
  parler() {
    console.log("Les animaux font du bruit.");
  }
}

class Chien extends Animal {
  parler() {
    super.parler(); // Appel de la méthode parente
    console.log("Le chien aboie.");
  }
}

// Utilisation de super pour appeler la méthode parente
const monChien = new Chien();
monChien.parler();

Dans cet exemple, la méthode parler de la classe Chien appelle d'abord la méthode parler de la classe Animal avant d'ajouter son propre comportement.

3.4 Méthodes statiques :

Les méthodes statiques sont associées à la classe elle-même plutôt qu'à ses instances. Elles sont appelées sur la classe plutôt que sur une instance. Voici un exemple :

javascript
class Utilitaire {
  static ajouter(a, b) {
    return a + b;
  }
}

// Utilisation d'une méthode statique
const resultat = Utilitaire.ajouter(5, 3);
console.log(resultat); // Affiche 8

5.3 Méthodes d'accès (getters et setters) :

Les méthodes d'accès, également appelées getters et setters, permettent d'accéder et de modifier les propriétés d'un objet de manière contrôlée. Exemple :

javascript
class Rectangle {
  constructor(longueur, largeur) {
    this._longueur = longueur;
    this._largeur = largeur;
  }

  get surface() {
    return this._longueur * this._largeur;
  }

  set longueur(value) {
    if (value > 0) {
      this._longueur = value;
    } else {
      console.error('La longueur doit être positive.');
    }
  }
}

// Utilisation des méthodes d'accès
const monRectangle = new Rectangle(5, 3);
console.log(monRectangle.surface); // Utilisation du getter
monRectangle.longueur = 8; // Utilisation du setter

Dans cet exemple, surface est un getter qui calcule la surface du rectangle, et longueur est un setter qui permet de modifier la longueur du rectangle.

Les méthodes de classe et les méthodes statiques peuvent souvent être utilisées de manière interchangeable. Elles sont utiles pour les fonctionnalités qui sont liées à la classe dans son ensemble plutôt qu'à des instances spécifiques.

    return `Nom : ${this.nom}, Age : ${this.age}`;
  }
}

const PersonneAvecLogger = avecLogger(Personne);
const personne1 = new PersonneAvecLogger('Alice', 30);
personne1.afficherDetails();

Dans cet exemple, avecLogger est un mixin qui ajoute la fonctionnalité de journalisation à la classe Personne.

  • Décorateurs : Les décorateurs sont une fonctionnalité qui permet d'ajouter des fonctionnalités à des classes, méthodes ou propriétés. Voici un exemple :
javascript
function logger(target, key, descriptor) {
  const original = descriptor.value;
  descriptor.value = function (...args) {
    console.log(`Logger : ${key} appelé avec ${args}`);
    return original.apply(this, args);
  };
  return descriptor;
}

class Calculatrice {
  @logger
  addition(a, b) {
    return a + b;
  }
}

const maCalculatrice = new Calculatrice();
maCalculatrice.addition(5, 3);

Dans cet exemple, le décorateur logger est appliqué à la méthode addition de la classe Calculatrice, ajoutant ainsi des logs à chaque appel de la méthode.

10.1 Nommer des classes de manière significative :

Le choix des noms pour vos classes est crucial pour la lisibilité et la maintenabilité de votre code. Utilisez des noms de classes significatifs qui reflètent clairement leur rôle ou leur responsabilité dans votre application. Évitez les noms génériques et choisissez des termes descriptifs.

javascript
// Mauvais exemple
class Xyz {
  // ...
}

// Bon exemple
class Utilisateur {
  // ...
}

10.2 Encapsulation et protection des propriétés :

Utilisez l'encapsulation pour protéger les propriétés de vos classes. En général, il est préférable de ne pas exposer directement les propriétés d'une classe, mais plutôt d'utiliser des méthodes d'accès (getters et setters) pour y accéder. Cela permet de mieux contrôler l'accès et la modification des données.

javascript
// Mauvais exemple
class Personne {
  constructor(nom, age) {
    this.nom = nom;
    this.age = age;
  }
}

// Bon exemple
class Personne {
  #nom;
  #age;

  constructor(nom, age) {
    this.#nom = nom;
    this.#age = age;
  }

  getNom() {
    return this.#nom;
  }

  setAge(age) {
    if (age > 0) {
      this.#age = age;
    }
  }
}

L'utilisation de l'encapsulation avec des champs privés (en utilisant #) et des méthodes d'accès permet de mieux contrôler l'accès aux propriétés.

10.3 Utilisation de l'héritage de manière judicieuse :

L'héritage peut être un outil puissant, mais il est important de l'utiliser judicieusement. Évitez l'héritage profond et préférez la composition lorsque cela a du sens. Choisissez l'héritage lorsque vous avez une véritable relation de spécialisation.

javascript
// Mauvais exemple (héritage inutile)
class Animal {
  // ...
}

class Chien extends Animal {
  // ...
}

// Bon exemple (composition)
class Animal {
  // ...
}

class Chien {
  constructor(animal) {
    this.animal = animal;
  }

  // ...
}

Dans le bon exemple, la classe Chien utilise la composition plutôt que l'héritage. Cela rend le code plus souple et facilite la réutilisation du code. L'héritage doit être utilisé de manière à refléter une relation "est-un", et non simplement pour partager du code.
