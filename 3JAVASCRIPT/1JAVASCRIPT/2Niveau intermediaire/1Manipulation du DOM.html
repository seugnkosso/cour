1.1 Qu'est-ce que le DOM?

Le Document Object Model, ou DOM, est une interface de programmation qui représente la structure d'un document sous forme d'arborescence d'objets. En d'autres termes, le DOM représente la page web chargée dans le navigateur et fournit une interface pour manipuler cette page à l'aide de langages de script tels que JavaScript.

Le DOM divise le document en un arbre hiérarchique, où chaque élément (balise HTML, texte, etc.) est représenté par un nœud dans cet arbre. Ces nœuds peuvent être manipulés dynamiquement en utilisant JavaScript pour changer le contenu, la structure et le style de la page sans avoir à recharger la page elle-même.

1.2 Structure hiérarchique du DOM

La structure du DOM est organisée sous forme d'arborescence, où chaque élément du document est représenté par un nœud dans l'arbre. Les principaux types de nœuds sont les suivants :

  • Document Node (nœud document): Représente tout le document.
  • Element Node (nœud élément): Représente une balise HTML.
  • Text Node (nœud texte): Représente le texte à l'intérieur d'un élément.
  • Attribute Node (nœud attribut): Représente un attribut d'un élément.
html
<!DOCTYPE html>
<html>
<head>
  <title>Exemple de DOM</title>
</head>
<body>
  <h1 id="titre">Bonjour</h1>
  <p class="paragraphe">Ceci est un exemple de paragraphe.</p>
</body>
</html>

Dans cet exemple, le DOM serait représenté comme suit :

  • Document Node
    • Element Node (html)
      • Element Node (head)
        • Element Node (title)
          • Text Node ("Exemple de DOM")
      • Element Node (body)
        • Element Node (h1)
          • Attribute Node (id)
          • Text Node ("Bonjour")
        • Element Node (p)
          • Attribute Node (class)
          • Text Node ("Ceci est un exemple de paragraphe.")

1.3 Différence entre DOM et HTML

Bien que le DOM et HTML soient liés, ils ne sont pas la même chose. HTML est un langage de balisage utilisé pour structurer le contenu d'une page web. Le DOM, d'autre part, est une représentation en mémoire de la structure d'un document HTML.

La principale différence réside dans le fait que le DOM est dynamique et peut être modifié à l'aide de JavaScript. Lorsqu'une page web est chargée dans un navigateur, le navigateur crée une représentation en mémoire de cette page sous forme de DOM. JavaScript peut ensuite être utilisé pour accéder et manipuler ces objets DOM, ce qui permet des modifications en temps réel sur la page sans nécessiter de rechargement complet.

En résumé, le HTML est la structure statique d'une page web, tandis que le DOM est une représentation dynamique de cette structure en mémoire, manipulable à l'aide de JavaScript.

2.1 getElementById

La méthode getElementById est utilisée pour sélectionner un élément en fonction de son attribut id. Elle renvoie un seul élément qui a l'identifiant spécifié.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Sélection d'éléments</title>
</head>
<body>
  <h1 id="titre">Titre de la page</h1>
  <p id="paragraphe">Ceci est un paragraphe.</p>

  <script>
    // Sélection de l'élément avec l'id "titre"
    var titreElement = document.getElementById("titre");
    console.log(titreElement.textContent);
  </script>
</body>
</html>

Dans cet exemple, titreElement contiendra la référence à l'élément <h1> avec l'id "titre".

2.2 getElementsByClassName

La méthode getElementsByClassName sélectionne tous les éléments qui ont une classe CSS spécifique. Elle retourne une collection d'éléments.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Sélection d'éléments</title>
</head>
<body>
  <p class="paragraphe">Paragraphe 1</p>
  <p class="paragraphe">Paragraphe 2</p>

  <script>
    // Sélection de tous les éléments avec la classe "paragraphe"
    var paragraphs = document.getElementsByClassName("paragraphe");
    
    for (var i = 0; i < paragraphs.length; i++) {
      console.log(paragraphs[i].textContent);
    }
  </script>
</body>
</html>

Cet exemple sélectionne tous les éléments avec la classe "paragraphe" et affiche leur contenu.

2.3 getElementsByTagName

La méthode getElementsByTagName sélectionne tous les éléments avec un nom de balise HTML spécifié. Elle retourne une collection d'éléments.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Sélection d'éléments</title>
</head>
<body>
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
  </ul>

  <script>
    // Sélection de tous les éléments li dans la liste
    var listItems = document.getElementsByTagName("li");
    
    for (var i = 0; i < listItems.length; i++) {
      console.log(listItems[i].textContent);
    }
  </script>
</body>
</html>

Cet exemple sélectionne tous les éléments <li> dans la liste et affiche leur contenu.

2.4 querySelector et querySelectorAll

La méthode querySelector sélectionne le premier élément qui correspond à un sélecteur CSS spécifié, tandis que querySelectorAll sélectionne tous les éléments qui correspondent au sélecteur.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Sélection d'éléments</title>
</head>
<body>
  <div id="container">
    <p class="important">Paragraphe important</p>
    <p>Paragraphe ordinaire</p>
  </div>

  <script>
    // Sélection du premier paragraphe avec la classe "important"
    var importantParagraph = document.querySelector("#container .important");
    console.log(importantParagraph.textContent);
  </script>
</body>
</html>

Dans cet exemple, importantParagraph contiendra le premier paragraphe avec la classe "important" à l'intérieur de l'élément avec l'id "container".

2.5 Traverser le DOM

Il existe plusieurs méthodes pour traverser le DOM, permettant de naviguer entre les nœuds. Voici quelques-unes des méthodes couramment utilisées :

  • parentNode: Renvoie le nœud parent.
  • childNodes: Renvoie une liste de tous les nœuds enfants.
  • firstChild et lastChild: Renvoient le premier et le dernier nœud enfant, respectivement.
  • nextSibling et previousSibling: Renvoient le nœud suivant et précédent, respectivement.
  • querySelector et querySelectorAll: Peuvent également être utilisés pour effectuer des sélections plus avancées en combinant les sélecteurs.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Traverser le DOM</title>
</head>
<body>
  <ul id="maListe">
    <li>Item 1</li>
    <li>Item 2</li>
    <li>Item 3</li>
  </ul>

  <script>
    // Traverser le DOM pour accéder au deuxième élément de la liste
    var liste = document.getElementById("maListe");
    var deuxiemeElement = liste.children[1];
    console.log(deuxiemeElement.textContent);
  </script>
</body>
</html>

Dans cet exemple, deuxiemeElement contiendra une référence au deuxième élément de la liste, et son contenu sera affiché dans la console.

4.1 Introduction aux événements

Les événements JavaScript permettent aux développeurs de détecter les actions de l'utilisateur et d'y répondre de manière dynamique. Les événements peuvent inclure des actions telles que le clic de la souris, l'appui sur une touche du clavier, le chargement de la page, etc.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Gestion des événements</title>
</head>
<body>

  <button id="monBouton">Cliquez-moi</button>

  <script>
    // Fonction de gestion de l'événement clic
    function handleClick() {
      alert("Bouton cliqué !");
    }

    // Ajout d'un écouteur d'événement au bouton
    var bouton = document.getElementById("monBouton");
    bouton.addEventListener("click", handleClick);
  </script>

</body>
</html>

Dans cet exemple, un événement de clic est détecté sur le bouton, et la fonction handleClick est appelée en réponse.

4.2 Ajouter des écouteurs d'événements

L'ajout d'écouteurs d'événements est une méthode cruciale pour gérer les interactions utilisateur. La méthode addEventListener est utilisée pour associer un gestionnaire d'événements à un élément spécifique.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Gestion des événements</title>
</head>
<body>

  <button id="monBouton">Cliquez-moi</button>

  <script>

    }

    // Ajout d'un écouteur d'événement au bouton
    var bouton = document.getElementById("monBouton");
    bouton.addEventListener("click", handleClick);
  </script>

</body>
</html>

Dans cet exemple, la fonction handleClick est appelée lorsque le bouton est cliqué.

4.3 Événements de la souris et du clavier

Il existe plusieurs événements liés à la souris et au clavier. Voici quelques-uns des événements couramment utilisés :

  • click: Se déclenche lorsqu'un élément est cliqué.
  • mouseover et mouseout: Se déclenchent lorsque la souris survole ou quitte un élément.
  • keydown et keyup: Se déclenchent lorsqu'une touche du clavier est enfoncée ou relâchée.
  • mousemove: Se déclenche lorsque la souris est déplacée.

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Gestion des événements</title>
  <style>
    #monElement {
      width: 100px;
      height: 100px;
      background-color: lightblue;
    }
  </style>
</head>
<body>

  <div id="monElement">Survolez-moi</div>

  <script>
    var monElement = document.getElementById("monElement");

    monElement.addEventListener("mouseover", function() {
      monElement.style.backgroundColor = "lightgreen";
    });

    monElement.addEventListener("mouseout", function() {
      monElement.style.backgroundColor = "lightblue";
    });
  </script>

</body>
</html>

Dans cet exemple, la couleur de fond de l'élément change lorsque la souris le survole.

4.4 Éviter la propagation d'événements (event propagation)

Lorsqu'un événement se produit sur un élément, il peut également affecter ses éléments parents ou enfants, en fonction de la propagation des événements. Vous pouvez arrêter la propagation des événements en utilisant les méthodes stopPropagation() ou stopImmediatePropagation().

Exemple :

html
<!DOCTYPE html>
<html>
<head>
  <title>Gestion des événements</title>
</head>
<body>

  <div id="parent">
    <button id="enfant">Cliquez-moi</button>
  </div>

  <script>
    var parent = document.getElementById("parent");
    var enfant = document.getElementById("enfant");

    parent.addEventListener("click", function() {
      alert("Clic sur le parent");
    });

    enfant.addEventListener("click", function(event) {
      alert("Clic sur l'enfant");
      event.stopPropagation(); // Arrête la propagation de l'événement
    });
  </script>

</body>
</html>

Dans cet exemple, sans l'appel à stopPropagation(), le clic sur le bouton déclencherait également l'alerte du parent. Cependant, avec stopPropagation(), la propagation de l'événement est stoppée, et seul l'événement de l'enfant est géré.

6.2 Validation des formulaires en utilisant le DOM

La validation des formulaires côté client peut être effectuée en utilisant le DOM pour vérifier si les champs sont remplis correctement.

Exemple de validation de formulaire :

html
<!DOCTYPE html>
<html>
<head>
  <title>Travailler avec les formulaires</title>
  <style>
    .erreur {
      color: red;
    }
  </style>
</head>
<body>

  <form id="monFormulaire">
    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom">

    <label for="email">Email :</label>
    <input type="email" id="email" name="email">
    
    <span id="messageErreurNom" class="erreur"></span>
    <span id="messageErreurEmail" class="erreur"></span>

    <input type="submit" value="Envoyer">
  </form>

  <script>
    var formulaire = document.getElementById("monFormulaire");
    var messageErreurNom = document.getElementById("messageErreurNom");
    var messageErreurEmail = document.getElementById("messageErreurEmail");

    formulaire.addEventListener("submit", function(event) {
      event.preventDefault(); // Empêche l'envoi du formulaire (pour cet exemple)

      // Validation du nom (doit être rempli)
      var nom = document.getElementById("nom").value;
      if (nom === "") {
        messageErreurNom.textContent = "Le nom est obligatoire.";
      } else {
        messageErreurNom.textContent = "";
      }

      // Validation de l'email (doit être une adresse email valide)
      var email = document.getElementById("email").value;
      var regexEmail = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
      if (!regexEmail.test(email)) {
        messageErreurEmail.textContent = "Veuillez entrer une adresse email valide.";
      } else {
        messageErreurEmail.textContent = "";
      }

      // Soumettre le formulaire si la validation réussit
      if (nom !== "" && regexEmail.test(email)) {
        // Simuler l'envoi du formulaire (remplacer par le vrai code d'envoi)
        alert("Formulaire soumis avec succès !");
      }
    });
  </script>

</body>
</html>

Dans cet exemple, la validation est effectuée en vérifiant si les champs sont remplis correctement. Les erreurs sont affichées à côté des champs correspondants.

8.1 Introduction à AJAX

AJAX (Asynchronous JavaScript and XML) est une technique de programmation qui permet de créer des applications web interactives. L'objectif principal d'AJAX est de permettre la mise à jour d'une page web sans recharger toute la page.

Avantages d'AJAX :

  • Mise à jour partielle des pages sans rechargement complet.
  • Réponses asynchrones du serveur.
  • Meilleure expérience utilisateur.

Exemple simple d'utilisation d'AJAX :

html
<!DOCTYPE html>
<html>
<head>
  <title>AJAX et manipulation asynchrone</title>
</head>
<body>

  <h1 id="monTitre">Cliquez pour charger les données</h1>
  <div id="contenu"></div>

  <script>
    var monTitre = document.getElementById("monTitre");
    var contenu = document.getElementById("contenu");

    monTitre.addEventListener("click", function() {
      // Création de l'objet XMLHttpRequest
      var xhr = new XMLHttpRequest();

      // Configuration de la requête
      xhr.open("GET", "donnees.json", true);

      // Gestionnaire d'événement de chargement
      xhr.onload = function() {
        if (xhr.status >= 200 && xhr.status < 300) {
          // Succès : la requête a réussi
          var donnees = JSON.parse(xhr.responseText);
          contenu.textContent = "Données chargées : " + donnees.message;
        } else {
          // Échec : la requête a échoué
          console.error("La requête a échoué avec le statut :", xhr.status);
        }
      };

      // Gestionnaire d'événement d'erreur
      xhr.onerror = function() {
        console.error("Erreur réseau lors de la requête.");
      };

      // Envoi de la requête
      xhr.send();
    });
  </script>

</body>
</html>

Dans cet exemple, un clic sur le titre déclenche une requête AJAX pour récupérer des données à partir d'un fichier JSON.

</body>

Dans cet exemple, le nom entré par l'utilisateur est sauvegardé dans localStorage, et lors du rechargement de la page, s'il y a déjà un nom sauvegardé, un message de bienvenue est affiché.

10.1 Éviter les attaques XSS (Cross-Site Scripting)

Les attaques XSS (Cross-Site Scripting) peuvent permettre à des attaquants d'injecter du code malveillant dans des pages web, compromettant ainsi la sécurité des utilisateurs. Pour éviter cela, suivez ces bonnes pratiques :

Utiliser textContent au lieu de innerHTML :

javascript
// Éviter
element.innerHTML = userInput;

// Préférer
element.textContent = userInput;

L'utilisation de textContent garantit que le contenu est traité comme du texte brut, réduisant ainsi le risque d'injection de scripts.

Échapper les caractères spéciaux :

javascript
// Éviter
element.innerHTML = userInput;

// Préférer
element.textContent = DOMPurify.sanitize(userInput);

Utilisez une bibliothèque de nettoyage de HTML comme DOMPurify pour échapper les caractères spéciaux et supprimer tout contenu malveillant.

10.2 Validation côté client vs côté serveur

La validation côté client est une bonne pratique pour améliorer l'expérience utilisateur, mais elle ne suffit pas à garantir la sécurité. La validation côté serveur est essentielle pour empêcher les attaques potentielles en cas de contournement de la validation côté client. Voici quelques recommandations :

Valider côté client pour une expérience utilisateur améliorée :

javascript
// Valider côté client (exemple avec une adresse email)
function validerEmail(email) {
  return /\S+@\S+\.\S+/.test(email);
}

Valider côté serveur pour une sécurité renforcée :

javascript
// Valider côté serveur (exemple avec Node.js)
app.post('/enregistrement', function(req, res) {
  if (validerEmail(req.body.email)) {
    // Traitement si la validation réussit
    res.send('Enregistrement réussi !');
  } else {
    // Gérer l'erreur si la validation échoue
    res.status(400).send('Adresse email invalide.');
  }
});

10.3 Optimisations de performance

L'optimisation des performances est cruciale pour offrir une expérience utilisateur fluide. Voici quelques bonnes pratiques :

Minification du code :

Utilisez des outils de minification pour réduire la taille des fichiers JavaScript, CSS et HTML, ce qui accélère le chargement des pages.

Utilisation de la mise en cache :

Configurez les en-têtes HTTP pour tirer parti de la mise en cache du navigateur. Cela réduit les demandes au serveur et accélère le chargement des pages pour les visiteurs récurrents.

Chargement asynchrone des scripts :

Utilisez l'attribut async pour les scripts non critiques afin qu'ils ne bloquent pas le rendu de la page.

html
<script async src="script-non-critique.js"></script>

Compression Gzip :

Configurez le serveur pour compresser les ressources avec Gzip. Cela réduit la taille des fichiers avant de les envoyer au client.

Chargement différé des images :

Retardez le chargement des images qui ne sont pas immédiatement visibles à l'écran en utilisant l'attribut loading ou des bibliothèques de chargement d'images paresseuses.

html
<img loading="lazy" src="image-non-critique.jpg" alt="Image non critique">

En suivant ces bonnes pratiques, vous améliorez la sécurité de votre application web, garantissez une validation robuste des données et optimisez les performances pour offrir une meilleure expérience utilisateur.
