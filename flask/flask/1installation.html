Installation¶

Version Python¶

Nous vous recommandons d'utiliser la dernière version de Python. Flask prend en charge Python 3.8 et plus récent.

Dépendances¶

Ces distributions seront installées automatiquement lors de l'installation de Flask.

  • Werkzeug implémente WSGI, l'interface Python standard entre applications et serveurs.

  • Jinja est un langage de modèle qui rend les pages de votre application sert.

  • MarkupSafe vient avec Jinja. Il échappe aux entrées non fiables lors du rendu modèles pour éviter les attaques par injection.

  • C'est dangereux signe en toute sécurité des données pour garantir son intégrité. Ceci est utilisé pour protéger le cookie de session de Flask.

  • Cliquez sur est un cadre pour l'écriture d'applications de ligne de commande. Il fournit le flask commande et permet d'ajouter des commandes de gestion personnalisées.

  • Clignotant fournit un support pour Signaux.

Dépendances facultatives¶

Ces distributions ne seront pas installées automatiquement. Flacon détectera et utilisez-les si vous les installez.

  • python-dotenv permet la prise en charge de Variables d'environnement de dotenv en cours d'exécution flask commandes.

  • Chien de garde fournit un rechargeur plus rapide et plus efficace pour le développement serveur.

écru¶

Vous pouvez choisir d'utiliser gevent ou eventlet avec votre application. Dans ce cas, greenlet > = 1.0 est requis. Lorsque vous utilisez PyPy, PyPy > = 7.3.7 est obligatoire.

Ce ne sont pas des versions prises en charge minimales, elles indiquent uniquement la première des versions qui ont ajouté les fonctionnalités nécessaires. Vous devez utiliser les dernières versions de chacun.

Environnements virtuels¶

Utilisez un environnement virtuel pour gérer les dépendances de votre projet, les deux développement et production.

Quel problème un environnement virtuel résout-il? Plus Python vous projette plus il est probable que vous deviez travailler avec différentes versions de Bibliothèques Python, ou même Python lui-même. Versions plus récentes des bibliothèques pour un le projet peut rompre la compatibilité dans un autre projet.

Les environnements virtuels sont des groupes indépendants de bibliothèques Python, un pour chacun projet. Les packages installés pour un projet n'affecteront pas d'autres projets ou les packages du système d'exploitation.

Python est livré avec le venv module pour créer virtuel environnements.

Créer un environnement¶

Créer un dossier de projet et un .venv dossier à l'intérieur:

$ mkdir myproject
$ cd myproject
$ python3 -m venv .venv

> mkdir myproject
> cd myproject
> py -3 -m venv .venv

Activer l'environnement¶

Avant de travailler sur votre projet, activez l'environnement correspondant:

$ . .venv/bin/activate

> .venv\Scripts\activate

Votre invite shell changera pour afficher le nom de l'activé environnement.

Installer Flask¶

Dans l'environnement activé, utilisez la commande suivante pour installer Flacon:

$ pip install Flask

Flask est maintenant installé. Découvrez le Démarrage ou aller au Documentation Aperçu.
