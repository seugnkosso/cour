Introduction à CSRF (Cross-Site Request Forgery) avec Flask

Définition de CSRF : CSRF (Cross-Site Request Forgery) est une attaque web malveillante qui exploite l'authentification implicite des utilisateurs sur un site web. L'attaque se produit lorsque l'utilisateur est authentifié sur un site (comme une application Flask) et qu'un attaquant parvient à faire exécuter des actions non autorisées sur ce site en utilisant les informations d'authentification de l'utilisateur sans son consentement.

Fonctionnement de l'attaque CSRF : L'attaque CSRF exploite le fait que de nombreux sites web utilisent des mécanismes d'authentification basés sur des sessions. Lorsqu'un utilisateur se connecte à un site, une session est créée et un identifiant de session unique est stocké sur le navigateur de l'utilisateur. Ce cookie de session est envoyé automatiquement avec chaque requête HTTP vers le site, permettant au serveur de reconnaître et de maintenir l'état de l'utilisateur.

L'attaque CSRF se déroule généralement de la manière suivante :

  1. L'utilisateur se connecte à un site web légitime (par exemple, une application Flask) et obtient un cookie de session.
  2. Pendant que l'utilisateur est toujours connecté, il visite un autre site malveillant contrôlé par l'attaquant.
  3. Le site malveillant contient du code (généralement du code JavaScript) qui envoie des requêtes HTTP vers le site légitime en utilisant les informations d'authentification de l'utilisateur (le cookie de session).
  4. Lorsque ces requêtes atteignent le site légitime, celui-ci les traite comme des requêtes légitimes de l'utilisateur, car elles incluent le cookie de session valide. Ainsi, l'attaquant peut effectuer des actions non autorisées au nom de l'utilisateur, telles que la modification des paramètres du compte, la publication de contenu, etc.

Mécanismes de contournement utilisés par les attaquants : Les attaquants utilisent diverses techniques pour exploiter les failles CSRF, notamment :

  1. Liens malveillants : Les attaquants peuvent créer des liens malveillants qui incitent les utilisateurs à cliquer dessus. Ces liens peuvent déclencher des actions non autorisées sur des sites où l'utilisateur est connecté.

  2. Images et balises : Les attaquants peuvent utiliser des balises <img> ou <script> pour charger des ressources à partir de sites légitimes tout en incluant des requêtes forgées dans les URL ou les paramètres de requête.

  3. Formulaires cachés : Les attaquants peuvent cacher des formulaires dans des pages web malveillantes et les soumettre automatiquement via JavaScript sans que l'utilisateur ne le remarque.

  4. Redirections : Les attaquants peuvent rediriger les utilisateurs vers des pages contenant des actions malveillantes après qu'ils se soient connectés à un site légitime.

Pour contrer les attaques CSRF, les applications web, y compris celles développées avec Flask, utilisent des mesures de protection telles que les jetons CSRF (Cross-Site Request Forgery tokens) et les en-têtes HTTP spécifiques pour vérifier l'origine des requêtes. Ces techniques aident à garantir que les requêtes proviennent d'une source légitime et ne sont pas forgées par des attaquants.

Risques associés à CSRF dans les applications web avec Flask

CSRF (Cross-Site Request Forgery) est une menace sérieuse pour la sécurité des applications web, y compris celles développées avec Flask. Voici les principaux risques associés à CSRF dans les applications web :

1. Altération non autorisée des données utilisateur : L'un des principaux risques de CSRF est que les attaquants peuvent altérer les données utilisateur sans autorisation. Par exemple, un attaquant peut envoyer une requête CSRF pour modifier les informations personnelles d'un utilisateur, telles que son adresse e-mail, son mot de passe, ses préférences de compte, etc. Cela peut entraîner une perte de confidentialité et une compromission de l'intégrité des données de l'utilisateur.

2. Exécution d'actions malveillantes au nom de l'utilisateur : Avec CSRF, les attaquants peuvent forcer les utilisateurs authentifiés à effectuer des actions malveillantes sans leur consentement. Par exemple, un attaquant peut envoyer une requête CSRF pour effectuer des transactions financières, publier du contenu inapproprié sur des forums ou des réseaux sociaux, supprimer des données importantes, etc. Ces actions peuvent nuire à la réputation de l'utilisateur et causer des dommages financiers ou autres.

3. Contournement des contrôles d'authentification : Les attaques CSRF contournent les contrôles d'authentification en exploitant l'authentification implicite de l'utilisateur sur un site web. Même si un site utilise des mécanismes robustes d'authentification, tels que des mots de passe forts, des vérifications en deux étapes, etc., les attaquants peuvent contourner ces contrôles en exploitant la session authentifiée de l'utilisateur. Cela peut permettre aux attaquants d'accéder à des fonctionnalités sensibles ou restreintes qui nécessitent une authentification valide.

Pour atténuer les risques associés à CSRF, il est essentiel de mettre en œuvre des mesures de protection appropriées dans les applications Flask, telles que l'utilisation de jetons CSRF, la vérification de l'origine des requêtes à l'aide d'en-têtes HTTP spécifiques, et la sensibilisation des utilisateurs aux dangers potentiels de l'ouverture de liens ou de l'exécution de scripts à partir de sources non fiables. De plus, une surveillance proactive des activités suspectes et des mises à jour régulières de sécurité sont indispensables pour prévenir et atténuer les risques de sécurité liés à CSRF.

Utilisation de jetons CSRF (CSRF tokens) pour protéger les formulaires dans Flask

Les jetons CSRF (Cross-Site Request Forgery) sont une mesure de sécurité cruciale pour protéger les applications web, y compris celles développées avec Flask, contre les attaques CSRF. Voici comment les utiliser pour sécuriser les formulaires dans votre application Flask :

1. Concept de jetons CSRF : Les jetons CSRF sont des valeurs aléatoires uniques associées à chaque session utilisateur. Lorsqu'un utilisateur authentifié soumet un formulaire, l'application web inclut ce jeton CSRF dans le formulaire sous forme d'un champ caché. Lorsque le formulaire est soumis, l'application vérifie que le jeton CSRF inclus dans la requête correspond à celui associé à la session utilisateur. Si les jetons ne correspondent pas, la requête est considérée comme suspecte et est rejetée.

2. Génération sécurisée de jetons CSRF : Dans Flask, vous pouvez générer des jetons CSRF en utilisant la fonction generate_csrf() fournie par le module csrf. Assurez-vous d'activer la protection CSRF dans votre application Flask en configurant la clé SECRET_KEY pour garantir que les jetons sont sécurisés. Voici un exemple de génération de jetons CSRF dans Flask :

python
from flask import Flask, render_template, request, csrf

app = Flask(__name__)
app.config['SECRET_KEY'] = 'votre_clé_secrète'

@app.route('/')
def index():
    csrf_token = csrf.generate_csrf()
    return render_template('index.html', csrf_token=csrf_token)

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous générons un jeton CSRF à l'aide de csrf.generate_csrf() et le passons à notre modèle index.html pour être inclus dans un formulaire.

3. Stockage et transmission des jetons CSRF : Une fois généré, le jeton CSRF doit être stocké côté serveur (généralement dans la session utilisateur) et transmis côté client (généralement dans un champ de formulaire caché). Lorsque l'utilisateur soumet un formulaire, le jeton CSRF est envoyé avec la requête HTTP. Vous pouvez utiliser des formulaires Flask-WTF pour simplifier la gestion des jetons CSRF. Voici un exemple d'utilisation de formulaires Flask-WTF avec protection CSRF :

python
from flask_wtf import FlaskForm
from wtforms import StringField, SubmitField
from wtforms.csrf.session import SessionCSRF

class MyForm(FlaskForm):
    name = StringField('Nom')
    submit = SubmitField('Envoyer', render_kw={'formnovalidate': True})

app.config['WTF_CSRF_SECRET_KEY'] = 'votre_clé_secrète'
app.config['WTF_CSRF_TIME_LIMIT'] = None  # Désactivez la limite de temps CSRF
app.config['WTF_CSRF_METHODS'] = ['POST', 'PUT', 'PATCH', 'DELETE']  # Méthodes pour lesquelles CSRF sera appliqué
app.csrf = SessionCSRF()

@app.route('/', methods=['GET', 'POST'])
def index():
    form = MyForm()
    if form.validate_on_submit():
        # Traiter le formulaire
        return 'Formulaire soumis avec succès !'
    return render_template('index.html', form=form)

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous utilisons Flask-WTF pour gérer les formulaires, ce qui inclut automatiquement la protection CSRF. La clé WTF_CSRF_SECRET_KEY est utilisée pour sécuriser les jetons CSRF, et WTF_CSRF_METHODS spécifie les méthodes HTTP pour lesquelles la protection CSRF est appliquée.

En utilisant des jetons CSRF dans vos formulaires Flask, vous pouvez renforcer considérablement la sécurité de votre application en protégeant contre les attaques CSRF.

Intégration de la protection CSRF dans les formulaires Flask

La protection CSRF (Cross-Site Request Forgery) est cruciale pour sécuriser les formulaires dans les applications Flask contre les attaques malveillantes. Vous pouvez l'intégrer à votre application de différentes manières, notamment en configurant la protection CSRF dans Flask, en utilisant la fonction csrf_protect de Flask-WTF, ou en mettant en œuvre manuellement la protection CSRF. Voici comment faire :

1. Configuration de la protection CSRF dans Flask : Vous pouvez activer la protection CSRF dans votre application Flask en configurant la clé SECRET_KEY et en utilisant l'extension CSRFProtect. Voici un exemple de configuration dans votre application Flask :

python
from flask import Flask
from flask_wtf.csrf import CSRFProtect

app = Flask(__name__)
app.config['SECRET_KEY'] = 'votre_clé_secrète'

# Activer la protection CSRF
csrf = CSRFProtect(app)

Avec cette configuration, Flask utilisera la clé secrète spécifiée pour générer et vérifier les jetons CSRF.

2. Utilisation de la fonction csrf_protect de Flask-WTF : Si vous utilisez Flask-WTF pour gérer les formulaires dans votre application Flask, vous pouvez simplement utiliser la fonction csrf_protect pour activer la protection CSRF. Voici un exemple :

python
from flask import Flask, render_template
from flask_wtf.csrf import CSRFProtect

app = Flask(__name__)
app.config['SECRET_KEY'] = 'votre_clé_secrète'

# Activer la protection CSRF
csrf = CSRFProtect(app)

@app.route('/', methods=['GET', 'POST'])
@csrf_protect
def index():
    # Votre logique de gestion de formulaire ici
    return render_template('index.html')

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, la fonction csrf_protect est utilisée comme un décorateur pour protéger la route index des attaques CSRF.

3. Implémentation manuelle de la protection CSRF : Si vous préférez implémenter manuellement la protection CSRF, vous pouvez le faire en générant et en vérifiant les jetons CSRF dans vos vues Flask. Voici un exemple de comment cela pourrait être fait :

python
from flask import Flask, render_template, request, session
import os

app = Flask(__name__)
app.config['SECRET_KEY'] = os.urandom(24)

@app.route('/', methods=['GET', 'POST'])
def index():
    if request.method == 'POST':
        # Vérifier si le jeton CSRF est valide
        token = session.pop('_csrf_token', None)
        if not token or token != request.form.get('_csrf_token'):
            abort(403)  # Jeton CSRF invalide, rejeter la requête
        # Traitement du formulaire ici
        return 'Formulaire soumis avec succès !'
    # Générer un jeton CSRF et l'enregistrer dans la session
    session['_csrf_token'] = os.urandom(24).hex()
    return render_template('index.html', csrf_token=session['_csrf_token'])

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous générons un jeton CSRF aléatoire pour chaque session utilisateur et le stockons dans la session Flask. Lorsque le formulaire est soumis, nous vérifions si le jeton CSRF inclus dans la requête correspond à celui stocké dans la session. Si les jetons ne correspondent pas, la requête est rejetée avec une erreur 403.

En intégrant la protection CSRF dans vos formulaires Flask, vous pouvez renforcer la sécurité de votre application en protégeant contre les attaques CSRF. Choisissez la méthode qui convient le mieux à votre application en fonction de vos besoins et de votre infrastructure existante.

1. Mise en œuvre d'un formulaire de connexion sécurisé :

Nous allons créer une application Flask avec un formulaire de connexion sécurisé, où nous appliquerons la protection CSRF.

python
from flask import Flask, render_template, request, session
from flask_wtf.csrf import CSRFProtect
from wtforms import StringField, PasswordField, SubmitField, Form
from wtforms.validators import DataRequired

app = Flask(__name__)
app.config['SECRET_KEY'] = 'your_secret_key'
csrf = CSRFProtect(app)

class LoginForm(Form):
    username = StringField('Nom d\'utilisateur', validators=[DataRequired()])
    password = PasswordField('Mot de passe', validators=[DataRequired()])
    submit = SubmitField('Se connecter')

@app.route('/login', methods=['GET', 'POST'])
def login():
    form = LoginForm()
    if form.validate_on_submit():
        # Ici, vous authentifiez l'utilisateur
        return f'Bienvenue, {form.username.data} !'
    return render_template('login.html', form=form)

if __name__ == '__main__':
    app.run(debug=True)

2. Application de la protection CSRF à différents types de formulaires :

Nous pouvons appliquer la protection CSRF à différents types de formulaires dans Flask en utilisant le décorateur @csrf_protect.

python
from flask_wtf import FlaskForm
from wtforms import StringField, SubmitField
from wtforms.validators import DataRequired

class MyForm(FlaskForm):
    name = StringField('Nom', validators=[DataRequired()])
    submit = SubmitField('Envoyer')

@app.route('/form', methods=['GET', 'POST'])
@csrf_protect
def my_form():
    form = MyForm()
    if form.validate_on_submit():
        # Traitement du formulaire
        return f'Données reçues : {form.name.data}'
    return render_template('form.html', form=form)
