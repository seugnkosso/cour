Prérequis

Avant de commencer, assurez-vous d'avoir Python et Flask installés sur votre système. Vous pouvez les installer en utilisant pip :

bash
pip install Flask

Création d'une Application Flask de Base

Commençons par créer une application Flask simple avec une route qui répond aux différentes méthodes HTTP.

python
from flask import Flask, request

app = Flask(__name__)

@app.route('/data', methods=['GET', 'POST', 'PUT', 'DELETE'])
def handle_data():
    if request.method == 'GET':
        return "Récupération de données"
    elif request.method == 'POST':
        return "Soumission de données"
    elif request.method == 'PUT':
        return "Modification de données"
    elif request.method == 'DELETE':
        return "Suppression de données"

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous avons une route /data qui accepte les méthodes GET, POST, PUT et DELETE.

Tester l'Application

Vous pouvez tester cette application en utilisant n'importe quel client HTTP, comme cURL ou Postman.

  1. GET : Pour récupérer des données

    bash
    curl -X GET http://127.0.0.1:5000/data

    Vous devriez obtenir comme réponse : "Récupération de données"

  2. POST : Pour soumettre des données

    bash
    curl -X POST http://127.0.0.1:5000/data

    Vous devriez obtenir comme réponse : "Soumission de données"

  3. PUT : Pour modifier des données

    bash
    curl -X PUT http://127.0.0.1:5000/data

    Vous devriez obtenir comme réponse : "Modification de données"

  4. DELETE : Pour supprimer des données

    bash
    curl -X DELETE http://127.0.0.1:5000/data

    Vous devriez obtenir comme réponse : "Suppression de données"

Conclusion

Félicitations ! Vous avez maintenant créé une application Flask de base qui gère les méthodes HTTP GET, POST, PUT et DELETE. Vous pouvez étendre cette application pour inclure la logique métier appropriée pour chaque méthode HTTP, par exemple en interagissant avec une base de données ou en effectuant d'autres opérations sur les données.

Extraction des Paramètres de l'URL (Query Parameters)

Les paramètres de l'URL, également appelés Query Parameters, sont des informations ajoutées à la fin de l'URL après un point d'interrogation (?). Par exemple, dans l'URL http://exemple.com/recherche?q=python, le paramètre q a la valeur python.

Voici comment récupérer ces paramètres dans Flask :

python
from flask import Flask, request

app = Flask(__name__)

@app.route('/search')
def search():
    query = request.args.get('q')
    return f"Recherche effectuée avec le terme : {query}"

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, la fonction request.args.get() est utilisée pour extraire les paramètres de la requête GET.

Analyse des Données Soumises via POST (Form Data)

Lorsque vous soumettez un formulaire HTML via la méthode POST, les données sont généralement envoyées sous forme de formulaire. Vous pouvez récupérer ces données dans Flask de la manière suivante :

python
from flask import Flask, request

app = Flask(__name__)

@app.route('/submit', methods=['POST'])
def submit_form():
    username = request.form.get('username')
    password = request.form.get('password')
    return f"Nom d'utilisateur : {username}, Mot de passe : {password}"

if __name__ == '__main__':
    app.run(debug=True)

Ici, request.form.get() est utilisé pour récupérer les données envoyées via POST.

Traitement des Données JSON envoyées dans le Corps de la Requête

Parfois, les données sont envoyées sous forme de JSON dans le corps de la requête. Voici comment les récupérer dans Flask :

python
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/json', methods=['POST'])
def process_json():
    data = request.json
    return jsonify(data)

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, request.json est utilisé pour obtenir les données JSON de la requête.

Conclusion

Vous avez appris à récupérer des données de requête dans Flask à partir de différents types de sources : les paramètres de l'URL, les données de formulaire et les données JSON. Ces techniques sont utiles pour traiter les entrées utilisateur dans vos applications web Flask.

Validation des Données de Formulaire

La validation des données de formulaire est essentielle pour s'assurer que les utilisateurs fournissent des informations valides. Voici un exemple de validation des données de formulaire en utilisant Flask :

python
from flask import Flask, request

app = Flask(__name__)

@app.route('/submit', methods=['POST'])
def submit_form():
    username = request.form.get('username')
    password = request.form.get('password')

    # Validation des données de formulaire
    if not username or not password:
        return "Veuillez fournir un nom d'utilisateur et un mot de passe", 400
    
    # Autres validations...
    
    return "Formulaire soumis avec succès"

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous vérifions simplement si le nom d'utilisateur et le mot de passe sont présents. Vous pouvez ajouter d'autres validations selon les besoins de votre application.

Vérification des Données de Requête JSON

Si vous recevez des données JSON dans le corps de la requête, vous devez également les valider. Voici un exemple de validation des données JSON en utilisant Flask :

python
from flask import Flask, request, jsonify

app = Flask(__name__)

@app.route('/json', methods=['POST'])
def process_json():
    data = request.json

    # Validation des données JSON
    if 'username' not in data or 'password' not in data:
        return "Veuillez fournir un nom d'utilisateur et un mot de passe", 400
    
    # Autres validations...
    
    return jsonify(data)

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous vérifions si les clés 'username' et 'password' sont présentes dans les données JSON reçues.

Utilisation de la Bibliothèque de Validation WTForms

WTForms est une bibliothèque de validation populaire pour Flask. Voici comment vous pouvez l'utiliser pour valider les données de formulaire :

python
from flask import Flask, request, render_template
from wtforms import Form, StringField, PasswordField, validators

app = Flask(__name__)

class MyForm(Form):
    username = StringField('Nom d\'utilisateur', validators=[validators.InputRequired()])
    password = PasswordField('Mot de passe', validators=[validators.InputRequired()])

@app.route('/submit', methods=['GET', 'POST'])
def submit_form():
    form = MyForm(request.form)

    if request.method == 'POST' and form.validate():
        # Traitement des données soumises
        return "Formulaire soumis avec succès"
    
    return render_template('form.html', form=form)

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous utilisons WTForms pour définir un formulaire avec des champs de nom d'utilisateur et de mot de passe, en spécifiant les validateurs requis.

Protection CSRF (Cross-Site Request Forgery)

La CSRF est une attaque qui exploite l'authentification d'un utilisateur sur un site web pour effectuer des actions non autorisées sur un autre site où l'utilisateur est également authentifié. Flask propose une protection CSRF intégrée pour prévenir ce type d'attaque.

Utilisation de Tokens CSRF dans les Formulaires

La protection CSRF dans Flask est basée sur l'utilisation de tokens CSRF dans les formulaires. Un token CSRF est un jeton unique généré côté serveur qui est inclus dans les formulaires HTML et qui doit être renvoyé avec chaque requête pour valider son authenticité.

Voici comment vous pouvez utiliser les tokens CSRF dans les formulaires Flask :

python
from flask import Flask, render_template_string, request
from flask_wtf.csrf import CSRFProtect

app = Flask(__name__)
csrf = CSRFProtect(app)

@app.route('/form', methods=['GET', 'POST'])
def form():
    if request.method == 'POST':
        # Traitement du formulaire
        return "Formulaire soumis avec succès"
    return render_template_string('''
        <form method="post">
            <input type="text" name="username" placeholder="Nom d'utilisateur">
            <input type="password" name="password" placeholder="Mot de passe">
            <input type="hidden" name="csrf_token" value="{{ csrf_token() }}">
            <button type="submit">Soumettre</button>
        </form>
    ''')

if __name__ == '__main__':
    app.run(debug=True)

Dans cet exemple, nous utilisons csrf_token() dans le modèle Jinja2 pour inclure un token CSRF caché dans le formulaire.

Configuration de la Protection CSRF dans Flask

La protection CSRF est activée par défaut dans Flask, mais vous pouvez la personnaliser selon vos besoins. Vous pouvez définir des options CSRF globales ou spécifiques à une vue.

Voici comment vous pouvez configurer la protection CSRF dans Flask :

python
from flask import Flask
from flask_wtf.csrf import CSRFProtect

app = Flask(__name__)
app.config['WTF_CSRF_ENABLED'] = True
app.config['WTF_CSRF_SECRET_KEY'] = 'secret_key'
csrf = CSRFProtect(app)

Dans cet exemple, nous activons la protection CSRF et définissons une clé secrète pour signer les tokens CSRF.

Gestion des Attaques CSRF dans Flask

Flask gère automatiquement les attaques CSRF en vérifiant la validité des tokens CSRF dans chaque requête. Si une requête est soumise sans un token CSRF valide, Flask génère une erreur 400 Bad Request.
