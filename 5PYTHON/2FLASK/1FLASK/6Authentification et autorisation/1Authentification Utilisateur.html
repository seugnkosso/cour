class LoginForm(FlaskForm):
    username = StringField('Nom d\'utilisateur', validators=[DataRequired(), Length(min=4, max=20)])

    submit = SubmitField('Se connecter')

Étape 1 : Introduction au hachage de mot de passe

Le hachage de mot de passe est une technique utilisée pour stocker les mots de passe de manière sécurisée en transformant le mot de passe en une chaîne de caractères aléatoire, rendant ainsi difficile la récupération du mot de passe d'origine à partir du hachage. Cela garantit que même si la base de données est compromise, les mots de passe des utilisateurs restent sécurisés.

Étape 2 : Utilisation de bibliothèques de hachage sécurisé comme Werkzeug

Flask utilise souvent la bibliothèque Werkzeug pour les tâches de sécurité, y compris le hachage de mot de passe. Voici comment l'utiliser dans votre application Flask :

python

# Générer un hachage de mot de passe

# Vérifier un mot de passe haché
check_password_hash(hashed_password, 'mot_de_passe')  # Renvoie True si le mot de passe correspond

Étape 1 : Comprendre le concept de session dans Flask

Dans Flask, une session est un moyen de stocker des informations utilisateur sur le serveur pendant plusieurs requêtes. Flask utilise un cookie de session pour stocker un identifiant de session sur le côté client, tandis que les données de session sont stockées sur le serveur.

Étape 2 : Utilisation de sessions pour gérer l'état d'authentification de l'utilisateur

Vous pouvez utiliser les sessions pour gérer l'état d'authentification de l'utilisateur en stockant son identifiant ou d'autres informations pertinentes dans la session. Voici un exemple :

python
from flask import Flask, session, redirect, url_for

app = Flask(__name__)
app.secret_key = 'your_secret_key'  # Clé secrète pour sécuriser les sessions

@app.route('/login')
def login():
    # Vérification des informations d'identification de l'utilisateur
    user_authenticated = True  # Mettez ici votre logique d'authentification
    if user_authenticated:
        session['user_id'] = 'user123'  # Stocker l'identifiant de l'utilisateur dans la session
        return redirect(url_for('dashboard'))
    else:
        return 'Authentification échouée'

@app.route('/dashboard')
def dashboard():
    if 'user_id' in session:
        # Récupérer l'identifiant de l'utilisateur à partir de la session
        user_id = session['user_id']
        return f'Bienvenue, utilisateur {user_id}'
    else:
        return redirect(url_for('login'))

@app.route('/logout')
def logout():
    session.pop('user_id', None)  # Supprimer l'identifiant de l'utilisateur de la session
    return redirect(url_for('login'))

if __name__ == '__main__':
    app.run(debug=True)

Étape 3 : Création de vues de connexion et de déconnexion pour gérer les sessions utilisateur

Dans l'exemple ci-dessus, les routes /login et /logout sont utilisées pour gérer les sessions utilisateur. La route /login authentifie l'utilisateur et stocke son identifiant dans la session, tandis que la route /logout supprime l'identifiant de l'utilisateur de la session.

Étape 4 : Protection des routes sensibles avec des décorateurs d'authentification personnalisés

Vous pouvez protéger les routes sensibles en utilisant des décorateurs personnalisés pour vérifier si l'utilisateur est authentifié. Voici un exemple de décorateur d'authentification :

python
from functools import wraps
from flask import session, redirect, url_for

def login_required(f):
    @wraps(f)
    def decorated_function(*args, **kwargs):
        if 'user_id' not in session:
            return redirect(url_for('login'))
        return f(*args, **kwargs)
    return decorated_function

@app.route('/profile')
@login_required
def profile():
    return 'Profil de l\'utilisateur'

Dans cet exemple, la route /profile est protégée par le décorateur login_required, ce qui signifie que l'utilisateur doit être authentifié pour accéder à cette route. Si l'utilisateur n'est pas authentifié, il est redirigé vers la page de connexion.

Avec ces étapes, vous avez appris à utiliser les sessions pour gérer l'authentification de l'utilisateur dans Flask, à créer des vues de connexion et de déconnexion, ainsi qu'à protéger les routes sensibles avec des décorateurs d'authentification personnalisés.

1. Récupération de mot de passe avec réinitialisation par e-mail

Pour permettre à un utilisateur de réinitialiser son mot de passe, vous pouvez suivre les étapes suivantes :

  • Créer une page où l'utilisateur peut entrer son adresse e-mail pour demander une réinitialisation de mot de passe.
  • Générer un jeton unique et l'associer à l'utilisateur dans la base de données.
  • Envoyer un e-mail à l'utilisateur avec un lien contenant ce jeton.
  • Lorsque l'utilisateur clique sur le lien, vérifiez le jeton et redirigez-le vers une page où il peut saisir un nouveau mot de passe.

2. Implémentation de la vérification en deux étapes pour une sécurité accrue

Pour ajouter une vérification en deux étapes, vous pouvez utiliser des méthodes telles que l'authentification à deux facteurs (2FA) ou l'envoi de codes de vérification par SMS ou e-mail. Voici un exemple d'implémentation avec l'authentification à deux facteurs :

  • Lorsque l'utilisateur se connecte, demandez son nom d'utilisateur et son mot de passe comme d'habitude.
  • Ensuite, demandez un deuxième facteur d'authentification, tel qu'un code généré par une application d'authentification comme Google Authenticator ou un code envoyé par SMS.
  • Vérifiez ce deuxième facteur avant de permettre à l'utilisateur d'accéder à son compte.

3. Intégration de fonctionnalités d'autorisation pour définir des rôles et des permissions utilisateur

Pour intégrer des fonctionnalités d'autorisation, vous pouvez suivre ces étapes :

  • Créez des rôles d'utilisateur (par exemple, administrateur, utilisateur standard, etc.) dans votre base de données.
  • Associez ces rôles à des permissions spécifiques (par exemple, accès à certaines pages, création/modification/suppression de contenu, etc.).
  • Lorsqu'un utilisateur se connecte, récupérez son rôle à partir de la base de données.
  • Utilisez ce rôle pour déterminer quelles fonctionnalités l'utilisateur est autorisé à accéder dans l'application.
