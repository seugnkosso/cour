Voici un tutoriel complet et approfondi sur la manière de passer des données depuis Flask vers les templates HTML :

a. Utilisation de render_template() pour envoyer des données

La fonction render_template() est utilisée pour rendre un modèle HTML avec des données envoyées depuis Flask vers le template.

python
from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def index():
    data = {'name': 'John', 'age': 30}
    return render_template('index.html', data=data)

if __name__ == '__main__':
    app.run(debug=True)

b. Passage de variables simples

Vous pouvez passer des variables simples directement à render_template().

python
@app.route('/')
def index():
    name = 'John'
    return render_template('index.html', name=name)

Dans le template index.html, vous pouvez accéder à la variable name :

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Flask Template</title>
</head>
<body>
    <h1>Hello, {{ name }}</h1>
</body>
</html>

c. Passage de listes et de dictionnaires

Vous pouvez également passer des listes et des dictionnaires.

python
@app.route('/')
def index():
    my_list = [1, 2, 3, 4, 5]
    my_dict = {'a': 1, 'b': 2, 'c': 3}
    return render_template('index.html', my_list=my_list, my_dict=my_dict)

Dans le template, vous pouvez itérer sur la liste et le dictionnaire :

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Flask Template</title>
</head>
<body>
    <ul>
    {% for item in my_list %}
        <li>{{ item }}</li>
    {% endfor %}
    </ul>
    
    <ul>
    {% for key, value in my_dict.items() %}
        <li>{{ key }}: {{ value }}</li>
    {% endfor %}
    </ul>
</body>
</html>

d. Utilisation de structures de contrôle pour manipuler les données dans les templates

Vous pouvez utiliser des structures de contrôle comme if, for, etc., dans les templates.

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Flask Template</title>
</head>
<body>
    {% if name %}
        <h1>Hello, {{ name }}</h1>
    {% else %}
        <h1>Hello, Stranger!</h1>
    {% endif %}
    
    <ul>
    {% for item in my_list %}
        <li>{{ item }}</li>
    {% endfor %}
    </ul>
    
    <ul>
    {% for key, value in my_dict.items() %}
        <li>{{ key }}: {{ value }}</li>
    {% endfor %}
    </ul>
</body>
</html>

Cela devrait vous donner un bon départ pour utiliser Flask avec des templates et passer des données de manière flexible.

Voici un tutoriel complet et approfondi sur la manière d'accéder aux variables dans les templates avec Flask :

a. Syntaxe de Jinja pour accéder aux variables

Jinja est le moteur de template utilisé par Flask. Voici quelques syntaxes clés :

  • {{ variable }} : Utilisé pour afficher une variable.
  • {% ... %} : Utilisé pour les instructions Jinja comme les boucles et les conditions.
  • {# ... #} : Utilisé pour les commentaires dans le template.

b. Afficher des variables simples dans les templates

Dans votre template HTML, vous pouvez simplement accéder aux variables envoyées depuis Flask :

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Flask Template</title>
</head>
<body>
    <h1>Hello, {{ name }}</h1>
</body>
</html>

c. Utilisation de boucles pour itérer sur les données passées depuis Flask

d. Utilisation de structures conditionnelles pour afficher des données dynamiquement

html
<!DOCTYPE html>

e. Filtrage des données dans les templates

Jinja fournit des filtres pour modifier les données affichées :

html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Flask Template</title>
</head>
<body>
    <p>{{ text|capitalize }}</p>
    <p>{{ number|format }}</p>
</body>
</html>

Dans ce cas, capitalize capitalise la première lettre de la chaîne de caractères, et format formate le nombre avec des milliers séparés par des virgules.

Cela devrait vous aider à manipuler efficacement les variables dans les templates avec Flask.

Voici un tutoriel complet et approfondi sur l'utilisation avancée des variables dans les templates avec Flask :

a. Utilisation de macros pour réutiliser du code HTML

Les macros sont des blocs de code réutilisables dans les templates. Vous pouvez les définir dans un fichier séparé ou directement dans le template.

html
{% macro hello(name) %}
    <h1>Hello, {{ name }}</h1>
{% endmacro %}

Pour utiliser une macro dans un template :

html
{{ hello('John') }}

b. Passage de données depuis les formulaires HTML vers Flask

Lorsque vous soumettez un formulaire HTML, les données peuvent être envoyées à Flask via la méthode POST. Vous pouvez accéder à ces données dans votre route Flask via request.form.

python
from flask import Flask, render_template, request

app = Flask(__name__)

@app.route('/submit', methods=['POST'])
def submit():
    name = request.form['name']
    return render_template('result.html', name=name)

if __name__ == '__main__':
    app.run(debug=True)

c. Utilisation de variables globales dans les templates

Vous pouvez définir des variables globales dans vos vues Flask et y accéder dans vos templates.

python
from flask import Flask, render_template

app = Flask(__name__)

@app.context_processor
def inject_global_variables():
    return dict(global_var='Global Value')

@app.route('/')
def index():
    return render_template('index.html')

if __name__ == '__main__':
    app.run(debug=True)

Dans votre template :

html
<p>{{ global_var }}</p>

d. Techniques pour sécuriser l'utilisation des variables dans les templates

  • Échappement automatique des variables: Jinja échappe automatiquement les variables pour empêcher les attaques XSS (Cross-Site Scripting).
  • Filtrage des données utilisateur: Avant de passer les données utilisateur aux templates, il est recommandé de filtrer et valider les données pour éviter les attaques.
  • Utilisation de Jinja Safe: Vous pouvez marquer une chaîne comme sûre à afficher en utilisant |safe, mais faites attention à ne le faire qu'avec des données de confiance.
  • Privilégier la séparation des responsabilités: Évitez de placer une logique métier complexe dans les templates, gardez-les aussi simples que possible.

En suivant ces bonnes pratiques, vous pouvez garantir que votre application Flask est sécurisée et fonctionne efficacement avec les variables dans les templates.
