Modèles¶

You’ve a écrit les vues d'authentification pour votre application, mais si you’re exécutant le serveur et essayez d'aller à l'une des URL, you’ll see a TemplateNotFound erreur. That’s parce que les vues appellent render_template(), mais vous avez déjà écrit les modèles. Les fichiers de modèle seront stockés dans le templates répertoire à l'intérieur le flaskr paquet.

Les modèles sont des fichiers qui contiennent des données statiques ainsi que des espaces réservés pour données dynamiques. Un modèle est rendu avec des données spécifiques pour produire un document final. Flask utilise le Jinja bibliothèque de modèles à rendre modèles.

Dans votre application, vous utiliserez des modèles pour rendre HTML qui s'affichera dans le navigateur user’s. Dans Flask, Jinja est configuré pour autoescape toutes les données qui sont rendues dans des modèles HTML. Cela signifie que it’s safe to render user input; tous les caractères qu'ils’ve ont entrés pourrait jouer avec le HTML, comme < et > sera échappé avec sûr les valeurs qui se ressemblent dans le navigateur mais ne’t provoquent des indésirables effets.

Jinja ressemble et se comporte principalement comme Python. Des délimiteurs spéciaux sont utilisés pour distinguer la syntaxe Jinja des données statiques dans le modèle. Tout entre {{ et }} est une expression qui sera sortie au document final. {% et %} désigne un flux de contrôle déclaration comme if et for. Contrairement à Python, les blocs sont notés par balises de début et de fin plutôt que par indentation depuis le texte statique à l'intérieur un bloc peut changer l'indentation.

La Mise en page de Base¶

Chaque page de l'application aura la même disposition de base autour d'un corps différent. Au lieu d'écrire la structure HTML entière dans chacun modèle, chaque modèle sera étendre un modèle de base et une substitution des sections spécifiques.

flaskr/templates/base.html¶
<!doctype html>
<title>{% block title %}{% endblock %} - Flaskr</title>
<link rel="stylesheet" href="{{ url_for('static', filename='style.css') }}">
<nav>
  <h1>Flaskr</h1>
  <ul>
    {% if g.user %}
      <li><span>{{ g.user['username'] }}</span>
      <li><a href="{{ url_for('auth.logout') }}">Log Out</a>
    {% else %}
      <li><a href="{{ url_for('auth.register') }}">Register</a>
      <li><a href="{{ url_for('auth.login') }}">Log In</a>
    {% endif %}
  </ul>
</nav>
<section class="content">
  <header>
    {% block header %}{% endblock %}
  </header>
  {% for message in get_flashed_messages() %}
    <div class="flash">{{ message }}</div>
  {% endfor %}
  {% block content %}{% endblock %}
</section>

g est automatiquement disponible dans les modèles. Basé sur if g.user est réglé (à partir de load_logged_in_user), soit le nom d'utilisateur et un lien de déconnexion s'affiche, ou des liens pour s'inscrire et se connecter sont affichés. url_for() est également disponible automatiquement et est utilisé pour générer des URL vers des vues au lieu de les écrire manuellement.

Après le titre de la page, et avant le contenu, le modèle boucle chaque message est retourné par get_flashed_messages(). Vous avez utilisé flash() dans les vues pour afficher les messages d'erreur, et c'est le code cela les affichera.

Il y a trois blocs définis ici qui seront remplacés dans l'autre modèles:

  1. {% block title %}modifiera le titre affiché dans le titre de l'onglet et de la fenêtre de Navigator’s.

  2. {% block header %} est similaire à title mais cela va changer le titre affiché sur la page.

  3. {% block content %}est où le contenu de chaque page va, tel comme le formulaire de connexion ou un article de blog.

Le modèle de base est directement dans le templates répertoire. Pour garder les autres organisés, les modèles pour un plan seront placés dans un répertoire portant le même nom que le plan.

S'inscrire¶

flaskr/templates/auth/register.html¶
{% extends 'base.html' %}

{% block header %}
  <h1>{% block title %}Register{% endblock %}</h1>
{% endblock %}

{% block content %}
  <form method="post">
    <label for="username">Username</label>
    <input name="username" id="username" required>
    <label for="password">Password</label>
    <input type="password" name="password" id="password" required>
    <input type="submit" value="Register">
  </form>
{% endblock %}

{% extends 'base.html' %} dit à Jinja que ce modèle devrait remplacez les blocs du modèle de base. Tout le contenu rendu doit apparaître à l'intérieur les balises qui remplacent les blocs de la base modèle.{% block %}

Un modèle utile utilisé ici est de placer intérieur . Cela définira le bloc de titre, puis la sortie la valeur de celui-ci dans le bloc d'en-tête, de sorte que la fenêtre et la page partagez le même titre sans l'écrire deux fois.{% block title %}{% block header %}

Le input les tags utilisent le required attribuer ici. Cela dit le navigateur ne doit pas soumettre le formulaire tant que ces champs ne sont pas remplis. Si l'utilisateur utilise un navigateur plus ancien qui ne supporte pas cet attribut, ou s'ils utilisent autre chose qu'un navigateur pour faire des demandes, vous vous souhaitez toujours valider les données dans la vue Flask. It’s important to toujours valider complètement les données sur le serveur, même si le client le fait une validation aussi.

Se connecter¶

Ceci est identique au modèle de registre à l'exception du titre et bouton d'envoi.

flaskr/templates/auth/login.html¶
{% extends 'base.html' %}

{% block header %}
  <h1>{% block title %}Log In{% endblock %}</h1>
{% endblock %}

{% block content %}
  <form method="post">
    <label for="username">Username</label>
    <input name="username" id="username" required>
    <label for="password">Password</label>
    <input type="password" name="password" id="password" required>
    <input type="submit" value="Log In">
  </form>
{% endblock %}

Enregistrer un Utilisateur¶

Maintenant que les modèles d'authentification sont écrits, vous pouvez enregistrer un utilisateur. Assurez-vous que le serveur est toujours en cours d'exécution ( si ce n'est pas le cas), puis allez à http://127.0.0.1:5000/auth/register.flask run

Essayez de cliquer sur le bouton “Register” sans remplir le formulaire et voir que le navigateur affiche un message d'erreur. Essayez de supprimer le required attributs de la register.html modèle et cliquez sur “Register” encore. Au lieu que le navigateur affiche une erreur, la page se recharge et l'erreur de flash() dans la vue sera montré.

Remplissez un nom d'utilisateur et mot de passe et vous’sll être redirigé vers le login page. Essayez d'entrer un nom d'utilisateur incorrect, ou le nom d'utilisateur correct et mot de passe incorrect. Si vous vous connectez, vous obtiendrez une erreur car there’s non index voir pour rediriger vers encore.
