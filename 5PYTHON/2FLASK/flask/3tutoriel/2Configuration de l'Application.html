Configuration de l'application¶

Une application Flask est une instance de la Flask classe. Tout sur l'application, comme la configuration et les URL être enregistré auprès de cette classe.

La façon la plus simple de créer une application Flask est de créer un monde Flask instance directement en haut de votre code, comme comment le “ Bonjour, Monde! ” exemple sur la page précédente. Pendant que c'est simple et utile dans certains cas, cela peut causer des problèmes délicats le projet se développe.

Au lieu de créer un Flask instance dans le monde entier, vous allez créer à l'intérieur d'une fonction. Cette fonction est connue sous le nom de application usine. Toute configuration, enregistrement et autre configuration les besoins de l'application se produiront à l'intérieur de la fonction, puis l'application sera retourné.

L' Application Factory¶

Il est temps de commencer à coder! Créer le flaskr répertoire et ajouter le __init__.py fichier. Le __init__.py sert double devoir: il le fera contient l'usine de l'application, et il indique à Python que le flaskr le répertoire doit être traité comme un paquet.

$ mkdir flaskr
flaskr/__init__.py¶
import os

from flask import Flask


def create_app(test_config=None):
    # create and configure the app
    app = Flask(__name__, instance_relative_config=True)
    app.config.from_mapping(
        SECRET_KEY='dev',
        DATABASE=os.path.join(app.instance_path, 'flaskr.sqlite'),
    )

    if test_config is None:
        # load the instance config, if it exists, when not testing
        app.config.from_pyfile('config.py', silent=True)
    else:
        # load the test config if passed in
        app.config.from_mapping(test_config)

    # ensure the instance folder exists
    try:
        os.makedirs(app.instance_path)
    except OSError:
        pass

    # a simple page that says hello
    @app.route('/hello')
    def hello():
        return 'Hello, World!'

    return app

create_appest la fonction d'usine d'application. You’ll ajouter à cela plus tard dans le tutoriel, mais il fait déjà beaucoup.

  1. app = Flask(__name__, instance_relative_config=True) crée le Flask instance.

    • __name__ est le nom du module Python actuel. L' application a besoin de savoir où il’s situé pour mettre en place des chemins, et __name__ c'est un moyen pratique de le dire.

    • instance_relative_config=True indique à l'application que les fichiers de configuration sont relatifs à dossier d'instance. Le dossier instance est situé à l'extérieur du flaskr paquet et peut tenir local les données qui devraient être engagées dans le contrôle de version, telles que secrets de configuration et fichier de base de données.

  2. app.config.from_mapping()ensembles une configuration par défaut que l'application utilisera:

    • SECRET_KEY est utilisé par Flask et les extensions pour conserver les données sûr. It’s réglé sur 'dev' fournir une valeur pratique pendant le développement, mais il devrait être remplacé par un aléatoire valeur lors du déploiement.

    • DATABASE est le chemin où le fichier de base de données SQLite sera sauvé. It’s under app.instance_path, qui est le chemin que Flask a choisi pour le dossier d'instance. You’l apprendre plus d'informations sur la base de données dans la section suivante.

  3. app.config.from_pyfile() remplace la configuration par défaut avec des valeurs prises de la config.py fichier dans le dossier d'instance s'il existe. Par exemple, lorsque déploiement, cela peut être utilisé pour définir un réel SECRET_KEY.

    • test_configpeut également être passé à l'usine, et sera utilisé à la place de la configuration de l'instance. C'est ainsi que les tests you’l écrit plus tard dans le tutoriel peut être configuré indépendamment des valeurs de développement que vous avez configurées.

  4. os.makedirs() assure que app.instance_path existe. Flacon does’t crée automatiquement le dossier instance, mais il doit l'être créé parce que votre projet créera le fichier de base de données SQLite là.

  5. @app.route() crée un itinéraire simple pour que vous puissiez voir l'application fonctionne avant d'entrer dans le reste de la tutoriel. Il crée une connexion entre l'URL /hello et a fonction qui renvoie une réponse, la chaîne in cette affaire.'Hello, World!'

Exécuter l'Application¶

Vous pouvez maintenant exécuter votre application en utilisant le flask commandement. De la terminal, dites à Flask où trouver votre application, puis exécutez-la mode de débogage. Rappelez-vous, vous devriez toujours être au plus haut niveau flask-tutorial répertoire, pas le flaskr paquet.

Le mode Débogage affiche un débogueur interactif chaque fois qu'une page soulève un exception, et redémarre le serveur chaque fois que vous apportez des modifications code. Vous pouvez le laisser en cours d'exécution et simplement recharger la page du navigateur que vous suivez le tutoriel.

$ flask --app flaskr run --debug

You’ll voir la sortie similaire à ceci:

* Serving Flask app "flaskr"
* Debug mode: on
* Running on http://127.0.0.1:5000/ (Press CTRL+C to quit)
* Restarting with stat
* Debugger is active!
* Debugger PIN: nnn-nnn-nnn

Visiter http://127.0.0.1:5000/hello dans un navigateur et vous devriez voir le “Hello, Monde!” message. Félicitations, you’re maintenant en cours d'exécution de votre Flask application web!

Si un autre programme utilise déjà le port 5000, vous verrez ou lorsque le le serveur essaie de démarrer. Voir Adresse déjà utilisée pour comment gère ça.OSError: [Errno 98]OSError: [WinError 10013]
