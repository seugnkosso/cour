Mise en page du Projet¶

Créez un répertoire de projet et entrez-le:

$ mkdir flask-tutorial
$ cd flask-tutorial

Puis suivez le instructions d'installation se mettre installez un environnement virtuel Python et Flask pour votre projet.

Le tutoriel supposera que vous travaillez à partir du flask-tutorial répertoire à partir de maintenant. Les noms de fichiers en haut de chaque bloc de code sont par rapport à ce répertoire.

Une application Flask peut être aussi simple qu'un seul fichier.

hello.py¶
from flask import Flask

app = Flask(__name__)


@app.route('/')
def hello():
    return 'Hello, World!'

Cependant, à mesure qu'un projet s'agrandit, il devient écrasant de tout garder le code dans un fichier. Les projets python utilisent paquets organiser le code dans plusieurs modules qui peuvent être importés si nécessaire, et le ce tutoriel le fera aussi.

Le répertoire du projet contiendra:

  • flaskr/, un paquet Python contenant votre code d'application et fichiers.

  • tests/, un répertoire contenant des modules de test.

  • .venv/, un environnement virtuel Python où Flask et autres des dépendances sont installées.

  • Fichiers d'installation indiquant à Python comment installer votre projet.

  • Configuration de contrôle de version, telle que git. Tu devrais prendre l'habitude de en utilisant un certain type de contrôle de version pour tous vos projets, peu importe la taille.

  • Tous les autres fichiers de projet que vous pourriez ajouter à l'avenir.

À la fin, la disposition de votre projet ressemblera à ceci:

/home/user/Projects/flask-tutorial
├── flaskr/
│   ├── __init__.py
│   ├── db.py
│   ├── schema.sql
│   ├── auth.py
│   ├── blog.py
│   ├── templates/
│   │   ├── base.html
│   │   ├── auth/
│   │   │   ├── login.html
│   │   │   └── register.html
│   │   └── blog/
│   │       ├── create.html
│   │       ├── index.html
│   │       └── update.html
│   └── static/
│       └── style.css
├── tests/
│   ├── conftest.py
│   ├── data.sql
│   ├── test_factory.py
│   ├── test_db.py
│   ├── test_auth.py
│   └── test_blog.py
├── .venv/
├── pyproject.toml
└── MANIFEST.in

Si vous utilisez le contrôle de version, les fichiers suivants sont générés lors de la gestion de votre projet, il faut ignorer. Il peut y avoir d'autres fichiers basé sur l'éditeur que vous utilisez. En général, ignorez les fichiers que vous n'avez pas écrire. Par exemple, avec git:

.gitignore¶
.venv/

*.pyc
__pycache__/

instance/

.pytest_cache/
.coverage
htmlcov/

dist/
build/
*.egg-info/
