Rendre le Projet Installable¶

Rendre votre projet installable signifie que vous pouvez construire un roue enregistrez et installez-le dans un autre environnement, tout comme vous avez installé Flask dans votre environnement project’s. Cela rend le déploiement de votre projet le même que l'installation de toute autre bibliothèque, de sorte que vous utiliser tous les outils Python standard pour tout gérer.

L'installation présente également d'autres avantages qui pourraient ne pas être évidents le tutoriel ou en tant que nouvel utilisateur Python, y compris:

  • Actuellement, Python et Flask comprennent comment utiliser le flaskr package uniquement parce que vous’re s'exécutant à partir de votre répertoire project’s. L'installation signifie que vous pouvez l'importer, peu importe d'où vous venez.

  • Vous pouvez gérer vos dépendances project’s comme les autres packages fais, alors les installe.pip install yourproject.whl

  • Les outils de test peuvent isoler votre environnement de test de votre développement environnement.

Note

Ceci est introduit tard dans le tutoriel, mais dans votre avenir projets que vous devriez toujours commencer par cela.

Décrivez le Projet¶

Le pyproject.toml le fichier décrit votre projet et comment le construire.

pyproject.toml¶
[project]
name = "flaskr"
version = "1.0.0"
description = "The basic blog app built in the Flask tutorial."
dependencies = [
    "flask",
]

[build-system]
requires = ["flit_core<4"]
build-backend = "flit_core.buildapi"

Voir le fonctionnaire Tutoriel d'emballage pour plus explication des fichiers et des options utilisées.

Installer le Projet¶

Utiliser pip pour installer votre projet dans l'environnement virtuel.

$ pip install -e .

Cela dit à pip de trouver pyproject.toml dans le répertoire courant et installez le projet en éditable ou développement mode. Le mode modifiable signifie que lorsque vous faites modifications apportées à votre code local, vous n'aurez besoin de réinstaller que si vous modifiez les métadonnées sur le projet, comme ses dépendances.

Vous pouvez constater que le projet est maintenant installé avec .pip list

$ pip list

Package        Version   Location
-------------- --------- ----------------------------------
click          6.7
Flask          1.0
flaskr         1.0.0     /home/user/Projects/flask-tutorial
itsdangerous   0.24
Jinja2         2.10
MarkupSafe     1.0
pip            9.0.3
Werkzeug       0.14.1

Rien ne change par rapport à la façon dont vous avez dirigé votre projet jusqu'à présent. --app est toujours prêt à flaskr et toujours en cours l'application, mais vous pouvez l'appeler de n'importe où, pas seulement le répertoire.flask runflask-tutorial
