1. Introduction à MongoDB

1.1. Définition de MongoDB

MongoDB est une base de données NoSQL open source, orientée document et conçue pour stocker, interroger et récupérer des données sous la forme de documents JSON. Les documents sont stockés dans des collections, et MongoDB offre une flexibilité élevée en termes de schéma, ce qui signifie que chaque document dans une collection peut avoir une structure différente.

1.2. Principes de base de NoSQL

Les bases de données NoSQL sont caractérisées par leur flexibilité de schéma, leur évolutivité horizontale, et leur capacité à gérer des volumes massifs de données non structurées. Elles sont souvent utilisées dans des applications modernes où les besoins en termes de données sont difficiles à prédire à l'avance.

1.3. Avantages de MongoDB par rapport aux bases de données relationnelles

  • Schéma flexible : Permet de stocker des données de différentes structures dans une même collection.
  • Évolutivité horizontale : Facile à faire évoluer en ajoutant de nouveaux serveurs au lieu d'augmenter la capacité d'un serveur existant.
  • Performances : Les opérations de lecture et d'écriture sont rapides grâce à l'indexation et à la réplication.
  • Support de données complexes : Gestion native de structures de données complexes telles que les tableaux et les documents imbriqués.

1.4. Installation de MongoDB

Pour installer MongoDB, suivez ces étapes (les détails peuvent varier selon votre système d'exploitation) :

  • Windows :

    1. Téléchargez l'installateur MSI depuis le site officiel MongoDB.
    2. Suivez l'assistant d'installation en acceptant les paramètres par défaut.

  • Linux :

    1. Ajoutez la clé GPG et le référentiel MongoDB à votre système.
      bash
      sudo apt-key adv --keyserver hkp://keyserver.ubuntu.com:80 --recv 4B7C549A058F8B6B
    2. Créez un fichier list pour MongoDB.
      bash
      echo "deb [ arch=amd64,arm64 ] http://repo.mongodb.org/apt/ubuntu $(lsb_release -sc)/mongodb-org/4.4 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-4.4.list
    3. Mettez à jour les paquets et installez MongoDB.
      bash
      sudo apt-get update
sudo apt-get install -y mongodb-org

  • macOS :

    1. Utilisez Homebrew pour installer MongoDB.
      bash
      brew tap mongodb/brew
brew install mongodb-community

2. Concepts de base de MongoDB dans le contexte de Spring

2.1. Collections et documents

  • Collections : Dans MongoDB, les données sont stockées dans des collections, qui sont des groupes de documents. Chaque document est une unité de données individuelle, similaire à une ligne dans une table SQL.

    java
    // Exemple de création d'une collection dans MongoDB
MongoCollection<Document> myCollection = mongoDatabase.getCollection("myCollection");

  • Documents : Les documents MongoDB sont au format BSON (Binary JSON) et peuvent contenir des champs et des valeurs de différents types.

    java
    // Exemple de création d'un document dans MongoDB avec Java
Document myDocument = new Document("name", "John Doe")
                        .append("age", 25)
                        .append("city", "New York");

2.2. JSON-like BSON format

MongoDB utilise BSON, une représentation binaire de JSON, pour stocker les données. Les types de données pris en charge incluent des types de base tels que chaînes, entiers, décimaux, tableaux, et des types plus complexes tels que les documents imbriqués.

2.3. Indexation et performances

  • Indexation : MongoDB utilise des index pour améliorer les performances des requêtes. Vous pouvez créer des index sur des champs spécifiques pour accélérer la recherche.

    java
    // Exemple de création d'un index dans MongoDB avec Java
myCollection.createIndex(Indexes.ascending("name"));

  • Réplication : MongoDB prend en charge la réplication pour garantir la disponibilité et la tolérance aux pannes. Les données peuvent être répliquées sur plusieurs serveurs.

2.4. Requêtes MongoDB

MongoDB propose une variété de requêtes pour interroger les données.

  • Requêtes de recherche :

    java
    // Requête pour trouver tous les documents dans une collection
FindIterable<Document> documents = myCollection.find();

  • Requêtes avec critères :

    java
    // Requête avec critères (exemple : âge supérieur à 21)
Bson filter = Filters.gt("age", 21);
FindIterable<Document> filteredDocuments = myCollection.find(filter);

  • Requêtes de mise à jour :

    java
    // Requête de mise à jour (exemple : mise à jour du champ "city" pour l'utilisateur avec le nom "John Doe")
Bson filter = Filters.eq("name", "John Doe");
Bson update = Updates.set("city", "San Francisco");
myCollection.updateOne(filter, update);

  • Requêtes de suppression :

    java
    // Requête de suppression (exemple : supprimer tous les documents avec l'âge inférieur à 18)
Bson filter = Filters.lt("age", 18);
myCollection.deleteMany(filter);

Conclusion

Vous avez maintenant une compréhension approfondie des concepts de base de MongoDB dans le contexte de Spring, y compris les collections et les documents, le format BSON, l'indexation, les performances et les requêtes MongoDB. 3. Configuration de l'environnement de développement avec Spring et MongoDB

3.1. Intégration de Spring dans un projet

Pour intégrer Spring dans votre projet, vous pouvez utiliser Spring Boot, qui simplifie la configuration et offre des conventions par défaut.

  • Maven : Ajoutez la dépendance Spring Boot à votre fichier pom.xml.

    xml
    <dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-data-mongodb</artifactId>
    </dependency>
</dependencies>

  • Gradle : Ajoutez la dépendance Spring Boot à votre fichier build.gradle.

    gradle
    implementation 'org.springframework.boot:spring-boot-starter-data-mongodb'

3.2. Configuration du fichier application.properties/application.yml

Spring Boot utilise des fichiers de configuration pour paramétrer l'application. Dans le cas de MongoDB, vous devez configurer l'URL de connexion et d'autres propriétés.

  • application.properties :

    properties
    spring.data.mongodb.uri=mongodb://localhost:27017/mydatabase

  • application.yml :

    yaml
    spring:
  data:
    mongodb:
      uri: mongodb://localhost:27017/mydatabase

    Assurez-vous de remplacer mydatabase par le nom de votre base de données.

3.3. Dépendances Maven/Gradle pour Spring Data MongoDB

Vous avez intégré Spring dans votre projet, configuré le fichier application.properties/application.yml pour la connexion à MongoDB, et ajouté les dépendances nécessaires pour Spring Data MongoDB. 5. Création de Repositories MongoDB avec Spring Data

5.1. Introduction aux repositories

Les repositories dans Spring Data MongoDB permettent d'effectuer des opérations CRUD sur la base de données sans avoir à écrire beaucoup de code. Ils suivent le modèle de conception Repository et utilisent des conventions pour générer des requêtes.

5.2. Création de repositories simples

Créez un repository pour la classe User que nous avons définie précédemment.

java
import org.springframework.data.mongodb.repository.MongoRepository;

public interface UserRepository extends MongoRepository<User, String> {

    // Aucune méthode n'est nécessaire ici pour les opérations CRUD de base,
    // elles sont fournies par MongoRepository.
}

Grâce à MongoRepository, vous bénéficiez déjà des méthodes de base telles que save, findById, findAll, delete, etc.

5.3. Utilisation de méthodes de requête personnalisées

Spring Data MongoDB permet de définir des méthodes de requête personnalisées en utilisant des conventions de nommage.

java
import java.util.List;

public interface UserRepository extends MongoRepository<User, String> {

    // Méthode de recherche par username
    List<User> findByUsername(String username);

    // Méthode de recherche par email
    User findByEmail(String email);

    // Méthode de recherche par username et email
    User findByUsernameAndEmail(String username, String email);

    // Méthode de recherche par username contenant une chaîne donnée (utilise la convention de nommage)
    List<User> findByUsernameContaining(String partialUsername);
}

Les méthodes de requête personnalisées sont générées automatiquement par Spring Data MongoDB en fonction du nom des méthodes.

Conclusion

Vous avez maintenant créé des repositories MongoDB avec Spring Data. Ces repositories simplifient considérablement les opérations CRUD en utilisant des conventions de nommage et fournissent déjà des méthodes de base. 5.5. Utilisation de Query DSL pour les requêtes complexes

Query DSL (Domain Specific Language) vous permet de créer des requêtes complexes en utilisant des méthodes générées par Spring Data MongoDB.

java
import static org.springframework.data.mongodb.core.query.Criteria.where;
import static org.springframework.data.mongodb.core.query.Query.query;

public class UserService {

    private final MongoTemplate mongoTemplate;

    public UserService(MongoTemplate mongoTemplate) {
        this.mongoTemplate = mongoTemplate;
    }

    public List<User> findUsersWithEmailLike(String partialEmail) {
        Query query = query(where("email").regex(partialEmail, "i"));
        return mongoTemplate.find(query, User.class);
    }
}

Ici, mongoTemplate est injecté via le constructeur, et la méthode findUsersWithEmailLike utilise Query DSL pour effectuer une recherche d'utilisateurs dont l'e-mail contient une chaîne donnée.

6. Mapping objet-document avec Spring Data MongoDB

6.1. Annotations de mapping de base

Utilisez les annotations de Spring Data MongoDB pour mapper les champs de vos classes aux champs dans les documents MongoDB.

java
import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.Document;

@Document(collection = "users")
public class User {

    @Id
    private String id;
    private String username;
    private String email;

    // Getters and setters
}

Dans cet exemple, l'annotation @Document spécifie le nom de la collection, et @Id indique l'identifiant du document.

6.2. Mapping d'objets imbriqués

Si votre document contient des objets imbriqués, utilisez l'annotation @DBRef pour créer une référence.

java
import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.DBRef;
import org.springframework.data.mongodb.core.mapping.Document;

@Document(collection = "orders")
public class Order {

    @Id
    private String id;
    private String orderNumber;

    @DBRef
    private User user;

    // Getters and setters
}

Ici, la propriété user est une référence à un document de la collection "users".

6.3. Gestion des relations entre documents

Utilisez les annotations @DBRef et @DBRef(lazy = true) pour gérer les relations entre documents.

java
import org.springframework.data.annotation.Id;
import org.springframework.data.mongodb.core.mapping.DBRef;
import org.springframework.data.mongodb.core.mapping.Document;

@Document(collection = "comments")
public class Comment {

    @Id
    private String id;
    private String text;

    @DBRef(lazy = true)
    private User user;

    // Getters and setters
}

Ici, la propriété user est une référence à un document de la collection "users". L'utilisation de @DBRef(lazy = true) rend le chargement de la référence optionnel, ce qui peut être utile pour éviter le chargement complet de l'objet lié à moins que ce ne soit nécessaire.

6.4. Utilisation de Converters personnalisés

Vous pouvez utiliser des Converters personnalisés pour personnaliser la conversion entre les types Java et les types BSON.

java
import org.springframework.core.convert.converter.Converter;
import org.springframework.data.convert.ReadingConverter;
import org.springframework.data.convert.WritingConverter;

@ReadingConverter
public class StringToEnumConverter implements Converter<String, UserRole> {

    @Override
    public UserRole convert(String source) {
        return UserRole.valueOf(source.toUpperCase());
    }
}

@WritingConverter
public class EnumToStringConverter implements Converter<UserRole, String> {

    @Override
    public String convert(UserRole source) {
        return source.name().toLowerCase();
    }
}

Dans cet exemple, nous avons créé des convertisseurs personnalisés pour convertir entre une chaîne et un type enum (UserRole dans ce cas).
