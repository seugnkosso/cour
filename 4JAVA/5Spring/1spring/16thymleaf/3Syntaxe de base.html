1. Intégration de Thymeleaf dans des fichiers HTML

Pour intégrer Thymeleaf dans des fichiers HTML, vous devez d'abord inclure les dépendances nécessaires dans votre projet Maven ou Gradle. Ensuite, vous pouvez commencer à utiliser Thymeleaf dans vos fichiers HTML de la manière suivante :

1.1. Configuration de Thymeleaf dans Spring Boot

Si vous utilisez Spring Boot, Thymeleaf est généralement configuré par défaut. Vous pouvez personnaliser la configuration dans le fichier application.properties ou application.yml.

Exemple de configuration YAML :

yaml
spring:
  thymeleaf:
    enabled: true
    prefix: classpath:/templates/
    suffix: .html

1.2. Utilisation de Thymeleaf dans les fichiers HTML

Une fois configuré, vous pouvez commencer à utiliser Thymeleaf dans vos fichiers HTML. Voici un exemple de fichier HTML avec Thymeleaf :

html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Exemple Thymeleaf</title>
</head>
<body>
    <h1 th:text="${pageTitle}">Titre par défaut</h1>
    <p th:text="${message}">Message par défaut</p>
</body>
</html>

Dans cet exemple :

  • xmlns:th="http://www.thymeleaf.org" définit l'espace de noms Thymeleaf.
  • th:text="${pageTitle}" récupère la valeur de la variable pageTitle et l'insère dans la balise où elle est définie.
  • th:text="${message}" fait la même chose avec la variable message.

2. Balises Thymeleaf

Thymeleaf offre diverses balises pour réaliser des opérations telles que l'itération, la condition, l'affichage de données, etc.

2.1. Affichage de données

Vous pouvez utiliser la balise th:text pour afficher des données dans votre modèle.

html
<p th:text="${user.name}">Nom par défaut</p>

2.2. Itération sur une liste

La balise th:each permet d'itérer sur une liste.

html
<ul>
    <li th:each="item : ${items}" th:text="${item}">Élément par défaut</li>
</ul>

Cela itère sur la liste items et affiche chaque élément dans une balise <li>.

Ceci est une introduction de base à Thymeleaf. Pour des informations plus approfondies sur les fonctionnalités et la syntaxe avancée, vous pouvez consulter la documentation officielle de Thymeleaf : Thymeleaf Documentation.

2.3. Expressions Thymeleaf

Les expressions Thymeleaf permettent d'accéder aux données du modèle dans les fichiers HTML et d'effectuer des opérations dynamiques.

2.3.1. Types d'expressions Thymeleaf

Thymeleaf prend en charge différents types d'expressions :

  • Variables : Accès aux données du modèle.
  • Sélection : Sélection d'une partie d'une variable (par exemple, une propriété d'un objet).
  • Opérations arithmétiques : Addition, soustraction, multiplication, etc.
  • Opérations logiques : AND, OR, NOT, etc.
  • Opérations de comparaison : Égalité, supériorité, infériorité, etc.
  • Opérations de collection : Itération sur les listes, les ensembles, les cartes, etc.

2.3.2. Utilisation des expressions dans les modèles HTML

Les expressions Thymeleaf sont utilisées dans les fichiers HTML pour accéder aux données du modèle. Voici quelques exemples :

  • Afficher une variable :

    html
    <p th:text="${user.name}">Nom par défaut</p>

  • Sélection d'une propriété d'un objet :

    html
    <p th:text="${user.address.city}">Ville par défaut</p>

2.3.3. Expressions conditionnelles

Vous pouvez utiliser des expressions conditionnelles pour afficher des éléments en fonction d'une condition :

html
<div th:if="${user.isAdmin}">
    <p>Bienvenue, administrateur!</p>
</div>

2.3.4. Expressions itératives

Les expressions itératives vous permettent d'itérer sur une liste et d'afficher les éléments :

html
<ul>
    <li th:each="item : ${items}" th:text="${item}">Élément par défaut</li>
</ul>

2.3.5. Expressions de vérification de la présence de valeurs

Thymeleaf fournit des moyens de vérifier si une valeur est présente ou non :

  • Vérification de la présence d'une valeur :
    html
    <div th:if="${user != null}">
    <p>Bienvenue, <span th:text="${user.name}">Utilisateur</span>!</p>
</div>

Cela affichera le message de bienvenue uniquement si l'utilisateur n'est pas nul.

Ceci conclut la partie sur les expressions Thymeleaf. Elles offrent un moyen puissant d'accéder et de manipuler les données du modèle dans vos modèles HTML. Pour plus d'informations, vous pouvez consulter la documentation officielle de Thymeleaf : Thymeleaf Documentation.

2.4. Utilisation de fragments

Les fragments dans Thymeleaf permettent de réutiliser des morceaux de code HTML dans plusieurs modèles.

2.4.1. Fragmentisation des modèles HTML

Pour fragmenter un modèle HTML, vous pouvez définir des sections réutilisables à l'aide de balises spéciales.

Exemple de fragment dans un fichier header.html :

html
<header th:fragment="pageHeader">
    <h1>Bienvenue sur notre site</h1>
    <nav>
        <ul>
            <li><a href="/">Accueil</a></li>
            <li><a href="/about">À propos</a></li>
            <li><a href="/contact">Contact</a></li>
        </ul>
    </nav>
</header>

Dans cet exemple, th:fragment="pageHeader" définit un fragment nommé "pageHeader" qui contient un en-tête de page.

2.4.2. Inclusion de fragments dans d'autres modèles

Une fois que vous avez défini un fragment, vous pouvez l'inclure dans d'autres modèles à l'aide de la balise th:include.

Exemple d'inclusion du fragment header.html dans un fichier home.html :

html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Accueil</title>
</head>
<body>
    <div th:include="header :: pageHeader"></div>
    <h2>Contenu de la page d'accueil</h2>
    <!-- Autres éléments de la page -->
</body>
</html>

Dans cet exemple, th:include="header :: pageHeader" inclut le fragment pageHeader du fichier header.html dans le modèle home.html.

2.4.3. Passer des données aux fragments

Vous pouvez également passer des données aux fragments pour les rendre plus dynamiques.

Exemple de passage de données à un fragment :

html
<header th:fragment="pageHeader(title)">
    <h1 th:text="${title}">Titre par défaut</h1>
    <!-- Autres éléments du header -->
</header>

Vous pouvez ensuite inclure ce fragment en passant une valeur pour le titre :

html
<div th:include="header :: pageHeader('Bienvenue sur notre site')"></div>

Ceci conclut la partie sur l'utilisation de fragments dans Thymeleaf. Les fragments offrent un moyen puissant de réutiliser du code HTML dans vos modèles, ce qui facilite la maintenance et la réutilisation du code. Pour plus d'informations, vous pouvez consulter la documentation officielle de Thymeleaf : Thymeleaf Documentation.
