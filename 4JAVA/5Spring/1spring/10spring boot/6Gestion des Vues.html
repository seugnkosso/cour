2. Configuration du bean ThymeleafTemplateResolver

Dans votre classe de configuration Java, configurez le bean ThymeleafTemplateResolver. Par exemple, si vous utilisez une classe de configuration Java, elle pourrait ressembler à ceci :

java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.thymeleaf.templateresolver.ITemplateResolver;
import org.thymeleaf.templateresolver.UrlTemplateResolver;

@Configuration
public class ThymeleafConfig {

    @Bean
    public ITemplateResolver templateResolver() {
        UrlTemplateResolver templateResolver = new UrlTemplateResolver();
        templateResolver.setPrefix("/templates/"); // Le répertoire où se trouvent vos templates
        templateResolver.setSuffix(".html"); // L'extension de vos templates
        templateResolver.setTemplateMode("HTML5");
        templateResolver.setCharacterEncoding("UTF-8");
        return templateResolver;
    }
}

3. Configuration du bean ThymeleafViewResolver

Configurer le bean ThymeleafViewResolver pour permettre à Spring de résoudre les vues Thymeleaf :

java
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.springframework.web.servlet.ViewResolver;
import org.thymeleaf.spring5.SpringTemplateEngine;
import org.thymeleaf.spring5.view.ThymeleafViewResolver;
import org.thymeleaf.templateresolver.ITemplateResolver;

@Configuration
public class ThymeleafConfig {

    // ... (bean ThymeleafTemplateResolver)

    @Bean
    public SpringTemplateEngine templateEngine(ITemplateResolver templateResolver) {
        SpringTemplateEngine templateEngine = new SpringTemplateEngine();
        templateEngine.setTemplateResolver(templateResolver);
        return templateEngine;
    }

    @Bean
    public ViewResolver thymeleafViewResolver(SpringTemplateEngine templateEngine) {
        ThymeleafViewResolver viewResolver = new ThymeleafViewResolver();
        viewResolver.setTemplateEngine(templateEngine);
        viewResolver.setCharacterEncoding("UTF-8");
        return viewResolver;
    }
}

Avec ces configurations en place, vous devriez être prêt à utiliser Thymeleaf dans votre application Spring. Assurez-vous que vos fichiers de modèle Thymeleaf sont placés dans le répertoire spécifié (dans cet exemple, "/templates/").

3.1 Syntaxe de base de Thymeleaf

Thymeleaf utilise une syntaxe spécifique dans les fichiers HTML pour permettre l'intégration de données dynamiques. Voici quelques éléments de base de la syntaxe Thymeleaf :

3.1.1 Affichage de variables

html
<p>Salut, [[${nom}]]!</p>

Dans cet exemple, la variable nom sera remplacée par sa valeur lors de l'exécution de la page.

3.1.2 Conditions

html
<div th:if="${condition}">
    <p>Cette partie sera affichée si la condition est vraie.</p>
</div>
<div th:unless="${condition}">
    <p>Cette partie sera affichée si la condition est fausse.</p>
</div>

3.1.3 Boucles

html
<ul>
    <li th:each="item : ${items}">
        <p th:text="${item}"></p>
    </li>
</ul>

Dans cet exemple, pour chaque élément de la liste items, un élément de liste est généré.

3.2 Intégration de Thymeleaf avec les modèles HTML

Créez un fichier HTML dans le répertoire spécifié dans la configuration (/templates/ dans notre exemple) et utilisez la syntaxe Thymeleaf dans ce fichier. Voici un exemple de modèle HTML avec Thymeleaf :

html
<!DOCTYPE html>
<html xmlns:th="http://www.thymeleaf.org">
<head>
    <title>Ma Page Thymeleaf</title>
</head>
<body>

    <h1 th:text="${pageTitle}">Titre par défaut</h1>

    <ul>
        <li th:each="item : ${items}" th:text="${item}"></li>
    </ul>

    <div th:if="${showMessage}">
        <p>Ce message sera affiché si showMessage est vrai.</p>
    </div>

</body>
</html>

Dans ce fichier, les expressions Thymeleaf (th:text, th:each, th:if) sont utilisées pour incorporer des données dynamiques dans le modèle. Vous pouvez passer ces données depuis votre contrôleur Spring lors du rendu de la page.

Conclusion

Avec ces bases, vous devriez être en mesure de créer des pages web dynamiques en utilisant Thymeleaf dans une application Spring. N'oubliez pas de connecter votre contrôleur Spring pour fournir les données nécessaires à vos modèles Thymeleaf.

En utilisant th:replace, vous pouvez inclure le fragment défini dans une autre page. Cela permet une gestion efficace des éléments réutilisables.

4.2 Comparaison entre Thymeleaf, FreeMarker et Velocity

4.2.1 Thymeleaf

  • Avantages:
    • Intégration transparente avec Spring.
    • Syntaxe naturelle et facile à apprendre.
    • Prise en charge de la validation HTML.
  • Inconvénients:
    • Peut être plus verbeux pour certaines tâches.

4.2.2 FreeMarker

  • Avantages:
    • Syntaxe simple et concise.
    • Gestion des erreurs plus stricte que Velocity.
    • Bonne performance.
  • Inconvénients:
    • Configuration initiale peut être plus détaillée.

4.2.3 Velocity

  • Avantages:
    • Facile à apprendre.
    • Configuration minimale.
    • Gestion des erreurs plus souple.
  • Inconvénients:
    • Moins performant que FreeMarker.
    • Moins de fonctionnalités intégrées.

4.3 Choix du moteur de template en fonction des besoins du projet

Le choix entre Thymeleaf, FreeMarker et Velocity dépend des besoins spécifiques du projet :

  • Thymeleaf est recommandé pour les projets Spring en raison de son intégration naturelle et de sa validation HTML intégrée.

  • FreeMarker est un bon choix si vous préférez une syntaxe plus concise et une gestion stricte des erreurs.

  • Velocity est approprié pour des projets simples avec une configuration minimale, mais peut être moins performant que FreeMarker.

Considérez la simplicité, la syntaxe, la performance et l'intégration avec Spring lors du choix du moteur de template en fonction des exigences spécifiques de votre projet.

5.1 Introduction aux contrôleurs Spring

Les contrôleurs dans Spring sont des composants responsables de la gestion des requêtes HTTP. Ils sont chargés de traiter les requêtes des utilisateurs et de retourner une réponse appropriée. Voici comment créer un contrôleur de base en utilisant l'annotation @Controller :

java
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;

@Controller
public class MonController {

    @GetMapping("/accueil")
    public String afficherPageAccueil() {
        return "accueil";
    }
}

Dans cet exemple, la classe MonController est annotée avec @Controller, ce qui indique à Spring qu'il s'agit d'un composant de gestion des requêtes. La méthode afficherPageAccueil est annotée avec @GetMapping("/accueil"), indiquant que cette méthode doit être appelée lorsqu'une requête GET est effectuée sur le chemin "/accueil". La méthode retourne le nom de la vue (sans l'extension) qui doit être rendue.

5.2 Création de méthodes de contrôleur pour gérer les différentes actions

5.2.1 Gestion des paramètres de requête

java
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;

@Controller
public class MonController {

    @GetMapping("/salutation")
    public String saluer(@RequestParam String nom) {
        return "salutation";
    }
}

Dans cet exemple, la méthode saluer utilise @RequestParam pour récupérer le paramètre "nom" de la requête. Par exemple, une requête vers "/salutation?nom=John" appellerait la méthode saluer avec le paramètre "nom" égal à "John".

5.2.2 Utilisation de chemins variables

java
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.PathVariable;

@Controller
public class MonController {

    @GetMapping("/utilisateur/{id}")
    public String afficherProfilUtilisateur(@PathVariable Long id) {
        // Logique pour afficher le profil de l'utilisateur avec l'ID spécifié
        return "profil";
    }
}

Dans cet exemple, la méthode afficherProfilUtilisateur utilise @PathVariable pour récupérer la valeur de la variable de chemin "{id}" de la requête. Par exemple, une requête vers "/utilisateur/123" appellerait la méthode avec l'ID égal à 123.

5.2.3 Retour de données JSON

java
import org.springframework.stereotype.Controller;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.ResponseBody;

@Controller
public class MonController {

    @GetMapping("/api/utilisateur")
    @ResponseBody
    public String obtenirDonneesUtilisateur() {
        // Logique pour récupérer les données de l'utilisateur (peut être un objet complexe)
        return "{\"nom\": \"John\", \"age\": 30}";
    }
}

En annotant la méthode avec @ResponseBody, Spring retourne directement le résultat de la méthode sous forme de données JSON. Vous pouvez également retourner des objets Java, et Spring les convertira automatiquement en JSON.

Avec ces exemples, vous pouvez créer des méthodes de contrôleur pour gérer différentes actions en fonction des besoins de votre application Spring.

5.4.1 Utilisation de @Controller et @RequestMapping

java
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;

@Controller
@RequestMapping("/exemples")

        return "page1";

    public String afficherPage2(Model model) {

        return "page2";
    }

Dans cet exemple, la classe MonController est annotée avec @Controller, indiquant qu'elle est un contrôleur Spring. De plus, la classe est annotée avec @RequestMapping("/exemples"), indiquant que toutes les méthodes de ce contrôleur traiteront les requêtes sous le chemin "/exemples".

5.4.2 Utilisation de @RequestParam

java
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestParam;

@Controller
@RequestMapping("/utilisateur")
public class MonController {

    @GetMapping("/profil")

        model.addAttribute("age", age);
        return "profil";
    }
}

Dans cet exemple, les paramètres de requête sont extraits à l'aide de @RequestParam. Par exemple, une requête vers "/utilisateur/profil?nom=John&age=30" appellerait la méthode avec les valeurs correspondantes.

En utilisant ces annotations, vous pouvez organiser et structurer votre code de manière claire et gérer efficacement les différentes actions de votre application Spring.
