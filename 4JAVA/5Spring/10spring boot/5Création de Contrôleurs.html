3.1 Introduction à @RestController

@RestController est une annotation spécifique à Spring qui combine @Controller et @ResponseBody. Elle est utilisée pour créer des contrôleurs RESTful. Les méthodes de ces contrôleurs retournent directement des données JSON plutôt que des vues.

3.2 Création d'un simple contrôleur REST

Créons un contrôleur REST basique :

java
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/api")
public class MonController {

    @GetMapping("/bonjour")
    public String direBonjour() {
        return "Bonjour, monde!";
    }
}

Dans cet exemple, @RestController indique que la classe est un contrôleur REST, et @RequestMapping définit le chemin de base pour toutes les méthodes du contrôleur.

La méthode direBonjour répond à une requête GET à l'URL "/api/bonjour" en renvoyant la chaîne "Bonjour, monde!".

3.3 Gestion des requêtes et des réponses HTTP

3.3.1 Paramètres de requête

Pour gérer les paramètres de requête, utilisez @RequestParam :

java
@GetMapping("/saluer")
public String saluer(@RequestParam(name = "nom", defaultValue = "Utilisateur") String nom) {
    return "Salut, " + nom + "!";
}

Dans cet exemple, la méthode prend un paramètre de requête "nom" avec une valeur par défaut de "Utilisateur".

3.3.2 Corps de la requête

Pour traiter le corps de la requête, utilisez @RequestBody :

java
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;

@PostMapping("/creer-utilisateur")
public Utilisateur creerUtilisateur(@RequestBody Utilisateur utilisateur) {
    // Logique de création d'utilisateur
    return utilisateur;
}

Assurez-vous d'avoir une classe Utilisateur définie avec les champs nécessaires.

3.3.3 Réponses personnalisées

Pour des réponses personnalisées, utilisez l'objet ResponseEntity :

java
import org.springframework.http.ResponseEntity;

@GetMapping("/info")
public ResponseEntity<String> getInfo() {
    // Logique de récupération d'information
    String info = "Informations importantes";
    return ResponseEntity.ok(info);
}

Dans cet exemple, la méthode renvoie une réponse HTTP 200 (OK) avec le contenu spécifié.

C'est une introduction de base à la création de contrôleurs REST avec Spring. Assurez-vous d'avoir ajouté les dépendances nécessaires dans votre fichier de configuration Maven ou Gradle pour Spring Web. Vous pouvez également explorer davantage les fonctionnalités de Spring MVC en consultant la documentation officielle de Spring.

4.1 Annotation @RequestMapping

L'annotation @RequestMapping est une annotation polyvalente de Spring qui est utilisée pour mapper les méthodes de contrôleurs aux requêtes HTTP. Elle peut être appliquée au niveau de la classe ou de la méthode, et permet de spécifier diverses options pour le mappage des URL, des méthodes HTTP, des en-têtes, etc.

4.1.1 Mappage au niveau de la classe

Au niveau de la classe, @RequestMapping peut être utilisée pour définir un préfixe commun pour toutes les méthodes de la classe :

java
@RestController
@RequestMapping("/api/employes")
public class EmployeController {

    // Les méthodes à l'intérieur de cette classe auront des URLs qui commencent par "/api/employes"
    
    @GetMapping
    public List<Employe> getAllEmployes() {
        // Logique pour récupérer tous les employés
    }

    @PostMapping
    public Employe createEmploye(@RequestBody Employe employe) {
        // Logique pour créer un nouvel employé
    }

    // Autres méthodes...
}

Ainsi, les méthodes de cette classe seront mappées sous l'URL de base "/api/employes".

4.1.2 Mappage au niveau de la méthode

Au niveau de la méthode, @RequestMapping est utilisée pour spécifier les détails de mappage pour une méthode spécifique :

java
@RestController
@RequestMapping("/api/produits")
public class ProduitController {

    @GetMapping("/tous")
    public List<Produit> getAllProduits() {
        // Logique pour récupérer tous les produits
    }

    @GetMapping("/{id}")
    public Produit getProduitById(@PathVariable Long id) {
        // Logique pour récupérer un produit par ID
    }

    @PostMapping
    public Produit createProduit(@RequestBody Produit produit) {
        // Logique pour créer un nouveau produit
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode getAllProduits sera mappée à l'URL "/api/produits/tous", la méthode getProduitById sera mappée à "/api/produits/{id}", où {id} est une variable de chemin, et la méthode createProduit sera mappée à l'URL "/api/produits" avec la méthode HTTP POST.

4.1.3 Utilisation des méthodes HTTP spécifiques

Vous pouvez spécifier les méthodes HTTP autorisées pour une méthode particulière à l'aide de l'attribut method de @RequestMapping :

java
@RestController
@RequestMapping("/api/commandes")
public class CommandeController {

    @RequestMapping(value = "/{id}", method = RequestMethod.GET)
    public Commande getCommandeById(@PathVariable Long id) {
        // Logique pour récupérer une commande par ID
    }

    @RequestMapping(value = "/nouvelle", method = RequestMethod.POST)
    public Commande createCommande(@RequestBody Commande commande) {
        // Logique pour créer une nouvelle commande
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode getCommandeById sera mappée uniquement aux requêtes HTTP GET, et la méthode createCommande sera mappée uniquement aux requêtes HTTP POST.

L'utilisation de @RequestMapping offre une grande flexibilité pour mapper vos méthodes de contrôleur aux différentes requêtes HTTP et définir des modèles d'URL. Vous pouvez également explorer d'autres annotations comme @GetMapping, @PostMapping, @PutMapping, et @DeleteMapping, qui sont des versions spécialisées de @RequestMapping pour des opérations HTTP spécifiques.

4.2 Utilisation de méthodes spécifiques (@GetMapping, @PostMapping, @PutMapping, @DeleteMapping)

Spring propose des annotations spécifiques pour les opérations HTTP courantes, offrant une syntaxe plus concise que @RequestMapping. Ces annotations incluent @GetMapping, @PostMapping, @PutMapping, et @DeleteMapping.

4.2.1 @GetMapping

L'annotation @GetMapping est utilisée pour mapper spécifiquement les requêtes HTTP GET à une méthode du contrôleur.

java
@RestController
@RequestMapping("/api/etudiants")
public class EtudiantController {

    @GetMapping("/tous")
    public List<Etudiant> getAllEtudiants() {
        // Logique pour récupérer tous les étudiants
    }

    @GetMapping("/{id}")
    public Etudiant getEtudiantById(@PathVariable Long id) {
        // Logique pour récupérer un étudiant par ID
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode getAllEtudiants sera mappée à l'URL "/api/etudiants/tous" pour les requêtes HTTP GET, et la méthode getEtudiantById sera mappée à l'URL "/api/etudiants/{id}" pour les requêtes HTTP GET.

4.2.2 @PostMapping

L'annotation @PostMapping est utilisée pour mapper spécifiquement les requêtes HTTP POST à une méthode du contrôleur.

java
@RestController
@RequestMapping("/api/enseignants")
public class EnseignantController {

    @PostMapping("/nouveau")
    public Enseignant creerEnseignant(@RequestBody Enseignant enseignant) {
        // Logique pour créer un nouvel enseignant
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode creerEnseignant sera mappée à l'URL "/api/enseignants/nouveau" pour les requêtes HTTP POST.

4.2.3 @PutMapping

L'annotation @PutMapping est utilisée pour mapper spécifiquement les requêtes HTTP PUT à une méthode du contrôleur.

java
@RestController
@RequestMapping("/api/projets")
public class ProjetController {

    @PutMapping("/{id}")
    public Projet mettreAJourProjet(@PathVariable Long id, @RequestBody Projet projet) {
        // Logique pour mettre à jour un projet par ID
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode mettreAJourProjet sera mappée à l'URL "/api/projets/{id}" pour les requêtes HTTP PUT.

4.2.4 @DeleteMapping

L'annotation @DeleteMapping est utilisée pour mapper spécifiquement les requêtes HTTP DELETE à une méthode du contrôleur.

java
@RestController
@RequestMapping("/api/cours")
public class CoursController {

    @DeleteMapping("/{id}")
    public void supprimerCours(@PathVariable Long id) {
        // Logique pour supprimer un cours par ID
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode supprimerCours sera mappée à l'URL "/api/cours/{id}" pour les requêtes HTTP DELETE.

L'utilisation de ces annotations spécifiques facilite la lecture et la compréhension du code en indiquant clairement le type d'opération HTTP attendu pour chaque méthode du contrôleur. Vous pouvez choisir l'annotation appropriée en fonction de l'opération CRUD que vous souhaitez implémenter.

4.3 Chemins d'URL et variables de chemin (@PathVariable)

Les variables de chemin et les annotations telles que @PathVariable permettent de capturer les valeurs dynamiques dans les URL des requêtes.

4.3.1 Utilisation de @PathVariable

La variable de chemin {id} dans l'URL peut être capturée et utilisée dans la méthode du contrôleur à l'aide de l'annotation @PathVariable :

java
@RestController
@RequestMapping("/api/employes")
public class EmployeController {

    @GetMapping("/{id}")
    public Employe getEmployeById(@PathVariable Long id) {
        // Logique pour récupérer un employé par ID
    }

    // Autres méthodes...
}

Dans cet exemple, lorsque la requête est faite à "/api/employes/123", la valeur "123" est automatiquement assignée à la variable id de la méthode getEmployeById.

4.3.2 Plusieurs variables de chemin

Vous pouvez avoir plusieurs variables de chemin dans une seule URL :

java
@RestController
@RequestMapping("/api/articles")
public class ArticleController {

    @GetMapping("/{categorie}/{id}")
    public Article getArticleByCategoryAndId(
        @PathVariable String categorie, 
        @PathVariable Long id) {
        // Logique pour récupérer un article par catégorie et ID
    }

    // Autres méthodes...
}

Dans cet exemple, l'URL "/api/articles/technologie/456" assigne "technologie" à la variable categorie et "456" à la variable id.

4.3.3 Utilisation de valeurs par défaut

Vous pouvez fournir des valeurs par défaut aux variables de chemin en utilisant l'attribut defaultValue de @PathVariable :

java
@RestController
@RequestMapping("/api/clients")
public class ClientController {

    @GetMapping("/{id}")
    public Client getClientById(
        @PathVariable(name = "id", defaultValue = "0") Long id) {
        // Logique pour récupérer un client par ID
    }

    // Autres méthodes...
}

Dans cet exemple, si l'ID n'est pas présent dans l'URL, la valeur par défaut de "0" sera utilisée.

4.3.4 Utilisation de regex pour les variables de chemin

Vous pouvez également utiliser des expressions régulières pour définir des modèles pour les variables de chemin :

java
@RestController
@RequestMapping("/api/produits")
public class ProduitController {

    @GetMapping("/{code:[A-Za-z0-9]+}")
    public Produit getProduitByCode(@PathVariable String code) {
        // Logique pour récupérer un produit par code
    }

    // Autres méthodes...
}

Dans cet exemple, l'URL "/api/produits/ABC123" correspond à la méthode uniquement si code est une séquence alphanumérique.

L'utilisation de @PathVariable offre une flexibilité pour capturer des valeurs dynamiques dans les URLs et les utiliser dans vos méthodes de contrôleur. Cela est particulièrement utile dans les cas où les données spécifiques sont identifiées par des portions variables de l'URL.

5.1 Paramètres de requête (@RequestParam)

La gestion des paramètres de requête est essentielle dans le développement d'applications Spring, car elle permet de traiter les données envoyées avec les requêtes HTTP. L'annotation @RequestParam est utilisée pour extraire les valeurs des paramètres de requête dans une méthode de contrôleur.

5.1.1 Paramètres simples

Pour traiter des paramètres de requête simples, vous pouvez utiliser @RequestParam de la manière suivante :

java
@RestController
@RequestMapping("/api/utilisateurs")
public class UtilisateurController {

    @GetMapping("/rechercher")
    public List<Utilisateur> rechercherUtilisateurs(
            @RequestParam(name = "nom") String nom,
            @RequestParam(name = "age", defaultValue = "0") int age) {
        // Logique pour rechercher les utilisateurs en fonction du nom et de l'âge
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode rechercherUtilisateurs prend deux paramètres de requête : nom et age. La valeur de nom est obligatoire, tandis que la valeur de age a une valeur par défaut de "0" si elle n'est pas fournie dans la requête.

5.1.2 Paramètres optionnels

Si un paramètre de requête est facultatif, vous pouvez utiliser required = false dans @RequestParam :

java
@GetMapping("/chercher")
public List<Utilisateur> chercherUtilisateurs(
        @RequestParam(name = "nom") String nom,
        @RequestParam(name = "ville", required = false) String ville) {
    // Logique pour rechercher les utilisateurs en fonction du nom et éventuellement de la ville
}

Dans cet exemple, le paramètre ville est facultatif. Si la valeur de ville n'est pas fournie dans la requête, elle sera null.

5.1.3 Liste de paramètres

Vous pouvez également traiter plusieurs valeurs d'un même paramètre en utilisant une List ou un tableau :

java
@GetMapping("/filtrer")
public List<Utilisateur> filtrerUtilisateurs(
        @RequestParam(name = "roles") List<String> roles) {
    // Logique pour filtrer les utilisateurs en fonction des rôles
}

Dans cet exemple, la méthode filtrerUtilisateurs prend une liste de rôles en tant que paramètre de requête. Vous pouvez appeler cette méthode avec une URL comme "/api/utilisateurs/filtrer?roles=admin&roles=utilisateur".

La gestion des paramètres de requête avec @RequestParam est une partie cruciale du développement d'applications Spring, et elle offre une manière simple et puissante de traiter les données envoyées avec les requêtes HTTP.

5.2 Corps de la requête (@RequestBody)

L'annotation @RequestBody est utilisée pour extraire les données du corps de la requête HTTP dans une méthode de contrôleur. Cela est particulièrement utile lors de l'envoi de données complexes, telles que des objets JSON, dans le corps de la requête.

5.2.1 Utilisation de @RequestBody avec des objets

Pour traiter un objet JSON envoyé dans le corps de la requête, utilisez @RequestBody avec la classe de l'objet attendu :

java
@RestController
@RequestMapping("/api/employes")
public class EmployeController {

    @PostMapping("/ajouter")
    public Employe ajouterEmploye(@RequestBody Employe employe) {
        // Logique pour ajouter un nouvel employé en utilisant les données du corps de la requête
    }

    // Autres méthodes...
}

Dans cet exemple, la méthode ajouterEmploye prend un objet Employe en tant que paramètre du corps de la requête. Lorsqu'une requête POST est effectuée avec un corps JSON contenant les données d'un employé, Spring convertit automatiquement le JSON en un objet Employe.

5.2.2 Utilisation de @RequestBody avec des types primitifs ou des chaînes

Vous pouvez également utiliser @RequestBody avec des types primitifs ou des chaînes pour extraire des données du corps de la requête :

java
@PostMapping("/mettre-a-jour-salaire")
public String mettreAJourSalaire(@RequestBody Double nouveauSalaire) {
    // Logique pour mettre à jour le salaire en utilisant les données du corps de la requête
}

Dans cet exemple, la méthode mettreAJourSalaire prend un Double en tant que paramètre du corps de la requête. Lorsqu'une requête POST est effectuée avec un corps JSON contenant un nombre à virgule flottante, Spring le convertit automatiquement en un Double.

5.2.3 Gestion des erreurs avec @Valid et BindingResult

Pour valider les données du corps de la requête, vous pouvez utiliser l'annotation @Valid en conjonction avec BindingResult. Ceci est particulièrement utile pour la validation des données d'entrée, par exemple, pour s'assurer que des champs obligatoires sont présents.

java
@PostMapping("/creer")
public ResponseEntity<String> creerUtilisateur(@Valid @RequestBody Utilisateur utilisateur, BindingResult result) {
    if (result.hasErrors()) {
        // Gérer les erreurs de validation
        return new ResponseEntity<>("Erreur de validation", HttpStatus.BAD_REQUEST);
    }

    // Logique pour créer un utilisateur en utilisant les données du corps de la requête
    return new ResponseEntity<>("Utilisateur créé avec succès", HttpStatus.CREATED);
}

Dans cet exemple, l'annotation @Valid indique à Spring de valider l'objet Utilisateur en utilisant les contraintes de validation définies dans sa classe. Les erreurs de validation sont ensuite récupérées à l'aide de BindingResult.

La gestion du corps de la requête avec @RequestBody est une fonctionnalité puissante qui permet de traiter des données complexes envoyées avec les requêtes HTTP. Elle est particulièrement utile dans le développement d'API RESTful où les clients envoient des objets JSON pour effectuer des opérations sur le serveur.
