Créer des classes et des objets en PHP est une partie fondamentale de la programmation orientée objet (POO). Voici un tutoriel complet et approfondi sur les classes et les objets en PHP.

1. Introduction à la POO en PHP

La programmation orientée objet (POO) est un paradigme de programmation qui utilise des objets, des classes et des méthodes. En PHP, la POO offre une manière structurée et modulaire de développer des applications.

2. Définir une classe en PHP

Une classe est un modèle pour les objets. Elle définit les propriétés et les méthodes que les objets créés à partir de cette classe auront.

php
<?php
class MaClasse {
    // Propriétés (attributs)
    public $nom;
    public $age;

    // Méthodes
    public function afficherInfos() {
        echo "Nom: {$this->nom}, Age: {$this->age}";
    }
}
?>

3. Instancier un objet

L'instanciation crée un objet à partir d'une classe.

php
<?php
// Instanciation d'un objet
$monObjet = new MaClasse();

// Accès aux propriétés et appel de méthodes
$monObjet->nom = "John Doe";
$monObjet->age = 30;
$monObjet->afficherInfos();
?>

4. Les propriétés et méthodes

  • Propriétés : Ce sont des variables définies à l'intérieur d'une classe. Elles représentent les caractéristiques de l'objet.

  • Méthodes : Ce sont des fonctions définies à l'intérieur d'une classe. Elles représentent le comportement de l'objet.

5. Encapsulation

Encapsulation signifie restreindre l'accès aux propriétés et méthodes d'une classe. En PHP, cela est souvent réalisé en utilisant les modificateurs d'accès tels que public, private et protected.

php
<?php
class MaClasse {
    private $nom; // accessible uniquement à l'intérieur de la classe

    public function setNom($nouveauNom) {
        $this->nom = $nouveauNom;
    }

    public function getNom() {
        return $this->nom;
    }
}
?>

6. Héritage

L'héritage permet à une classe d'hériter des propriétés et méthodes d'une autre classe.

php
<?php
class Personne {
    public $nom;
    public $age;

    public function afficherInfos() {
        echo "Nom: {$this->nom}, Age: {$this->age}";
    }
}

class Employe extends Personne {
    public $salaire;

    public function afficherInfos() {
        parent::afficherInfos(); // Appel de la méthode parente
        echo ", Salaire: {$this->salaire}";
    }
}
?>

7. Polymorphisme

Le polymorphisme permet à une classe d'implémenter une méthode de manière différente.

php
<?php
interface Forme {
    public function calculerSurface();
}

class Carre implements Forme {
    private $cote;

    public function __construct($cote) {
        $this->cote = $cote;
    }

    public function calculerSurface() {
        return $this->cote * $this->cote;
    }
}

class Cercle implements Forme {
    private $rayon;

    public function __construct($rayon) {
        $this->rayon = $rayon;
    }

    public function calculerSurface() {
        return pi() * $this->rayon * $this->rayon;
    }
}
?>

Vous pouvez explorer davantage ces concepts et les appliquer à des projets plus complexes pour améliorer votre compréhension de la programmation orientée objet.

1. Encapsulation

L'encapsulation signifie regrouper les données (propriétés) et les méthodes qui les manipulent au sein d'une classe, en limitant l'accès à ces éléments depuis l'extérieur de la classe. Les modificateurs d'accès tels que public, private, et protected sont utilisés pour définir la visibilité des membres de la classe.

php
<?php
class MaClasse {
    private $attributPrive;

    public function setAttributPrive($valeur) {
        $this->attributPrive = $valeur;
    }

    public function getAttributPrive() {
        return $this->attributPrive;
    }
}
?>

2. Héritage

L'héritage permet à une classe d'hériter des propriétés et méthodes d'une autre classe. Cela favorise la réutilisation du code.

php
<?php
class Animal {
    protected $nom;

    public function manger() {
        echo "L'animal mange.";
    }
}

class Chien extends Animal {
    public function aboyer() {
        echo "Le chien aboie.";
    }
}
?>

3. Polymorphisme

Le polymorphisme permet à des objets de différentes classes d'être traités comme des objets d'une même classe parente. Cela se manifeste souvent par l'implémentation de méthodes avec la même signature dans des classes différentes.

php
<?php
interface Forme {
    public function calculerSurface();
}

class Carre implements Forme {
    private $cote;

    public function __construct($cote) {
        $this->cote = $cote;
    }

    public function calculerSurface() {
        return $this->cote * $this->cote;
    }
}

class Cercle implements Forme {
    private $rayon;

    public function __construct($rayon) {
        $this->rayon = $rayon;
    }

    public function calculerSurface() {
        return pi() * $this->rayon * $this->rayon;
    }
}
?>

4. Utilisation pratique

Mise en œuvre d'un exemple concret pour montrer comment encapsulation, héritage et polymorphisme peuvent être utilisés ensemble.

php
<?php
class Animal {
    protected $nom;

    public function __construct($nom) {
        $this->nom = $nom;
    }

    public function manger() {
        echo "{$this->nom} mange.";
    }
}

class Chien extends Animal {
    public function aboyer() {
        echo "{$this->nom} aboie.";
    }
}

class Chat extends Animal {
    public function miauler() {
        echo "{$this->nom} miaule.";
    }
}

// Utilisation d'héritage et de polymorphisme
$chien = new Chien("Rex");
$chat = new Chat("Whiskers");

$chien->manger(); // Appel de la méthode de la classe parente
$chien->aboyer(); // Appel de la méthode de la classe enfant

$chat->manger();
$chat->miauler();
?>

1. Interfaces

Les interfaces définissent des contrats que les classes doivent implémenter. Elles déclarent les méthodes qu'une classe doit fournir, mais ne contiennent pas d'implémentation.

php
<?php
interface Forme {
    public function calculerSurface();
    public function calculerPerimetre();
}

class Carre implements Forme {
    private $cote;

    public function __construct($cote) {
        $this->cote = $cote;
    }

    public function calculerSurface() {
        return $this->cote * $this->cote;
    }

    public function calculerPerimetre() {
        return 4 * $this->cote;
    }
}
?>

2. Traits

Les traits permettent la réutilisation horizontale de méthodes dans plusieurs classes. Ils sont semblables aux classes, mais ne peuvent pas être instanciés directement.

php
<?php
trait Enregistrement {
    public function sauvegarder() {
        // Logique de sauvegarde
        echo "Enregistrement sauvegardé.";
    }
}

class Utilisateur {
    use Enregistrement;

    public function __construct($nom) {
        $this->nom = $nom;
    }
}

class Produit {
    use Enregistrement;

    public function __construct($nom) {
        $this->nom = $nom;
    }
}
?>

3. Utilisation pratique des interfaces et des traits

Combinaison d'interfaces et de traits pour créer une structure flexible.

php
<?php
interface A {
    public function methodeA();
}

interface B {
    public function methodeB();
}

trait C {
    public function methodeC() {
        echo "Méthode C";
    }
}

class MaClasse implements A, B {
    use C;

    public function methodeA() {
        echo "Méthode A";
    }

    public function methodeB() {
        echo "Méthode B";
    }
}

$obj = new MaClasse();
$obj->methodeA();
$obj->methodeB();
$obj->methodeC();
?>

4. Interfaces et traits en action

Combinaison d'interfaces et de traits dans un exemple pratique.

php
<?php
interface Enregistrement {
    public function sauvegarder();
}

trait Log {
    public function journaliser() {
        echo "Enregistrement sauvegardé dans le journal.";
    }
}

class Utilisateur implements Enregistrement {
    use Log;

    public function sauvegarder() {
        echo "Utilisateur sauvegardé.";
    }
}

$user = new Utilisateur();
$user->sauvegarder(); // Utilisation de la méthode de l'interface
$user->journaliser(); // Utilisation de la méthode du trait
?>
