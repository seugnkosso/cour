1. Qu'est-ce que Swagger ?

Swagger est un ensemble d'outils open-source pour la conception, la création, la documentation et la consommation de services web RESTful. Il fournit un ensemble de spécifications standard pour décrire l'interface d'un service web, permettant ainsi aux développeurs de comprendre rapidement comment interagir avec le service sans accéder au code source.

Swagger utilise le langage de description API OpenAPI (anciennement connu sous le nom de Swagger Specification) pour décrire la structure de l'API.

2. Objectif de Swagger PHP

Swagger PHP est une bibliothèque qui permet de générer automatiquement la documentation Swagger pour une API REST développée en PHP. L'objectif principal est de simplifier le processus de documentation en générant automatiquement des spécifications OpenAPI à partir du code PHP.

3. Avantages de l'utilisation de Swagger PHP

  • Documentation Automatique : Swagger PHP génère automatiquement la documentation de l'API à partir des annotations du code PHP, réduisant ainsi la charge de travail manuelle.

  • Synchronisation avec le Code : En utilisant Swagger PHP, la documentation reste synchronisée avec le code source, évitant ainsi les problèmes de désynchronisation entre le code et la documentation.

  • Facilité d'utilisation : Swagger UI permet aux développeurs d'explorer et de tester l'API directement à partir de la documentation, ce qui facilite le processus de développement.

4. Installation de Swagger PHP

4.1. Composer

Assurez-vous que Composer est installé sur votre machine. Si ce n'est pas le cas, suivez les instructions sur getcomposer.org.

composer require zircote/swagger-php

4.3. Utilisation dans le Code PHP

php
<?php

require 'vendor/autoload.php';

use OpenApi\Annotations as OA;

/**
 * @OA\Info(
 *     title="Mon API",
 *     version="1.0.0"
 * )
 */
class MonApi
{
    /**
     * @OA\Get(
     *     path="/endpoint",
     *     summary="Description de l'endpoint",
     *     @OA\Response(response="200", description="Réponse OK")
     * )
     */
    public function endpoint()
    {
        // Logique de l'endpoint
    }
}

// Génération de la documentation au format JSON
$swagger = \OpenApi\scan(['MonApi']);
file_put_contents('swagger.json', $swagger);

2. Configuration de Swagger PHP

2.1. Configuration de base

La configuration de base de Swagger PHP peut se faire en initialisant un objet de configuration. Vous pouvez le faire dans le fichier d'entrée de votre application ou dans un point central de votre code.

php
<?php

use OpenApi\Annotations as OA;

// Initialisation de la configuration de base
$config = [
    'swagger' => '2.0',
    'info' => [
        'title' => 'Mon API',
        'version' => '1.0.0',
    ],
];

// Autres configurations possibles...

// Génération de la documentation au format JSON
$swagger = \OpenApi\scan(['MonApi'], $config);
file_put_contents('swagger.json', $swagger);

Dans cet exemple, nous avons défini le titre et la version de l'API. Vous pouvez ajouter d'autres configurations selon vos besoins.

2.2. Configuration avancée

Swagger PHP offre des options avancées de configuration pour personnaliser la documentation générée. Voici un exemple avec des configurations avancées :

php
<?php

use OpenApi\Annotations as OA;

// Configuration avancée
$config = [
    'swagger' => '2.0',
    'info' => [
        'title' => 'Mon API',
        'version' => '1.0.0',
    ],
    'host' => 'api.example.com',
    'schemes' => ['https'],
    'basePath' => '/v1',
    'produces' => ['application/json'],
    'consumes' => ['application/json'],
    'securityDefinitions' => [
        'apiKey' => [
            'type' => 'apiKey',
            'name' => 'Authorization',
            'in' => 'header',
        ],
    ],
];

// Génération de la documentation au format JSON
$swagger = \OpenApi\scan(['MonApi'], $config);
file_put_contents('swagger.json', $swagger);

Dans cet exemple, nous avons ajouté des configurations telles que host, schemes, basePath, produces, consumes, et securityDefinitions. Vous pouvez adapter ces configurations en fonction des besoins spécifiques de votre API.

2.3. Annotations Swagger PHP

Swagger PHP utilise des annotations spéciales pour décrire les éléments de votre API. Voici quelques-unes des annotations les plus couramment utilisées :

  • @OA\Info : Informations générales sur l'API (titre, version, etc.).
  • @OA\Get, @OA\Post, @OA\Put, @OA\Delete, etc. : Annotations pour définir les opérations HTTP.
  • @OA\Parameter : Définit les paramètres d'une opération.
  • @OA\Response : Définit les réponses possibles d'une opération.
  • @OA\Schema : Définit la structure d'un objet JSON.

Exemple d'utilisation d'annotations dans une classe PHP :

php
/**
 * @OA\Info(
 *     title="Mon API",
 *     version="1.0.0"
 * )
 */
class MonApi
{
    /**
     * @OA\Get(
     *     path="/endpoint",
     *     summary="Description de l'endpoint",
     *     @OA\Response(response="200", description="Réponse OK")
     * )
     */
    public function endpoint()
    {
        // Logique de l'endpoint
    }
}

Ces annotations permettent à Swagger PHP de générer la documentation correcte pour chaque endpoint.

3. Annotations Swagger PHP

Swagger PHP utilise un ensemble d'annotations spéciales pour documenter les différentes parties de votre API. Voici les annotations les plus couramment utilisées :

3.1. @SWG\Swagger

L'annotation @SWG\Swagger définit les informations générales sur l'API. Elle est généralement placée au-dessus de la classe principale de votre API.

php
/**
 * @SWG\Swagger(
 *     basePath="/api",
 *     host="api.example.com",
 *     schemes={"http", "https"},
 *     @SWG\Info(
 *         title="Mon API",
 *         version="1.0.0"
 *     )
 * )
 */
class MonApi
{
    // ...
}

3.2. @SWG\Info

L'annotation @SWG\Info fournit des informations générales sur votre API, telles que le titre et la version.

php
/**
 * @SWG\Info(
 *     title="Mon API",
 *     version="1.0.0"
 * )
 */
class MonApi
{
    // ...
}

3.3. @SWG\Definition

L'annotation @SWG\Definition est utilisée pour définir la structure d'un modèle de données (par exemple, un objet JSON).

php
/**
 * @SWG\Definition(
 *     definition="Utilisateur",
 *     required={"id", "nom"},
 *     @SWG\Property(property="id", type="integer"),
 *     @SWG\Property(property="nom", type="string")
 * )
 */
class MonApi
{
    // ...
}

3.4. @SWG\Parameter

L'annotation @SWG\Parameter définit les paramètres d'une opération.

php
/**
 * @SWG\Get(
 *     path="/utilisateur/{id}",
 *     summary="Obtenir un utilisateur par ID",
 *     @SWG\Parameter(
 *         name="id",
 *         in="path",
 *         type="integer",
 *         description="ID de l'utilisateur"
 *     ),
 *     @SWG\Response(response="200", description="Utilisateur trouvé"),
 *     @SWG\Response(response="404", description="Utilisateur non trouvé")
 * )
 */
public function getUtilisateur($id)
{
    // Logique de l'opération
}

3.5. @SWG\Response

L'annotation @SWG\Response définit les réponses possibles d'une opération.

php
/**
 * @SWG\Get(
 *     path="/utilisateur/{id}",
 *     summary="Obtenir un utilisateur par ID",
 *     @SWG\Parameter(
 *         name="id",
 *         in="path",
 *         type="integer",
 *         description="ID de l'utilisateur"
 *     ),
 *     @SWG\Response(response="200", description="Utilisateur trouvé"),
 *     @SWG\Response(response="404", description="Utilisateur non trouvé")
 * )
 */
public function getUtilisateur($id)
{
    // Logique de l'opération
}

3.6. @SWG\Schema

L'annotation @SWG\Schema est utilisée pour définir la structure d'un objet JSON.

php
/**
 * @SWG\Schema(
 *     schema="Utilisateur",
 *     required={"id", "nom"},
 *     @SWG\Property(property="id", type="integer"),
 *     @SWG\Property(property="nom", type="string")
 * )
 */
class MonApi
{
    // ...
}

3.7. @SWG\Tag

L'annotation @SWG\Tag permet de regrouper les opérations dans la documentation.

php
/**
 * @SWG\Get(
 *     path="/utilisateur/{id}",
 *     summary="Obtenir un utilisateur par ID",
 *     @SWG\Parameter(
 *         name="id",
 *         in="path",
 *         type="integer",
 *         description="ID de l'utilisateur"
 *     ),
 *     @SWG\Response(response="200", description="Utilisateur trouvé"),

{
    // Logique de l'opération

3.8. @SWG\SecurityScheme

L'annotation @SWG\SecurityScheme permet de définir les schémas de sécurité utilisés par l'API.

php
/**

 *     securityDefinition="apiKey",
 *     type="apiKey",

    // ...

4. Documentation des API avec Swagger PHP

4.1. Documentation automatique des endpoints

Swagger PHP permet de générer automatiquement la documentation des endpoints en utilisant des annotations spéciales dans le code PHP. Dans l'exemple suivant, nous définissons un endpoint GET pour obtenir un utilisateur par ID :

php
/**
 * @SWG\Get(
 *     path="/utilisateur/{id}",
 *     summary="Obtenir un utilisateur par ID",
 *     @SWG\Parameter(
 *         name="id",
 *         in="path",
 *         type="integer",
 *         description="ID de l'utilisateur"
 *     ),
 *     @SWG\Response(response="200", description="Utilisateur trouvé"),
 *     @SWG\Response(response="404", description="Utilisateur non trouvé"),
 *     tags={"Utilisateurs"}
 * )
 */
public function getUtilisateur($id)
{
    // Logique de l'opération
}

Les annotations spécifient le chemin de l'endpoint, le type d'opération (GET dans cet exemple), les paramètres, les réponses possibles, et les tags pour regrouper les opérations dans la documentation.

4.2. Personnalisation de la documentation

Vous pouvez personnaliser la documentation générée en utilisant des configurations avancées dans le fichier d'initialisation de Swagger PHP. Par exemple :

php
// Configuration avancée
$config = [
    'swagger' => '2.0',
    'info' => [
        'title' => 'Mon API',
        'version' => '1.0.0',
    ],
    'host' => 'api.example.com',
    'schemes' => ['https'],
    'basePath' => '/v1',
    'produces' => ['application/json'],
    'consumes' => ['application/json'],
    'securityDefinitions' => [
        'apiKey' => [
            'type' => 'apiKey',
            'name' => 'Authorization',
            'in' => 'header',
        ],
    ],
];

// Génération de la documentation au format JSON
$swagger = \OpenApi\scan(['MonApi'], $config);
file_put_contents('swagger.json', $swagger);

Dans cet exemple, nous avons défini des paramètres tels que host, schemes, basePath, produces, consumes, et securityDefinitions pour personnaliser la documentation générée.

4.3. Génération de la documentation au format JSON et YAML

Swagger PHP génère la documentation au format JSON par défaut. Vous pouvez également générer la documentation au format YAML en utilisant l'option appropriée lors de la génération :

php
// Génération de la documentation au format YAML
$swaggerYaml = \OpenApi\scan(['MonApi'], $config);
file_put_contents('swagger.yaml', $swaggerYaml);

Pour cela, assurez-vous d'avoir l'extension YAML installée (ext-yaml).

4.4. Intégration de la documentation Swagger dans l'application

Vous pouvez intégrer la documentation Swagger directement dans votre application, par exemple, en utilisant Swagger UI. Téléchargez Swagger UI et déployez-le sur votre serveur web. Ensuite, référencez le fichier JSON ou YAML généré par Swagger PHP dans Swagger UI.

html
<!DOCTYPE html>
<html>
<head>
    <title>Documentation de l'API</title>
    <link rel="stylesheet" type="text/css" href="swagger-ui.css">
</head>
<body>
    <div id="swagger-ui"></div>
    <script src="swagger-ui-bundle.js"></script>
    <script src="swagger-ui-standalone-preset.js"></script>
    <script>
        window.onload = function() {
            // Charge la documentation Swagger
            SwaggerUIBundle({
                url: "swagger.json",  // ou "swagger.yaml" si au format YAML
                dom_id: '#swagger-ui',
                presets: [
                    SwaggerUIBundle.presets.apis,
                    SwaggerUIStandalonePreset
                ],
                layout: "StandaloneLayout"
            });
        }
    </script>
</body>
</html>

Remplacez les références des fichiers CSS et JS par les bons chemins sur votre serveur. Cette intégration permet aux développeurs d'explorer et de tester l'API directement à partir de la documentation Swagger.

Ceci conclut le tutoriel complet sur la documentation des API avec Swagger PHP. Utilisez ces informations pour améliorer la documentation de votre API et faciliter la collaboration avec d'autres développeurs.

5. Validation des Données avec Swagger PHP

La validation des données est essentielle pour garantir que votre API fonctionne correctement et que les utilisateurs la consomment de manière appropriée. Swagger PHP facilite la validation des paramètres de requête, des réponses, et offre des mécanismes pour gérer les erreurs de validation.

5.1. Validation des paramètres de requête

Utilisez l'annotation @SWG\Parameter pour définir les paramètres de requête et spécifiez les règles de validation à l'aide de l'annotation @SWG\Schema. Par exemple, pour valider un paramètre d'ID utilisateur :

php
/**
 * @SWG\Get(
 *     path="/utilisateur/{id}",
 *     summary="Obtenir un utilisateur par ID",
 *     @SWG\Parameter(
 *         name="id",
 *         in="path",
 *         type="integer",
 *         description="ID de l'utilisateur",
 *         minimum=1
 *     ),
 *     @SWG\Response(response="200", description="Utilisateur trouvé"),
 *     @SWG\Response(response="404", description="Utilisateur non trouvé"),
 *     tags={"Utilisateurs"}
 * )
 */
public function getUtilisateur($id)
{
    // Logique de l'opération
}

Dans cet exemple, minimum=1 spécifie que l'ID de l'utilisateur doit être au moins égal à 1. Vous pouvez utiliser d'autres règles de validation telles que maximum, pattern, etc.

5.2. Validation des réponses

Pour valider les réponses, utilisez l'annotation @SWG\Response et spécifiez le schéma de validation à l'aide de @SWG\Schema. Par exemple, pour valider une réponse JSON d'utilisateur :

php
/**
 * @SWG\Get(
 *     path="/utilisateur/{id}",
 *     summary="Obtenir un utilisateur par ID",
 *     @SWG\Parameter(
 *         name="id",
 *         in="path",
 *         type="integer",
 *         description="ID de l'utilisateur",
 *         minimum=1
 *     ),
 *     @SWG\Response(
 *         response="200",
 *         description="Utilisateur trouvé",
 *         @SWG\Schema(
 *             type="object",
 *             @SWG\Property(property="id", type="integer"),
 *             @SWG\Property(property="nom", type="string")
 *         )
 *     ),
 *     @SWG\Response(response="404", description="Utilisateur non trouvé"),
 *     tags={"Utilisateurs"}
 * )
 */
public function getUtilisateur($id)
{
    // Logique de l'opération
    // Retourne un objet utilisateur conforme au schéma spécifié
}

Dans cet exemple, le schéma spécifie que la réponse doit être un objet JSON avec les propriétés id de type integer et nom de type string.

5.3. Gestion des erreurs de validation

Swagger PHP ne gère pas directement les erreurs de validation, mais vous pouvez ajouter une logique de gestion des erreurs dans votre code. Par exemple, si la validation d'un paramètre échoue, lancez une exception et traitez-la dans un gestionnaire global des exceptions.

php
/**
 * @SWG\Get(
 *     path="/utilisateur/{id}",
 *     summary="Obtenir un utilisateur par ID",
 *     @SWG\Parameter(
 *         name="id",
 *         in="path",
 *         type="integer",
 *         description="ID de l'utilisateur",
 *         minimum=1
 *     ),
 *     @SWG\Response(response="200", description="Utilisateur trouvé"),
 *     @SWG\Response(response="404", description="Utilisateur non trouvé"),
 *     tags={"Utilisateurs"}
 * )
 */
public function getUtilisateur($id)
{
    if ($id < 1) {
        // Lancez une exception si l'ID n'est pas valide
        throw new InvalidArgumentException("L'ID de l'utilisateur doit être supérieur à zéro.");
    }

    // Logique de l'opération
}

Dans ce cas, une exception InvalidArgumentException sera lancée si l'ID de l'utilisateur n'est pas valide. Vous pouvez ensuite ajouter un gestionnaire d'exceptions global pour traiter ces erreurs et renvoyer une réponse appropriée à l'utilisateur.

Ceci conclut le tutoriel sur la validation des données avec Swagger PHP. Utilisez ces techniques pour garantir que votre API respecte les contraintes spécifiées et fournisse des réponses cohérentes et fiables.

6. Sécurité avec Swagger PHP

La sécurité est une préoccupation majeure lors du développement d'une API. Swagger PHP offre des fonctionnalités intégrées pour configurer les schémas de sécurité, utiliser JWT (JSON Web Tokens) pour l'authentification, et gérer les autorisations.

6.1. Configuration des schémas de sécurité

Utilisez l'annotation @SWG\SecurityScheme pour configurer les schémas de sécurité dans Swagger PHP. Par exemple, pour configurer un schéma d'authentification par clé API :

php
/**
 * @SWG\SecurityScheme(
 *     securityDefinition="apiKey",
 *     type="apiKey",
 *     in="header",
 *     name="Authorization"
 * )
 */
class MonApi
{
    // ...
}

Dans cet exemple, le schéma de sécurité apiKey utilise le type "apiKey" dans l'en-tête avec le nom "Authorization". Vous pouvez personnaliser ces paramètres en fonction de vos besoins.

6.2. Utilisation de JWT (JSON Web Tokens) pour l'authentification

Si vous utilisez JWT pour l'authentification, vous pouvez configurer le schéma de sécurité correspondant. Par exemple, pour un schéma d'authentification JWT :

php
/**
 * @SWG\SecurityScheme(
 *     securityDefinition="jwt",
 *     type="apiKey",
 *     in="header",
 *     name="Authorization"
 * )
 */
class MonApi
{
    // ...
}

Assurez-vous d'ajuster la configuration JWT en fonction de la manière dont vous générez et utilisez les tokens JWT dans votre application.

6.3. Gestion des autorisations

Swagger PHP permet également de spécifier les autorisations nécessaires pour accéder à certaines parties de votre API. Utilisez l'annotation @SWG\Operation pour spécifier les autorisations nécessaires pour une opération spécifique. Par exemple, pour spécifier que l'accès à une opération nécessite un token JWT valide :

php
/**
 * @SWG\Get(
 *     path="/ressource-protegee",
 *     summary="Récupérer une ressource protégée",
 *     @SWG\Response(response="200", description="Ressource récupérée"),
 *     @SWG\Response(response="401", description="Non autorisé"),
 *     @SWG\Operation(
 *         security={
 *             {"jwt": {}}
 *         }
 *     )
 * )
 */
public function getRessourceProtegee()
{
    // Logique de l'opération
}

Dans cet exemple, la section security spécifie que l'accès à cette opération nécessite un token JWT valide. Vous pouvez adapter cette configuration en fonction de vos besoins d'autorisation spécifiques.

Ceci conclut le tutoriel sur la sécurité avec Swagger PHP. Utilisez ces fonctionnalités pour sécuriser votre API en définissant des schémas de sécurité, en utilisant JWT pour l'authentification, et en spécifiant les autorisations nécessaires pour chaque opération.
