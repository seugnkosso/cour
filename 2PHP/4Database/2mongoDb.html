1.1 Introduction à MongoDB

1.1.1 Qu'est-ce que MongoDB ?

MongoDB est une base de données NoSQL, orientée document, qui stocke les données sous forme de documents BSON (Binary JSON). Il offre une flexibilité exceptionnelle et permet de stocker des données non structurées ou semi-structurées.

1.1.2 Avantages de l'utilisation de MongoDB avec PHP

  • Schémas Flexibles : Contrairement aux bases de données relationnelles, MongoDB n'impose pas de schéma fixe, ce qui permet une évolution facile de la structure des données.
  • Haute Performance : Les opérations de lecture/écriture sont rapides, et il prend en charge le sharding pour la mise à l'échelle horizontale.
  • Géospatial : MongoDB offre des fonctionnalités intégrées pour le traitement des données géospatiales.
  • Évolutivité : Possibilité de distribuer les données sur plusieurs serveurs.

1.1.3 Installation de MongoDB et des extensions PHP nécessaires

  • Installation de MongoDB : Suivez les instructions d'installation sur le site officiel de MongoDB : Install MongoDB

  • Installation des extensions PHP : Utilisez Composer pour installer les extensions MongoDB pour PHP.

    bash
    composer require mongodb/mongodb

1.2 Fondamentaux de MongoDB

1.2.1 Collections et documents

  • Collections : Les collections sont des groupes de documents MongoDB. Chaque document dans une collection peut avoir une structure différente.

  • Documents : Les données sont stockées sous forme de documents BSON (similaires à JSON). Chaque document est une unité de données dans une collection.

1.2.2 Schémas flexibles

MongoDB n'a pas de schéma rigide. Vous pouvez ajouter des champs à vos documents à tout moment, sans avoir à mettre à jour le schéma de la base de données.

1.2.3 Opérations CRUD de base

  • Create (Création) :

    php
    $collection->insertOne(['name' => 'John Doe', 'age' => 30]);

  • Read (Lecture) :

    php
    $result = $collection->findOne(['name' => 'John Doe']);

  • Update (Mise à jour) :

    php
    $collection->updateOne(['name' => 'John Doe'], ['$set' => ['age' => 31]]);

  • Delete (Suppression) :

    php
    $collection->deleteOne(['name' => 'John Doe']);

Ceci marque le début de l'intégration de MongoDB avec PHP. Vous pouvez maintenant explorer davantage les fonctionnalités avancées, telles que les index, les opérations de groupe, les agrégations, etc. N'oubliez pas de consulter la documentation officielle de MongoDB et de l'extension PHP pour des informations plus détaillées.

2.1 Configuration de la connexion à MongoDB

2.1.1 Configuration du fichier php.ini

Avant de commencer à utiliser MongoDB avec PHP, assurez-vous que vous avez correctement configuré votre fichier php.ini pour inclure l'extension MongoDB. Suivez ces étapes :

  1. Trouver le fichier php.ini : Vous pouvez localiser le fichier php.ini en exécutant la commande suivante dans le terminal (pour Linux) ou l'invite de commandes (pour Windows) :

    bash
    php --ini

    Cela vous donnera le chemin du fichier php.ini.

  2. Modifier php.ini : Ouvrez le fichier php.ini dans un éditeur de texte et assurez-vous que la ligne suivante est décommentée (c'est-à-dire, sans le point-virgule au début) :

    ini
    extension=mongodb

    Sauvegardez le fichier et redémarrez votre serveur PHP.

2.1.2 Connexion à MongoDB avec MongoClient

La classe MongoClient était utilisée dans les versions antérieures de l'extension MongoDB pour PHP. Cependant, à partir de la version 1.0.0, elle a été remplacée par MongoDB\Client. Utilisez la version la plus récente pour bénéficier des dernières fonctionnalités et correctifs.

php

$mongoClient = new MongoClient('mongodb://localhost:27017');

// Sélection d'une base de données

// ...
?>

$collection->updateMany(['age' => ['$gt' => 30]], ['$inc' => ['age' => 1]]);

// ...
?>

4.1 Indexation dans MongoDB

4.1.1 Création d'index simples

Les index peuvent améliorer considérablement les performances des requêtes en accélérant l'accès aux données. Voici comment créer un index simple :

php
<?php

// Création d'un index simple sur le champ "nom"
$collection->createIndex(['nom' => 1]);

// ...
?>

Dans cet exemple, 1 indique un index ascendant, et -1 indiquerait un index descendant.

4.1.2 Index composite et index textuels

Un index composite peut être créé sur plusieurs champs pour optimiser les requêtes qui utilisent ces champs en combinaison.

php
<?php

// Création d'un index composite sur les champs "nom" et "age"

// ...
?>

Pour les recherches de texte, vous pouvez utiliser un index textuel pour une recherche plus efficace :

php
<?php

// Création d'un index textuel sur le champ "contenu"

// Recherche textuelle
$results = $collection->find(['$text' => ['$search' => 'mot_clé']]);
foreach ($results as $document) {
    print_r($document);
}

// ...
?>

4.2.2 Utilisation de l'agrégation pour des performances optimales

L'agrégation est puissante pour manipuler les données de manière avancée. Utilisez des pipelines d'agrégation pour effectuer des opérations complexes côté serveur et minimiser les données transférées.

php
<?php

// Exemple d'utilisation de l'agrégation
$pipeline = [
    ['$match' => ['age' => ['$gt' => 30]]],
    ['$group' => ['_id' => '$nom', 'total' => ['$sum' => 1]]],
    ['$sort' => ['total' => -1]],
    ['$limit' => 5]
];

$aggregationResult = $collection->aggregate($pipeline);
foreach ($aggregationResult as $result) {
    print_r($result);
}

// ...
?>

L'agrégation permet de réaliser des opérations complexes directement dans la base de données, réduisant ainsi la charge sur l'application PHP.

6.1 Transactions avec MongoDB

6.1.1 Gestion des transactions dans MongoDB 4.0 et supérieur

MongoDB prend en charge les transactions depuis la version 4.0. Les transactions permettent d'effectuer plusieurs opérations de lecture/écriture de manière atomique.

php
<?php

// Démarrez une transaction
$session = $mongoClient->startSession();
$session->startTransaction();

try {
    // Opérations de lecture/écriture
    $collection->updateOne(['_id' => 1], ['$set' => ['nom' => 'John Doe']]);
    $collection->deleteMany(['age' => ['$gt' => 30]]);

    // Validez la transaction
    $session->commitTransaction();
} catch (Exception $e) {
    // Annulez la transaction en cas d'erreur
    $session->abortTransaction();
}

// ...
?>
