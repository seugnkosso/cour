7.1. Qu'est-ce que Psalm?

Psalm est un outil d'analyse statique pour les projets PHP. Il est conçu pour détecter des erreurs potentielles, améliorer la qualité du code et fournir des informations utiles sur la sécurité du code. Psalm utilise une analyse statique pour inspecter le code sans l'exécuter réellement, ce qui permet de repérer des erreurs avant même d'exécuter le code. Cela aide les développeurs à écrire un code plus sûr et plus robuste.

Les principales fonctionnalités de Psalm incluent la détection des types, l'analyse de la sécurité du code, la prise en charge des annotations de type PHPDoc, la vérification des appels de méthodes et des propriétés, etc.

7.2. Installation de Psalm

Pour installer Psalm, vous pouvez utiliser Composer, le gestionnaire de dépendances PHP. Voici les étapes à suivre :

  1. Assurez-vous d'avoir Composer installé. Si ce n'est pas le cas, vous pouvez le télécharger à partir de getcomposer.org.

  2. Créez un fichier composer.json s'il n'existe pas déjà dans votre projet. Ajoutez la dépendance Psalm en tant que développement :

    json
    {
    "require-dev": {
        "vimeo/psalm": "^4.0"
    }
}

    Vous pouvez vérifier la dernière version disponible sur packagist.org.

  3. Exécutez la commande Composer pour installer Psalm :

    bash
    composer install

    Ou, si vous n'avez pas encore de fichier composer.json, vous pouvez exécuter directement :

    bash
    composer require --dev vimeo/psalm:^4.0

  4. Une fois l'installation terminée, vous pouvez exécuter Psalm à l'aide de la commande :

    bash
    ./vendor/bin/psalm

    Cela exécutera Psalm et affichera les résultats de l'analyse statique de votre code.

7.3. Configuration de Psalm

Psalm propose un fichier de configuration pour personnaliser son comportement. Vous pouvez créer un fichier psalm.xml à la racine de votre projet. Voici un exemple de configuration de base :

xml
<!-- psalm.xml -->
<psalm>
    <projectFiles>
        <directory name="src" />
        <ignoreFiles>
            <directory name="vendor" />
        </ignoreFiles>
    </projectFiles>
    <!-- Ajoutez d'autres configurations selon vos besoins -->
</psalm>

Dans cet exemple, le répertoire src est spécifié comme répertoire source du projet, et le répertoire vendor est ignoré. Vous pouvez ajuster cette configuration en fonction de la structure de votre projet.

Pour en savoir plus sur la configuration de Psalm, consultez la documentation officielle sur psalm.dev.

8.1. Annotations de types

Les annotations de types sont un aspect crucial de l'utilisation de Psalm. Elles permettent de spécifier les types des variables, des paramètres de fonction et des valeurs de retour pour améliorer la précision de l'analyse statique. Voici quelques exemples d'annotations de types couramment utilisées :

  • Types scalaires :

    php
    /** @var string */
$nom = "John";

/** @var int */
$age = 25;

  • Tableaux associatifs :

    php
    /** @var array<string, int> */
$scores = ["John" => 90, "Doe" => 85];

  • Tableaux indexés :

    php
    /** @var array<int, string> */
$noms = [1 => "John", 2 => "Doe"];

  • Objets :

    php
    /** @var MyClass */
$objet = new MyClass();

  • Nullable :

    php
    /** @var string|null */
$peutEtreNull = null;

  • Types d'union :

    php
    /** @var int|string */
$valeur = 42;

  • Types personnalisés :

    php
    /** @var MonType */
$variableCustom = new MonType();

En utilisant correctement les annotations de types, vous permettez à Psalm de détecter des erreurs potentielles dans votre code et d'offrir des suggestions pour les améliorer.

8.2. Analyse des erreurs courantes

Psalm peut détecter diverses erreurs courantes dans le code PHP. Certains exemples incluent :

  • Types incompatibles :

    php
    /** @var string */
$nom = "John";

// Erreur, car $nom est de type string, et nous essayons de le concaténer avec un entier.
$resultat = $nom + 42;

  • Appels de méthodes ou propriétés inexistantes :

    php
    /** @var MyClass */
$objet = new MyClass();

// Erreur, car la méthode nonExistant n'existe pas dans MyClass.
$objet->nonExistant();

  • Variables non déclarées :

    php
    // Erreur, car $inconnu n'est pas déclaré.
echo $inconnu;

L'analyse statique avec Psalm permet de détecter ces erreurs avant l'exécution du code, améliorant ainsi la qualité du code.

8.3. Utilisation avancée de Psalm

  • Configuration avancée : Vous pouvez ajuster finement la configuration de Psalm en utilisant le fichier psalm.xml. Par exemple, vous pouvez spécifier des plugins, définir des règles spécifiques, exclure des fichiers ou des répertoires de l'analyse, etc. Consultez la documentation de Psalm Configuration pour plus de détails.

  • Annotations personnalisées : Vous pouvez créer des annotations personnalisées pour étendre les capacités de Psalm. Cela peut être utile pour décrire des contrats spécifiques dans votre code. Consultez la documentation sur Custom Annotations pour en savoir plus.

  • Intégration avec des outils CI/CD : Intégrez Psalm dans vos pipelines CI/CD pour automatiser l'analyse statique à chaque déploiement. Assurez-vous que les erreurs détectées par Psalm sont résolues avant de fusionner du code dans votre branche principale.

En conclusion, en utilisant les annotations de types, en comprenant les erreurs courantes détectées par Psalm, et en explorant ses fonctionnalités avancées, vous pouvez profiter pleinement de l'analyse statique pour améliorer la qualité de votre code PHP.

9.1. Intégration avec des outils de construction (e.g., Composer, Make)

L'intégration de Psalm dans les outils de construction de votre projet est essentielle pour automatiser l'analyse statique du code à chaque étape du processus de développement. Les deux principaux outils que nous aborderons ici sont Composer et Make.

Intégration avec Composer :

Vous pouvez ajouter une commande personnalisée dans le fichier composer.json pour exécuter Psalm. Voici un exemple :

  1. Ajoutez la commande à la section "scripts" de votre fichier composer.json :

    json
    {
    "scripts": {
        "psalm": "./vendor/bin/psalm"
    }
}

  2. Exécutez Psalm en utilisant la commande Composer :

    bash
    composer psalm

Vous pouvez également ajouter des options supplémentaires à la commande, par exemple, pour générer un rapport HTML :

json
{
    "scripts": {
        "psalm": "./vendor/bin/psalm --shepherd --report=summary --report-file=psalm-report.html"
    }
}

Intégration avec Make :

Make est un autre outil populaire pour automatiser des tâches de construction. Vous pouvez ajouter une règle dans votre fichier Makefile pour exécuter Psalm.

  1. Ajoutez une règle pour Psalm dans votre fichier Makefile :

    make
    psalm:
	./vendor/bin/psalm

  2. Exécutez Psalm avec la commande Make :

    bash
    make psalm

Ces intégrations vous permettent d'exécuter Psalm en un seul appel de commande, facilitant ainsi l'intégration dans votre processus de développement.

9.2. Intégration avec des IDE (e.g., VSCode, PhpStorm)

Intégrer Psalm avec votre IDE peut améliorer l'expérience de développement en fournissant des commentaires en temps réel sur les erreurs potentielles directement dans l'éditeur.

Intégration avec VSCode :

  1. Installez l'extension "psalm" depuis le marché VSCode.

  2. Créez un fichier psalm.xml à la racine de votre projet pour configurer Psalm selon vos besoins.

  3. L'extension détectera automatiquement le fichier psalm.xml et commencera à fournir des suggestions en temps réel.

Intégration avec PhpStorm :

  1. Installez le plugin "Psalm" depuis le marché des plugins PhpStorm.

  2. Configurez le chemin de l'exécutable Psalm dans les paramètres du plugin.

  3. L'extension analysera automatiquement votre code pendant que vous travaillez dans PhpStorm et signalera les erreurs potentielles.

L'intégration avec l'IDE vous permet de bénéficier des fonctionnalités de l'analyse statique de façon plus interactive et immédiate pendant que vous écrivez du code.

En résumé, l'intégration de Psalm dans votre processus de construction et votre IDE facilite l'analyse statique du code et contribue à améliorer la qualité du code PHP de manière continue.

10.1. Utilisation efficace des types

L'utilisation efficace des types avec Psalm est essentielle pour tirer pleinement parti de l'analyse statique. Voici quelques bonnes pratiques :

  • Utiliser des types précis : Utilisez des types aussi précis que possible pour les variables, les paramètres de fonction et les valeurs de retour. Cela permet à Psalm de détecter des erreurs potentielles plus précisément.

    php
    /** @var User */
$utilisateur = new User();

  • Types nullable : Utilisez ?Type pour indiquer que la variable peut être nulle. Cela permet à Psalm de détecter les erreurs liées à la nullité.

    php
    /** @var string|null */
$peutEtreNull = null;

  • Éviter les types mixtes : Évitez autant que possible les types mixtes. Les types mixtes (par exemple, mixed) diminuent l'utilité de l'analyse statique.

    php
    /** @var mixed */
$variableMixte = getDynamicValue();

  • Utiliser des types d'union : Lorsqu'une variable peut avoir plusieurs types, utilisez des types d'union pour informer Psalm.

    php
    /** @var int|string */
$valeur = 42;

  • Annoter les tableaux : Annoter les types des tableaux pour améliorer la précision de l'analyse.

    php
    /** @var array<string, int> */
$scores = ["John" => 90, "Doe" => 85];

10.2. Écriture de tests avec Psalm

Écrire des tests avec Psalm contribue à garantir la robustesse de votre code. Voici quelques bonnes pratiques :

  • Tests unitaires : Écrivez des tests unitaires pour chaque fonction ou méthode de votre code. Cela permet à Psalm de détecter les erreurs potentielles liées à des types incorrects ou à des appels de méthode/propriété inexistants.

  • Annotations dans les tests : Utilisez des annotations de types dans vos tests pour décrire les types attendus et garantir que votre code est correctement testé.

    php
    /**
 * @test
 * @return void
 */
public function testSomeFunction(): void {
    /** @var MyClass */
    $objet = new MyClass();

    // ...
}

  • Intégration des annotations dans les assertions : Lorsque vous utilisez des assertions dans vos tests, ajoutez des annotations de types pour informer Psalm sur les types attendus.

    php
    /** @var MyClass */
$resultat = $myObject->someMethod();

// Assert that $result is an instance of MyClass
$this->assertInstanceOf(MyClass::class, $result);

10.3. Intégration continue et Psalm

L'intégration continue (CI) est un élément clé pour maintenir la qualité du code tout au long du développement. Intégrez Psalm dans vos workflows CI pour automatiser l'analyse statique.

  • Configuration de CI avec Composer : Ajoutez une étape dans votre fichier de configuration CI pour exécuter Psalm après l'installation des dépendances.

    Exemple pour GitHub Actions :

    yaml
    name: Tests

on:
  push:
    branches:
      - main

jobs:
  tests:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v2

      - name: Install dependencies
        run: composer install --prefer-dist --no-interaction

      - name: Run Psalm
        run: ./vendor/bin/psalm

  • Analyse de la sortie de Psalm : Configurez votre workflow CI pour analyser la sortie de Psalm et échouer si des erreurs sont détectées. Cela garantit que les erreurs potentielles ne sont pas introduites dans le code principal.

    yaml
    - name: Run Psalm
  run: |
    if ./vendor/bin/psalm; then
        echo "Psalm analysis passed"
    else
        echo "Psalm analysis failed"
        exit 1
    fi

  • Rapports de couverture : Si vous utilisez des outils de couverture de code comme PHPUnit, configurez votre CI pour générer et analyser ces rapports en parallèle avec Psalm.

    yaml
    - name: Run PHPUnit and Psalm
  run: |
    ./vendor/bin/phpunit
    ./vendor/bin/psalm

L'intégration continue avec Psalm garantit que votre code est analysé à chaque étape du développement, offrant ainsi une assurance continue de la qualité du code.
